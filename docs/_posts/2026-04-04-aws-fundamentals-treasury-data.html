Why AWS IAM Matters From Day One

When you create an AWS account, you're given a root user. Root can do anything: delete the account, spend unlimited money, revoke other users' access. You should never use root for daily work. Think of it like using sudo for every terminal command — it works, but it's reckless.

Instead, create an IAM (Identity and Access Management) user with only the permissions it needs. For this learning project, I created a user called finbytes with broad managed policies (S3, RDS, Lambda, etc.) since I'm exploring many services. In production, you'd scope permissions much tighter.

The IAM user gets its own access keys, which you use to configure the AWS CLI:

aws configure --profile quant-lab
#   Access Key ID:     AKIA...
#   Secret Access Key: ...
#   Region:            us-east-1
#   Output format:     json

aws sts get-caller-identity --profile quant-lab

The --profile flag means you can have multiple AWS accounts (personal, work, different projects) on the same machine without stepping on each other.

The $5 Safety Net

AWS has a generous free tier, but it's easy to accidentally leave something running. A single NAT Gateway left on for a month is $32. An RDS instance running past the free tier is about $15/month. The cheapest insurance is a CloudWatch billing alarm that emails you when estimated charges cross a threshold.

Setting one up requires an SNS topic (for the notification), a confirmed email subscription, and a CloudWatch alarm on the EstimatedCharges metric. Importantly, billing metrics only exist in us-east-1, so the alarm must be created there even if you're using other regions for your actual workloads.

Enter FRED: Free, Reliable Market Data

FRED (Federal Reserve Economic Data) is a goldmine. Maintained by the St. Louis Fed, it exposes over 800,000 economic and financial time series through a free API. For fixed income work, it's the cleanest source of daily US Treasury yields across every major maturity.

The series follow a predictable naming convention:

TREASURY_SERIES = {
    "DGS1MO": "1M",   # 1-month T-bill
    "DGS3MO": "3M",
    "DGS6MO": "6M",
    "DGS1": "1Y",
    "DGS2": "2Y",
    "DGS3": "3Y",
    "DGS5": "5Y",
    "DGS7": "7Y",
    "DGS10": "10Y",   # the benchmark
    "DGS20": "20Y",
    "DGS30": "30Y",
}

What Par Yields Actually Represent

When you see "the 10Y Treasury yield is 4.31%", that's the par yield — the coupon rate at which a newly issued 10-year Treasury bond would price at exactly 100 (face value). Different maturities have different par yields, and together they form the par curve — the starting point for every other yield curve.

The shape of the curve tells you something about the economy:

  • Normal (upward sloping): long-term rates exceed short-term. Investors demand a premium to lock money up longer. This is the baseline healthy state.
  • Inverted: short rates exceed long rates. Historically predicts recession — the market expects the central bank to cut rates ahead, so long rates have priced that in.
  • Flat: rates similar across maturities. Transition periods, policy uncertainty.
  • Humped: mid-term peak. Unusual, often reflects specific policy expectations around particular horizons.

Fetching the Curve in Python

The fredapi library wraps FRED's REST API. Fetching a single series is one line; fetching the whole curve is a loop:

from datetime import date, timedelta
from fredapi import Fred


def fetch_par_yields(api_key: str, as_of: date | None = None) -> dict[str, float]:
    """Fetch US Treasury par yields from FRED.

    Returns a dict mapping maturity label to yield percent.
    Maturities with no data (e.g. holidays) are omitted.
    """
    fred = Fred(api_key=api_key)
    target = as_of or date.today()
    start = target - timedelta(days=7)  # look back for weekends/holidays

    yields = {}
    for series_id, label in TREASURY_SERIES.items():
        data = fred.get_series(series_id, observation_start=start, observation_end=target)
        data = data.dropna()
        if len(data) > 0:
            yields[label] = round(float(data.iloc[-1]), 2)
    return yields

The 7-day lookback handles a real data-quality issue: Treasury yields aren't published on weekends or US holidays. Without the buffer, a Monday morning query for "today's yields" would return nothing.

Classifying the Curve Shape

Given a yield dict, we can mechanically classify the curve:

def classify_curve_shape(yields: dict[str, float]) -> str:
    ordered = [(label, yields[label]) for label in MATURITY_ORDER if label in yields]
    if len(ordered) < 3:
        return "flat"

    short_rate = ordered[0][1]
    long_rate = ordered[-1][1]
    mid_rate = ordered[len(ordered) // 2][1]
    spread = long_rate - short_rate

    # Humped: mid-term rate higher than both short and long ends
    if mid_rate > short_rate + 0.2 and mid_rate > long_rate + 0.2:
        return "humped"

    if abs(spread) <= 0.2:
        return "flat"
    elif spread > 0:
        return "normal"
    else:
        return "inverted"

These thresholds (0.2 percentage points) are arbitrary — real analysts look at the 2Y-10Y spread specifically, and use historical distributions to judge what's "steep" vs "flat". But for a quick visual classification, this works.

Testing Without Hitting FRED

The real FRED API is rate-limited and flaky (I hit a 502 Bad Gateway during development). Tests should never depend on external services. Using unittest.mock, we can mock the Fred client entirely:

from unittest.mock import patch, MagicMock
import pandas as pd

@patch("treasury_yields.Fred")
def test_returns_dict_of_maturities_and_rates(mock_fred_class):
    mock_fred = MagicMock()
    mock_fred_class.return_value = mock_fred

    def mock_get_series(series_id, observation_start, observation_end):
        rates = {"DGS1MO": 5.25, "DGS10": 4.30, "DGS30": 4.45, ...}
        return pd.Series([rates[series_id]])

    mock_fred.get_series.side_effect = mock_get_series

    result = fetch_par_yields(api_key="fake-key")
    assert result["10Y"] == 4.30

The @patch decorator replaces Fred inside the module under test with a mock that returns canned data. The test runs in milliseconds, no network required.
