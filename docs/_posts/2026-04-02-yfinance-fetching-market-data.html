What yfinance Does

yfinance is a Python library that pulls historical and real-time market data from Yahoo Finance. It's free, requires no API key, and returns pandas DataFrames. It's not suitable for production trading systems (rate limits, no SLA), but it's perfect for learning, prototyping, and backtesting.

In this exercise we build a three-stage pipeline: fetch closing prices, clean the data, and compute daily returns.

Fetching Closing Prices

import yfinance as yf

def fetch_closing_prices(tickers: list[str], period: str = "1y") -> pd.DataFrame:
    data = yf.download(tickers, period=period, auto_adjust=True, progress=False)

    if data.empty:
        raise ValueError(f"No data returned for tickers: {tickers}")

    # yfinance returns MultiIndex for multiple tickers
    if isinstance(data.columns, pd.MultiIndex):
        data = data["Close"]
    elif "Close" in data.columns:
        data = data[["Close"]]
        data.columns = tickers

    return data

Three things to notice:

  • auto_adjust=True — returns adjusted closing prices that account for stock splits and dividends. For returns calculations, you always want adjusted prices. Without this, a 2:1 stock split looks like a 50% crash.
  • MultiIndex handling — when downloading multiple tickers, yfinance returns columns like ("Close", "AAPL"), ("Open", "AAPL"), etc. We extract just the "Close" level to get a clean DataFrame with ticker names as columns.
  • Fail on empty — if the ticker doesn't exist or the API fails silently, we raise immediately rather than passing empty data downstream.

Data Quality Issues in Real Market Data

Real market data is messy. Download a year of prices for any ticker and you'll likely find:

  • NaN values — the stock was halted, the exchange had a holiday, or the data provider has a gap
  • Leading NaN — the ticker IPO'd partway through your date range, so early dates have no data
  • Splits — a $200 stock becomes $100 overnight (handled by auto_adjust)

Our validate_data function handles these:

def validate_data(prices: pd.DataFrame) -> pd.DataFrame:
    # Drop rows where ALL values are NaN
    prices = prices.dropna(how="all")

    # Drop leading rows with any NaN (ticker didn't exist yet)
    first_valid = prices.apply(lambda col: col.first_valid_index()).max()
    if first_valid is not None:
        prices = prices.loc[first_valid:]

    # Forward-fill remaining gaps
    prices = prices.ffill()

    return prices

Why Forward-Fill, Not Interpolation?

If a stock was $100 on Monday and $110 on Wednesday with no data for Tuesday, there are two approaches:

  • Interpolation would say $105 on Tuesday — drawing a line between known points
  • Forward-fill says $100 on Tuesday — using the last known price

Forward-fill is the correct choice for financial data. Maybe Tuesday was a trading halt because of a merger announcement. There WAS no price on Tuesday. The last known price ($100) is the best available information until the market opens again. Interpolation would fabricate a price that never existed.

Computing Returns

import numpy as np

def compute_returns(prices: pd.DataFrame) -> np.ndarray:
    values = prices.values
    returns = np.diff(values, axis=0) / values[:-1]
    return np.nan_to_num(returns, nan=0.0)

This converts a price DataFrame into a numpy array of daily percentage returns. Five prices become four returns. The formula is (price_today - price_yesterday) / price_yesterday, which np.diff / values[:-1] computes in one vectorized operation.

np.nan_to_num is a safety net — if any NaN slips through (a zero price, for example), it becomes 0.0 rather than poisoning downstream calculations.

Pandas vs Numpy — When to Use Which

StageToolWhy
Fetch & cleanpandasDate indices, column names, ffill(), dropna() — data wrangling needs labels
Compute returnsnumpyPure numerical computation, no labels needed, faster

prices.values is the bridge — it converts a pandas DataFrame to a numpy array, stripping away the index and column names. From there, np.diff and array division are pure vectorized math. This is the standard pattern in quantitative finance: pandas for data management, numpy for computation.

Market Data Gotchas

Two biases worth knowing about, even though they don't appear in this exercise:

Survivorship bias — if you download "S&P 500 companies" today and backtest over 10 years, you're only looking at companies that survived. The ones that went bankrupt aren't in your dataset. Your backtest will look better than reality.

Look-ahead bias — using future information to make past decisions. Cleaning data by removing bad tickers after downloading everything is fine for learning. In a real trading system, you'd need to handle bad data in real-time without knowing the future.
