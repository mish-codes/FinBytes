A Complete Walkthrough of Portfolio Risk Metrics

This post is a self-contained reference for the core risk metrics used in portfolio management and credit risk. Every concept comes with a worked numerical example so you can follow the arithmetic by hand. We cover daily returns, volatility, Value at Risk, Expected Shortfall, the Sharpe ratio, portfolio diversification, Cholesky decomposition, and Monte Carlo credit VaR.

1 — Daily Returns

Definition: A daily return measures how much an asset's price changed in one day, expressed as a fraction (or percentage) of the previous day's price.

Simple return:

rt = (Pt − Pt−1) / Pt−1

Log return:

rt = ln(Pt / Pt−1)

Worked example

A stock trades at $100, then $103, then $99.

DayPriceSimple returnLog return
0$100——
1 $103 (103 − 100) / 100 = +3.00% ln(103 / 100) = ln(1.03) = +2.96%
2 $99 (99 − 103) / 103 = −3.88% ln(99 / 103) = ln(0.9612) = −3.96%

Why log returns? Two practical reasons:

  1. Additive over time. The two-day log return equals the sum of daily log returns: 2.96% + (−3.96%) = −1.00%, which matches ln(99/100) = −1.005%. Simple returns don't add this way.
  2. Symmetric around zero. A +5% log move up and a −5% log move down are the same magnitude. Simple returns are not: +5% then −5% leaves you at $99.75, not $100.

2 — Volatility

Definition: Volatility is the standard deviation of returns. It measures how much an asset's returns typically deviate from their average.

σdaily = stdev(r1, r2, …, rn)

To annualize (assuming 252 trading days per year):

σannual = σdaily × √252

Worked example

Five daily returns: +1.2%, −0.8%, +0.5%, −1.5%, +0.9%

Step 1 — Mean:

μ = (1.2 + (−0.8) + 0.5 + (−1.5) + 0.9) / 5 = 0.3 / 5 = 0.06%

Step 2 — Deviations from the mean:

Returnr − μ(r − μ)²
+1.20%+1.14%0.012996
−0.80%−0.86%0.007396
+0.50%+0.44%0.001936
−1.50%−1.56%0.024336
+0.90%+0.84%0.007056

Step 3 — Variance (using sample variance, dividing by n−1 = 4):

sum of squared deviations = 0.012996 + 0.007396 + 0.001936 + 0.024336 + 0.007056 = 0.05372

variance = 0.05372 / 4 = 0.01343

Step 4 — Standard deviation:

σdaily = √0.01343 = 1.05% per day

Step 5 — Annualize:

σannual = 1.05% × √252 = 1.05% × 15.87 = 16.7% per year

Interpretation: "This stock typically moves ±16.7% per year." Higher volatility means more risk (and more opportunity for traders). Lower volatility means more predictable returns.

3 — Value at Risk (VaR)

Definition: VaR answers the question: "What is the maximum loss I can expect at a given confidence level over a given time horizon?" A 95% 1-day VaR of $25K means: "On 95% of days, my loss will be less than $25K."

Method A — Historical simulation

Take observed daily returns, sort them from worst to best, and read off the percentile.

Worked example: 20 daily returns for a $1M portfolio (in percent):

Unsorted:  +0.8  -0.3  +1.2  -1.8  +0.4  -0.6  +0.2  -2.5  +1.0  -0.1

For 95% VaR with 20 observations, the 5th percentile is the 1st worst observation (20 × 0.05 = 1).

VaR (95%, 1-day) = −2.50%

In dollar terms: $1,000,000 × 2.50% = $25,000

Meaning: on 95% of days, this portfolio loses less than $25,000.

Method B — Parametric VaR (normal assumption)

If returns are normally distributed with mean μ and standard deviation σ:

VaR = −μ + z × σ

where z is the standard normal quantile (z = 1.645 for 95%, z = 2.326 for 99%).

Worked example: Portfolio value = $1,000,000, daily vol = 1.5%, assume μ ≈ 0:

VaR (95%) = $1,000,000 × 1.645 × 0.015 = $24,675

VaR (99%) = $1,000,000 × 2.326 × 0.015 = $34,890

Key limitation: VaR tells you the threshold but nothing about what happens beyond it. If you breach VaR, your loss could be $25K or $250K — VaR does not distinguish between the two. This is why we need CVaR.

4 — CVaR (Conditional VaR / Expected Shortfall)

Definition: CVaR (also called Expected Shortfall, ES) answers: "If I do have a loss worse than VaR, what is the average loss?" It captures the severity of tail events, not just the threshold.

Worked example

Using the same 20 sorted returns from Section 3:

At 95% confidence with 20 observations, the VaR threshold is the 1st worst return (−2.50%). CVaR is the average of all returns at or below that threshold.

With only one observation in the tail (the 1st), CVaR equals that single value:

CVaR (95%) = −2.50% = $25,000

For a richer example, consider 90% VaR (10th percentile = 2nd worst observation):

VaR (90%) = −1.80% (the 2nd worst return)

CVaR (90%) = average of the 2 worst returns = (−2.50% + −1.80%) / 2 = −2.15%

In dollars: $1,000,000 × 2.15% = $21,500

Compare: VaR (90%) said $18,000; CVaR (90%) says the average tail loss is $21,500 — worse, as expected.

Why regulators prefer CVaR:

  • Basel III (specifically the Fundamental Review of the Trading Book, FRTB) replaced VaR with Expected Shortfall as the primary market risk measure.
  • CVaR is a coherent risk measure (it satisfies sub-additivity: diversification always helps). VaR is not coherent.
  • CVaR is always ≥ VaR by definition — it averages the tail, which is always at least as bad as the threshold.

5 — Sharpe Ratio

Definition: The Sharpe ratio measures risk-adjusted return — how much excess return you earn per unit of risk.

Sharpe = (Rportfolio − Rrisk-free) / σportfolio

Worked example

A portfolio returned 12% over the past year. The risk-free rate (e.g., 1-year Treasury) is 4%. The portfolio's annualized volatility is 20%.

Sharpe = (12% − 4%) / 20% = 8% / 20% = 0.40

Interpretation: For every 1% of risk (volatility) taken, the portfolio earned 0.40% of excess return above the risk-free rate.

Rules of thumb:

Sharpe ratioAssessment
< 0.5Poor — not enough return for the risk taken
0.5 – 1.0Decent — acceptable risk-adjusted performance
1.0 – 2.0Good — strong risk-adjusted returns
> 2.0Exceptional — rare outside short time windows

Limitations:

  • Assumes returns are normally distributed. Fat tails make the Sharpe misleading.
  • Penalizes upside volatility the same as downside. A fund that occasionally has large gains gets a lower Sharpe even though those gains are desirable. (The Sortino ratio fixes this by using only downside deviation.)
  • Sensitive to the time period chosen. Annualizing monthly Sharpe by multiplying by √12 assumes i.i.d. returns.

6 — Portfolio Weights and Diversification

Weight = the fraction of total portfolio value in each asset. Weights must sum to 1.0 (100%).

Portfolio return:

Rportfolio = w1 × R1 + w2 × R2 + …

Portfolio variance (the key insight — correlation matters):

σp² = w1²σ1² + w2²σ2² + 2 w1 w2 ρ σ1 σ2

Worked example

A $100,000 portfolio: 60% stocks (w1 = 0.60) and 40% bonds (w2 = 0.40).

  • σstocks = 20% (annual)
  • σbonds = 5% (annual)
  • ρstocks,bonds = 0.2 (low positive correlation)

Step 1 — Compute each term:

TermFormulaValue
Stocks variancew1² × σ1² = 0.60² × 0.20²0.36 × 0.04 = 0.0144
Bonds variancew2² × σ2² = 0.40² × 0.05²0.16 × 0.0025 = 0.0004
Cross term2 × w1 × w2 × ρ × σ1 × σ22 × 0.60 × 0.40 × 0.2 × 0.20 × 0.05 = 0.00192

Step 2 — Portfolio variance and volatility:

σp² = 0.0144 + 0.0004 + 0.00192 = 0.01672

σp = √0.01672 = 12.93%

Step 3 — Compare to the naive weighted average:

Naive: 0.60 × 20% + 0.40 × 5% = 12% + 2% = 14%

Actual portfolio vol: 12.93%

Diversification benefit: 14% − 12.93% = 1.07% saved

Because stocks and bonds are not perfectly correlated (ρ < 1), the portfolio is less risky than the weighted average of individual risks. This is the mathematical basis of diversification. If ρ = 1.0, the benefit disappears and σp = 14% exactly.

7 — Cholesky Decomposition for Correlated Simulation

The problem: Monte Carlo simulation needs random draws for each risk factor, but risk factors are correlated (e.g., credit spreads of two bonds in the same sector). Drawing independent random numbers ignores this correlation.

The solution: Cholesky factorization. Given a correlation (or covariance) matrix Σ, find a lower-triangular matrix L such that Σ = L LT. Then multiply independent standard normal draws by L to produce correlated draws.

Worked example (2×2)

Correlation matrix:

Σ = | 1.0   0.6 |
      | 0.6   1.0 |

Step 1 — Find L (lower-triangular):

L11 = √Σ11 = √1.0 = 1.0

L21 = Σ21 / L11 = 0.6 / 1.0 = 0.6

L22 = √(Σ22 − L21²) = √(1.0 − 0.36) = √0.64 = 0.8

L = | 1.0   0.0 |
    | 0.6   0.8 |

Step 2 — Verify: L × LT should equal Σ.

L × L^T = | 1.0×1.0 + 0.0×0.0   1.0×0.6 + 0.0×0.8 |   =   | 1.0   0.6 |
            | 0.6×1.0 + 0.8×0.0   0.6×0.6 + 0.8×0.8 |       | 0.6   1.0 |   ✓

Step 3 — Generate correlated draws. Start with independent standard normal draws Z = [0.5, −0.3].

Correlated draws X = L × Z (matrix-vector multiply):

X1 = 1.0 × 0.5 + 0.0 × (−0.3) = 0.50

X2 = 0.6 × 0.5 + 0.8 × (−0.3) = 0.30 − 0.24 = 0.06

The output [0.50, 0.06] is a pair of correlated draws. X1 is strongly positive, so X2 is pulled slightly positive too (because ρ = 0.6). If the correlation were zero, X2 would have stayed at −0.3.

8 — Monte Carlo Credit VaR

The full process:

  1. Define portfolio positions with notionals, CS01 sensitivities, and spread volatilities.
  2. Build a correlation matrix for the credit spreads.
  3. For each simulation path: draw correlated standard normals → scale by spread vol to get spread changes → compute P&L via CS01.
  4. Collect all simulated P&Ls and take the percentile.

Worked example — 2-position portfolio

PositionNotionalCS01 (%/bp)Spread vol (bps/day)
Bond A$500,0000.04550
Bond B$300,0000.08080

Correlation between A and B spreads: ρ = 0.6

Cholesky factor (same as Section 7): L = [[1.0, 0.0], [0.6, 0.8]]

P&L formula for each position:

P&Li = −Notionali × CS01i × Δspreadi

where Δspreadi = voli × Xi (correlated shock, in bps)

The negative sign: a spread widening (positive Δspread) causes a loss.

5 simulated paths (using a fixed seed for reproducibility):

import numpy as np

np.random.seed(42)
L = np.array([[1.0, 0.0], [0.6, 0.8]])
notional = np.array([500_000, 300_000])
cs01     = np.array([0.045, 0.080])       # percent per bp, i.e. 0.045% = 0.00045
spread_vol = np.array([50, 80])           # bps per day

n_paths = 5
Z = np.random.standard_normal((n_paths, 2))  # independent draws
X = Z @ L.T                                   # correlated draws

for i in range(n_paths):
    ds = spread_vol * X[i]                     # spread change in bps
    pnl = -notional * (cs01 / 100) * ds        # P&L in dollars
    total = pnl.sum()
    print(f"Path {i+1}: Z={Z[i].round(2)}, X={X[i].round(2)}, "
          f"dS={ds.round(1)} bps, P&L_A=${pnl[0]:+,.0f}, "
          f"P&L_B=${pnl[1]:+,.0f}, Total=${total:+,.0f}")

With only 5 paths, the 95th-percentile VaR is simply the worst: VaR (95%) = $31,113. In practice you would run 10,000+ paths for a stable estimate.

Why correlation matters: In Path 4, Z1 was strongly positive (+1.52), and because ρ = 0.6, the correlated draw for B was also pushed positive (+0.73), causing both positions to lose simultaneously. If the correlation were zero, the same independent draws would have given ΔSB = 80 × (−0.23) = −18.4 bps (a gain), partially offsetting the loss on A. High correlation means positions lose together in the tail — diversification benefit shrinks.

Scaling to production

def monte_carlo_credit_var(notional, cs01, spread_vol, corr_matrix,
                           n_paths=10_000, confidence=0.95, seed=None):
    """
    Monte Carlo VaR for a credit portfolio.

    Parameters
    ----------
    notional   : array of notional amounts ($)
    cs01       : array of CS01 values (% per bp, e.g. 0.045)
    spread_vol : array of daily spread volatilities (bps)
    corr_matrix: correlation matrix (n x n)
    n_paths    : number of simulation paths
    confidence : VaR confidence level (e.g. 0.95)
    seed       : random seed for reproducibility

    Returns
    -------
    var  : VaR at the given confidence level ($, positive number)
    cvar : CVaR (Expected Shortfall) at the same level ($)
    pnls : array of all simulated total P&Ls
    """
    rng = np.random.default_rng(seed)
    n = len(notional)
    L = np.linalg.cholesky(corr_matrix)

    Z = rng.standard_normal((n_paths, n))
    X = Z @ L.T                                   # correlated shocks
    dS = spread_vol * X                            # spread changes (bps)
    pnl = -(notional * cs01 / 100) * dS            # P&L per position
    total_pnl = pnl.sum(axis=1)                    # total P&L per path

    var_threshold = np.percentile(total_pnl, (1 - confidence) * 100)
    cvar = total_pnl[total_pnl <= var_threshold].mean()

    return -var_threshold, -cvar, total_pnl

9 — How It All Connects

The nine concepts above form two interlocking pipelines:

Pipeline 1 — Equity / general portfolio risk:

Prices  →  Returns  →  Volatility  →  VaR / CVaR  →  Sharpe ratio
                              ↑
                    Portfolio weights + correlation
                    (diversification reduces vol)

Pipeline 2 — Credit risk (Monte Carlo):

CS01 sensitivities  →  Correlation matrix  →  Cholesky L
                                                       ↓
Independent draws Z  →  Correlated shocks X = LZ  →  Spread changes ΔS
                                                               ↓
                                              P&L = -Notional × CS01 × ΔS
                                                               ↓
                                                     Sort → VaR / CVaR

The common thread:

  • Returns (Section 1) feed into volatility (Section 2), which feeds into VaR/CVaR (Sections 3-4) and Sharpe (Section 5).
  • Portfolio weights (Section 6) and correlation determine whether diversification reduces risk.
  • Cholesky decomposition (Section 7) is the engine that introduces correlation into Monte Carlo simulation (Section 8).
  • The end product is always the same: a dollar-denominated risk number (VaR or CVaR) that tells the portfolio manager, the risk committee, or the regulator how much capital is at stake.

Quick Python summary — all metrics in one block

import numpy as np

# --- Sample data ---
returns = np.array([0.012, -0.008, 0.005, -0.015, 0.009,
                    0.003, -0.004, 0.011, -0.007, 0.002])

# Volatility
vol_daily  = returns.std(ddof=1)
vol_annual = vol_daily * np.sqrt(252)

# VaR (historical, 90% with 10 observations)
var_90 = -np.percentile(returns, 10)              # positive number = loss

# CVaR
threshold = np.percentile(returns, 10)
cvar_90 = -returns[returns <= threshold].mean()

# Sharpe (annualized, assuming daily returns)
mean_annual = returns.mean() * 252
rf = 0.04                                         # 4% annual risk-free
sharpe = (mean_annual - rf) / vol_annual

print(f"Daily vol:    {vol_daily:.4%}")
print(f"Annual vol:   {vol_annual:.2%}")
print(f"VaR (90%):    {var_90:.4%}")
print(f"CVaR (90%):   {cvar_90:.4%}")
print(f"Sharpe ratio: {sharpe:.2f}")
