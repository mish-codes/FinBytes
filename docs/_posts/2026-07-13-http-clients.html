Same API, Eight Clients

The task: GET a JSON endpoint, POST a JSON body, handle errors, and set a timeout. These are the four operations every HTTP client must do well. Here is each one in eight libraries across six languages.

1. Python requests (sync)

import requests

# GET with JSON parsing
resp = requests.get("https://api.example.com/quotes/AAPL", timeout=10)
resp.raise_for_status()
data = resp.json()  # {"ticker": "AAPL", "price": 189.50}

# POST with JSON body
payload = {"ticker": "AAPL", "quantity": 100, "side": "buy"}
resp = requests.post("https://api.example.com/orders", json=payload, timeout=10)
resp.raise_for_status()
order = resp.json()

# Error handling
try:
    resp = requests.get("https://api.example.com/bad", timeout=10)
    resp.raise_for_status()
except requests.exceptions.Timeout:
    print("Request timed out")
except requests.exceptions.HTTPError as e:
    print(f"HTTP {e.response.status_code}: {e.response.text}")

The simplest HTTP client in any language. Synchronous, blocking, no async support. A new TCP connection per request unless you use a Session object for connection pooling.

2. Python httpx (sync + async)

import httpx

# Sync — drop-in replacement for requests
resp = httpx.get("https://api.example.com/quotes/AAPL", timeout=10.0)
resp.raise_for_status()
data = resp.json()

# Async
async with httpx.AsyncClient(timeout=10.0) as client:
    resp = await client.get("https://api.example.com/quotes/AAPL")
    resp.raise_for_status()
    data = resp.json()

    resp = await client.post(
        "https://api.example.com/orders",
        json={"ticker": "AAPL", "quantity": 100, "side": "buy"},
    )
    order = resp.json()

httpx supports both sync and async with the same API. It also handles HTTP/2, follows redirects by default, and has built-in connection pooling via the client context manager. The modern replacement for requests.

3. Python aiohttp (async only)

import aiohttp

async with aiohttp.ClientSession(
    timeout=aiohttp.ClientTimeout(total=10)
) as session:
    # GET
    async with session.get("https://api.example.com/quotes/AAPL") as resp:
        resp.raise_for_status()
        data = await resp.json()

    # POST
    async with session.post(
        "https://api.example.com/orders",
        json={"ticker": "AAPL", "quantity": 100, "side": "buy"},
    ) as resp:
        order = await resp.json()

Session-based, async-only. The session manages a connection pool. Designed for high-concurrency scenarios where you make thousands of requests. More verbose than httpx but slightly lower overhead in benchmarks.

4. Go net/http (stdlib)

package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "io"
    "net/http"
    "time"
)

func main() {
    client := &http.Client{Timeout: 10 * time.Second}

    // GET
    resp, err := client.Get("https://api.example.com/quotes/AAPL")
    if err != nil { panic(err) }
    defer resp.Body.Close()

    var quote struct {
        Ticker string  `json:"ticker"`
        Price  float64 `json:"price"`
    }
    json.NewDecoder(resp.Body).Decode(&quote)
    fmt.Printf("%s: %.2f\n", quote.Ticker, quote.Price)

    // POST
    body, _ := json.Marshal(map[string]any{
        "ticker": "AAPL", "quantity": 100, "side": "buy",
    })
    resp, err = client.Post(
        "https://api.example.com/orders",
        "application/json",
        bytes.NewReader(body),
    )
    if err != nil { panic(err) }
    defer resp.Body.Close()

    if resp.StatusCode >= 400 {
        b, _ := io.ReadAll(resp.Body)
        fmt.Printf("Error %d: %s\n", resp.StatusCode, b)
    }
}

Verbose but zero external dependencies. Go's stdlib HTTP client has connection pooling, keep-alive, and redirect handling built in. You must close the response body and check status codes manually—there is no raise_for_status().

5. Rust reqwest (async)

use reqwest::Client;
use serde::{Deserialize, Serialize};
use std::time::Duration;

#[derive(Deserialize)]
struct Quote { ticker: String, price: f64 }

#[derive(Serialize)]
struct Order { ticker: String, quantity: i64, side: String }

#[tokio::main]
async fn main() -> Result<(), reqwest::Error> {
    let client = Client::builder()
        .timeout(Duration::from_secs(10))
        .build()?;

    // GET
    let quote: Quote = client
        .get("https://api.example.com/quotes/AAPL")
        .send().await?
        .error_for_status()?
        .json().await?;
    println!("{}: {:.2}", quote.ticker, quote.price);

    // POST
    let order = Order {
        ticker: "AAPL".into(), quantity: 100, side: "buy".into(),
    };
    let resp = client
        .post("https://api.example.com/orders")
        .json(&order)
        .send().await?
        .error_for_status()?;
    println!("Status: {}", resp.status());

    Ok(())
}

Builder pattern, async by default, typed deserialization via serde. error_for_status() converts 4xx/5xx into errors. reqwest is effectively the only HTTP client in the Rust ecosystem—it covers sync and async, TLS, proxies, and connection pooling.

6. TypeScript fetch (built-in)

// GET
const resp = await fetch("https://api.example.com/quotes/AAPL", {
    signal: AbortSignal.timeout(10_000),
});
if (!resp.ok) throw new Error(`HTTP ${resp.status}: ${await resp.text()}`);
const data = await resp.json() as { ticker: string; price: number };

// POST
const orderResp = await fetch("https://api.example.com/orders", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ ticker: "AAPL", quantity: 100, side: "buy" }),
    signal: AbortSignal.timeout(10_000),
});
if (!orderResp.ok) throw new Error(`HTTP ${orderResp.status}`);
const order = await orderResp.json();

Built into browsers and Node 18+. No external dependency. The timeout uses AbortSignal.timeout() (clean, modern API). Note: fetch does NOT throw on 4xx/5xx status codes—you must check resp.ok yourself. This trips up almost everyone the first time.

7. C# HttpClient (async)

using System.Net.Http;
using System.Net.Http.Json;

// Create ONCE and reuse (do NOT dispose per request — socket exhaustion)
var client = new HttpClient { Timeout = TimeSpan.FromSeconds(10) };

// GET
var quote = await client.GetFromJsonAsync<Quote>(
    "https://api.example.com/quotes/AAPL"
);
Console.WriteLine($"{quote.Ticker}: {quote.Price:F2}");

// POST
var order = new Order("AAPL", 100, "buy");
var resp = await client.PostAsJsonAsync("https://api.example.com/orders", order);
resp.EnsureSuccessStatusCode();
var result = await resp.Content.ReadFromJsonAsync<OrderResult>();

// Records for deserialization
record Quote(string Ticker, double Price);
record Order(string Ticker, int Quantity, string Side);
record OrderResult(string OrderId, string Status);

HttpClient is async by design. The critical gotcha: do not create a new HttpClient per request. Each instance holds a socket pool. Creating and disposing them rapidly exhausts ephemeral ports. Use a single instance or IHttpClientFactory in ASP.NET Core.

8. C++ cpp-httplib (header-only)

#include "httplib.h"
#include "nlohmann/json.hpp"

using json = nlohmann::json;

int main() {
    httplib::Client cli("https://api.example.com");
    cli.set_read_timeout(10);
    cli.set_connection_timeout(5);

    // GET
    auto res = cli.Get("/quotes/AAPL");
    if (res && res->status == 200) {
        auto data = json::parse(res->body);
        printf("%s: %.2f\n",
            data["ticker"].get<std::string>().c_str(),
            data["price"].get<double>());
    }

    // POST
{% raw %}    json order = {{"ticker", "AAPL"}, {"quantity", 100}, {"side", "buy"}};{% endraw %}
    auto post_res = cli.Post("/orders", order.dump(), "application/json");
    if (!post_res || post_res->status >= 400) {
        printf("Error: %d\n", post_res ? post_res->status : -1);
    }

    return 0;
}

cpp-httplib is a single-header library. Combined with nlohmann/json, you get a usable HTTP client in about 15 lines. The API is synchronous. For async, you would need Boost.Beast or libcurl with a custom event loop—significantly more complex.

When to Use What

Python

Scripts and CLIs: requests. It is the simplest HTTP client in any language. No async needed, no context managers required for basic use.

Async APIs and new projects: httpx. Same API as requests but supports async, HTTP/2, and has better timeout control. The migration path from requests is trivial.

High-concurrency (thousands of requests): aiohttp. Slightly faster than httpx in benchmarks for pure async workloads. The session-based API is more verbose but gives fine-grained control.

Go

The stdlib net/http is the only option you need. It is well-designed, performant, and every Go developer knows it. Third-party wrappers like resty add convenience but no fundamental capability.

Rust

reqwest is the only real choice. The ecosystem has converged on it. The builder pattern is clean, serde integration is seamless, and it handles TLS, proxies, and cookies out of the box.

TypeScript

Use fetch. It is built into Node 18+ and every browser. No dependency to install, no bundler configuration. Just remember: fetch does not throw on HTTP errors—check resp.ok.

C#

HttpClient is the standard. Use IHttpClientFactory in ASP.NET Core for proper lifetime management. Never create and dispose HttpClient in a loop.

C++

For simple use, cpp-httplib (header-only, small footprint). For production systems, libcurl (battle-tested, every feature imaginable, terrible API) or Boost.Beast (async, modern, steep learning curve).
