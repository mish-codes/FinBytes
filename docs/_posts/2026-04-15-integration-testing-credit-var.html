From Single-Name Risk to Portfolio Risk

Everything up to this point has been single-name or single-curve work: what's the default probability for this issuer, what's the spread on this bond, what's the OAS on this callable. Exercise 20 shifts to portfolio risk — how much can I lose across a book of credit positions when spreads move together.

The answer requires three ingredients: a model for how spreads move (correlated shocks), a measure of how much each position's price changes when its spread moves (spread duration), and a simulation framework that puts them together (Monte Carlo VaR).

Cholesky Decomposition: Making Shocks Correlated

Monte Carlo starts with independent random draws. But credit spreads don't move independently — when the market sells off, BBB industrials and BBB utilities both widen, just not by the same amount. The correlation structure matters because it determines how much diversification you actually get.

The standard technique is Cholesky decomposition. Given a correlation matrix Σ, you find the lower-triangular matrix L such that LLT = Σ. Then:

independent draws:   Z ~ N(0, I)
correlated draws:    X = Z @ L^T   → X ~ N(0, Σ)
def generate_correlated_spread_shocks(correlation_matrix, num_simulations, seed=None):
    rng = np.random.default_rng(seed)
    n = correlation_matrix.shape[0]
    L = cholesky(correlation_matrix, lower=True)
    Z = rng.standard_normal((num_simulations, n))
    return Z @ L.T

The test verifies this works: generate 10,000 draws with target correlation 0.8, measure the realised correlation, and check it's within 0.05 of the target. With 10K samples the sampling error is small enough to make the test reliable.

Why correlation drives portfolio VaR

Two positions with correlation 0.2 partially offset each other — when one widens, the other might not. Two positions with correlation 0.9 tend to widen together, so the tail losses compound. Same positions, same individual risks, but the portfolio VaR can differ by 30-50% depending on the correlation assumption. This is why getting the correlation matrix right matters more than getting any individual spread volatility right.

Spread Duration: CS01

Spread duration (also called CS01 or credit spread sensitivity) measures how much a bond's price changes per 1 bp spread move. It's the credit analogue of modified duration for interest rates:

ΔP ≈ −SD × Δspread × notional

We compute it via central finite difference: bump the spread up 1 bp, bump it down 1 bp, take the price difference:

def spread_duration(price, coupon, maturity_years, spread_bps, face=100.0, bump_bps=1.0):
    price_up = _price_at_spread(spread_bps + bump_bps)
    price_down = _price_at_spread(spread_bps - bump_bps)
    return abs((price_up - price_down) / (2 * bump_bps))

Longer-maturity bonds have higher spread duration (more cash flows further out, more sensitive to discounting changes). A 10-year bond with CS01 of 0.08 loses $0.08 per $100 face per 1 bp widening. On a $50M position, a 100 bp widening costs $4M. That's why portfolio risk managers care about this number.

Monte Carlo VaR: Putting It Together

The full simulation:

  1. Generate N paths of correlated standard-normal shocks (Cholesky).
  2. Scale each shock by the position's spread volatility to get spread changes in bps.
  3. For each position and each path: P&L = −CS01 × Δspread × quantity × face/100.
  4. Sum across positions to get portfolio P&L per path.
  5. VaR = 5th percentile of the P&L distribution (for 95% confidence).
def portfolio_credit_var(positions, correlation_matrix, spread_volatilities,
                         confidence=0.95, num_simulations=10000, seed=None):
    shocks = generate_correlated_spread_shocks(correlation_matrix, num_simulations, seed)
    spread_changes = shocks * spread_volatilities

    durations = [spread_duration(p["price"], p["coupon"], ...) for p in positions]

    pnl = np.zeros(num_simulations)
    for i, pos in enumerate(positions):
        pnl += -durations[i] * spread_changes[:, i] * pos["quantity"] * pos["face"] / 100

    return float(np.percentile(pnl, (1 - confidence) * 100))

The test suite verifies two key properties: VaR is negative (it's a loss measure), and higher correlation produces worse (more negative) VaR. Both are economic invariants — if either breaks, the model is wrong.

Limitations and What Production Would Need

This is a teaching implementation. A production credit VaR engine would add:

  • Full revaluation instead of the duration approximation. For large spread moves, the linear approximation breaks down (convexity). Full reval reprices every bond on every path.
  • Historical or GARCH volatilities rather than assumed constants. Spread vol is time-varying and clusters during crises.
  • Stressed correlation. Correlations increase during selloffs (the "correlation breakdown" problem). Using calm-period correlations underestimates tail risk.
  • Default simulation. This model only captures spread risk (mark-to-market losses from spread widening). A full credit VaR model also simulates actual defaults, which cause jump-to-default losses.
  • Convergence testing. 10K simulations is enough for a 95% VaR estimate but noisy for 99% or 99.9%. Production systems run 100K-1M paths and report confidence intervals around the VaR estimate.
