Why Know More Than One Charting Library

If you build finance dashboards in Python, you will reach for Plotly by default — it is interactive, declarative, and works natively in Streamlit. But every library makes different trade-offs. Matplotlib gives pixel-level control for publication figures. Altair compresses complex charts into a few lines of grammar-of-graphics. Bokeh offers server-side interactivity without JavaScript. Knowing when to use each one is a practical skill.

This post compares all four using the same OHLCV dataset, then digs into how each handles outlier values — the most common visual problem in financial data.

Plotly

Plotly is the default choice for Streamlit finance dashboards. It has native Candlestick and OHLC chart types, interactive zoom/hover out of the box, and a high-level Express API for quick plots. The trade-off: figure objects are heavy (JSON-serialised) and customisation beyond the built-in options requires verbose update_layout calls.

import plotly.graph_objects as go

fig = go.Figure(data=[go.Candlestick(
    x=df.index,
    open=df["Open"], high=df["High"],
    low=df["Low"], close=df["Close"],
)])
st.plotly_chart(fig, use_container_width=True)

Matplotlib

The foundation of Python visualisation. Every other library either wraps it (Seaborn) or was built as a reaction to it (Altair, Plotly). Matplotlib gives you axes, artists, and transforms — full control over every pixel. Candlesticks require manual bar + vlines construction since there is no built-in financial chart type (without the mplfinance extension).

import matplotlib.pyplot as plt
import matplotlib.dates as mdates

fig, ax = plt.subplots()
up = df[df["Close"] >= df["Open"]]
ax.bar(mdates.date2num(up.index), (up["Close"] - up["Open"]),
       bottom=up["Open"], color="#2ea043", width=0.6)
ax.vlines(mdates.date2num(up.index), up["Low"], up["High"], color="#2ea043")
st.pyplot(fig)

Altair

Altair implements the grammar of graphics via Vega-Lite. You describe what to plot (encodings, marks, transforms) rather than how to draw it. A candlestick is a layered mark_rule (wicks) + mark_bar (bodies) with a colour encoding on the sign of close minus open. The learning curve is the grammar itself, but once internalised, charts are remarkably concise.

import altair as alt

rule = alt.Chart(df).mark_rule().encode(
    x="Date:T", y="Low:Q", y2="High:Q",
    color=alt.Color("color:N", scale=None),
)
bar = alt.Chart(df).mark_bar(size=6).encode(
    x="Date:T", y="Open:Q", y2="Close:Q",
    color=alt.Color("color:N", scale=None),
)
st.altair_chart(rule + bar, use_container_width=True)

Bokeh

Bokeh renders to HTML Canvas and provides its own toolbar (pan, zoom, reset, save). Unlike Plotly, Bokeh supports server-side callbacks and custom widgets without JavaScript. In Streamlit, st.bokeh_chart only supports Bokeh 2.4.x, so for newer versions we render via components.html using Bokeh’s file_html function. Candlesticks use segment (wicks) + vbar (bodies).

from bokeh.plotting import figure
from bokeh.embed import file_html
from bokeh.resources import CDN
import streamlit.components.v1 as components

p = figure(x_axis_type="datetime", width=800, height=350)
p.segment(df.index, df["High"], df.index, df["Low"], color="black")
p.vbar(up.index, w, up["Open"], up["Close"], fill_color="#2ea043")

components.html(file_html(p, CDN), height=400)

Handling Outliers in Financial Charts

Financial data regularly contains extreme values — a flash crash, a stock split that was not adjusted, a fat-finger trade. A single outlier can compress the y-axis so severely that 99% of the data becomes an unreadable flat line. Every charting library has tools to handle this, but the approaches differ.

The Problem

Imagine a 6-month AAPL price history where one closing price was accidentally entered as 10x the real value. The chart auto-scales to fit the spike, and every other data point is crushed to the bottom. The distribution histogram grows a long tail with one bar far from the rest. This is not a rare edge case — it is what happens when you display raw market data without validation.

Detection: 3-Sigma Flagging

A simple first pass: for each numeric column, compute the z-score. Any value more than 3 standard deviations from the column mean is flagged. This catches extreme values without being sensitive to minor fluctuations.

z_scores = (df - df.mean()).abs() / df.std()
outliers = z_scores > 3.0

Display Approaches by Library

Plotly — Annotations

Plotly’s add_annotation places a text label with an arrow pointing at the outlier. Combined with a separate scatter trace using red x markers, outliers are visually distinct without distorting the chart. For the histogram, adjusting nbinsx or switching to a log y-axis prevents the spike bin from dominating.

fig.add_trace(go.Scatter(
    x=outlier_dates, y=outlier_values,
    mode="markers", marker=dict(color="red", size=10, symbol="x"),
))

Matplotlib — Annotate with Arrows

ax.annotate with arrowprops draws a labelled arrow to the outlier point. For axis management, ax.set_ylim can clip the y-axis to a sane range, pushing the outlier off-screen but keeping the rest readable. Alternatively, ax.set_yscale("log") compresses the dynamic range.

ax.annotate("outlier", xy=(date, value),
            xytext=(0, 15), textcoords="offset points",
            arrowprops=dict(arrowstyle="->", color="red"))

Altair — Conditional Colour Encoding

Altair’s declarative model handles this elegantly: add a boolean is_outlier column and use a separate mark_point layer to colour outlier points differently. No manual loop required. For histograms, the bin transform with maxbins can be tuned, and Altair supports log-scale axes via alt.Scale(type="log").

points = alt.Chart(df[df["is_outlier"]]).mark_point(
    color="red", size=100, shape="cross",
).encode(x="Date:T", y="Close:Q")

Bokeh — Glyph Overlay

Bokeh lets you overlay a separate glyph renderer. Add p.cross or p.circle for outlier points with a distinct colour. For axis control, set p.y_range = Range1d(start, end) to clip, or use p.y_range.bounds = (min, max) to constrain interactive zooming.

p.cross(outlier_dates, outlier_values,
        color="red", size=12, line_width=2)

When to Use What

Plotly — your default for Streamlit dashboards and anything interactive. Native finance chart types, minimal code, good hover tooltips. Use it unless you have a reason not to.

Matplotlib — when you need pixel-perfect control, publication-quality static figures, or when you are working in Jupyter notebooks and want inline plots. Also the only option when mplfinance-style charts are required.

Altair — when you want concise, declarative charts and your data fits the grammar-of-graphics mental model. Excellent for exploratory data analysis where you are layering encodings and transforms.

Bokeh — when you need server-side interactivity, custom widget callbacks, or are building a standalone web app outside Streamlit. Overkill for simple dashboard charts but powerful for bespoke tools.

Honourable mentions: Seaborn wraps Matplotlib with better defaults and statistical plotting (violin, swarm, pair plots) — great for EDA but not distinct enough to warrant its own tab here. mplfinance provides one-liner candlestick charts on top of Matplotlib — useful if you only need financial charts and do not want to build them manually.
