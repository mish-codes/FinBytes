Why decompose a time series

Every financial time series is a mix of signals: a long-term trend, repeating seasonal patterns, and unpredictable residual noise. Decomposition separates these so you can analyze each in isolation. A rising trend with shrinking seasonality tells a different story than a flat trend with growing volatility.

Generate synthetic data

import numpy as np
import pandas as pd

np.random.seed(42)

# 3 years of monthly data
dates = pd.date_range("2023-01-01", periods=36, freq="MS")

trend = np.linspace(100, 160, 36)
seasonal = 8 * np.sin(2 * np.pi * np.arange(36) / 12)
noise = np.random.normal(0, 3, 36)

prices = trend + seasonal + noise

ts = pd.Series(prices, index=dates, name="price")
print(ts.head(12))

We build a series with a known trend (linear rise from 100 to 160), known seasonality (annual sine wave), and Gaussian noise. This lets us verify the decomposition recovers the original components.

Seasonal decomposition

from statsmodels.tsa.seasonal import seasonal_decompose

result = seasonal_decompose(ts, model="additive", period=12)

print("Trend (first 5):")
print(result.trend.dropna().head())
print("\nSeasonal (first 12):")
print(result.seasonal.head(12))
print("\nResidual std:", result.resid.dropna().std().round(2))

model="additive" assumes the series equals trend + seasonal + residual. Use "multiplicative" when the seasonal amplitude grows with the level (common in revenue data). The period=12 tells the algorithm to look for a 12-observation cycle.

Stationarity test (ADF)

from statsmodels.tsa.stattools import adfuller

def adf_test(series: pd.Series, name: str = "series") -> dict:
    """Run Augmented Dickey-Fuller test and return results."""
    stat, p_value, used_lag, n_obs, critical, _ = adfuller(series.dropna())
    result = {
        "series": name,
        "adf_statistic": round(stat, 4),
        "p_value": round(p_value, 4),
        "used_lag": used_lag,
        "stationary": p_value < 0.05,
    }
    print(f"{name}: ADF={stat:.4f}, p={p_value:.4f} -> "
          f"{'Stationary' if p_value < 0.05 else 'Non-stationary'}")
    return result

adf_test(ts, "Original prices")
adf_test(ts.diff().dropna(), "First difference")

The ADF test checks whether a unit root is present. A p-value below 0.05 means we reject the null hypothesis of a unit root, i.e., the series is stationary. Raw prices are almost always non-stationary. First-differencing (returns) typically makes them stationary, which is a prerequisite for most time series models.

ACF and PACF

from statsmodels.tsa.stattools import acf, pacf

returns = ts.diff().dropna()

acf_values = acf(returns, nlags=15, fft=True)
pacf_values = pacf(returns, nlags=15)

fig = make_subplots(rows=1, cols=2, subplot_titles=["ACF", "PACF"])

for i, (vals, name) in enumerate([(acf_values, "ACF"), (pacf_values, "PACF")]):
    fig.add_trace(go.Bar(x=list(range(len(vals))), y=vals,
                         marker_color="steelblue", name=name),
                  row=1, col=i + 1)
    # 95% confidence band
    ci = 1.96 / np.sqrt(len(returns))
    fig.add_hline(y=ci, line_dash="dash", line_color="red", row=1, col=i + 1)
    fig.add_hline(y=-ci, line_dash="dash", line_color="red", row=1, col=i + 1)

fig.update_layout(height=350, width=800, template="plotly_white",
                  title="Autocorrelation Analysis", showlegend=False)
fig.show()

ACF (autocorrelation function) measures the correlation between a series and its lagged values. PACF (partial autocorrelation) removes the effect of intermediate lags. Together they guide model selection:

ACF patternPACF patternSuggested model
Decays slowlyCuts off after lag pAR(p)
Cuts off after lag qDecays slowlyMA(q)
Both decayBoth decayARMA(p, q)

Putting it together

# Full pipeline: decompose, test stationarity, check autocorrelation
def analyze_time_series(series: pd.Series, period: int = 12):
    """Run decomposition, ADF test, and ACF/PACF analysis."""
    decomp = seasonal_decompose(series, model="additive", period=period)

    adf_raw = adf_test(series, "Raw")
    diff = series.diff().dropna()
    adf_diff = adf_test(diff, "Differenced")

    acf_vals = acf(diff, nlags=min(15, len(diff) // 2 - 1), fft=True)
    pacf_vals = pacf(diff, nlags=min(15, len(diff) // 2 - 1))

    return {
        "decomposition": decomp,
        "adf_raw": adf_raw,
        "adf_diff": adf_diff,
        "acf": acf_vals,
        "pacf": pacf_vals,
    }

results = analyze_time_series(ts, period=12)

This pipeline is the starting point for any time series modeling task. Decompose to understand the structure, difference to achieve stationarity, then use ACF/PACF to choose the right model order.

Real-world notes

  • Financial returns often have near-zero autocorrelation in levels but significant autocorrelation in squared returns (volatility clustering). This is why GARCH models exist.
  • For intraday data, seasonality is intra-day (U-shaped volume, higher volatility at open/close). Set period to the number of observations per day.
  • If your series has multiple seasonal patterns (daily + weekly), use STL decomposition (from statsmodels.tsa.seasonal import STL) which handles this more flexibly.
