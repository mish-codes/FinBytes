Why FastAPI for Financial Services?

Financial applications need three things from an API framework: speed (market data waits for no one), validation (bad input means bad trades), and documentation (other teams need to integrate with you). FastAPI delivers all three out of the box.

FastAPI is built on Starlette (async HTTP) and Pydantic (data validation). Type hints aren't optional decoration — they power automatic request validation, response serialization, and OpenAPI documentation. You write Python type annotations and get a validated, documented API for free.

A Health Check — The Simplest Endpoint

from fastapi import FastAPI

app = FastAPI(title="Portfolio API", version="0.1.0")

@app.get("/health")
def health():
    return {"status": "ok"}

That's a complete endpoint. @app.get("/health") registers a GET route. The return dict is automatically serialized to JSON. Hit /docs in your browser and you'll see auto-generated Swagger documentation — no extra configuration needed.

Watchlist CRUD — Status Codes and Error Handling

A watchlist lets users track tickers they're interested in. This gives us a chance to use the full range of HTTP status codes.

from fastapi import HTTPException, status
from pydantic import BaseModel, Field

_watchlist: dict[str, dict] = {}

class TickerIn(BaseModel):
    ticker: str = Field(..., min_length=1, max_length=10)

class TickerOut(BaseModel):
    ticker: str


@app.get("/watchlist", response_model=list[TickerOut])
def list_watchlist():
    return list(_watchlist.values())


@app.post("/watchlist", response_model=TickerOut, status_code=status.HTTP_201_CREATED)
def add_to_watchlist(body: TickerIn):
    ticker = body.ticker.upper()
    if ticker in _watchlist:
        raise HTTPException(status_code=409, detail=f"{ticker} already in watchlist")
    _watchlist[ticker] = {"ticker": ticker}
    return _watchlist[ticker]


@app.delete("/watchlist/{ticker}", status_code=status.HTTP_204_NO_CONTENT)
def remove_from_watchlist(ticker: str):
    ticker = ticker.upper()
    if ticker not in _watchlist:
        raise HTTPException(status_code=404, detail=f"{ticker} not in watchlist")
    del _watchlist[ticker]

Key things to notice:

  • response_model=list[TickerOut] — FastAPI serializes the response through this Pydantic model, stripping any extra fields. This is your API contract.
  • status_code=status.HTTP_201_CREATED — POST returns 201 (created), not 200. DELETE returns 204 (no content). Using the right status codes isn't pedantic — clients depend on them.
  • HTTPException(status_code=409) — 409 Conflict for duplicate tickers, 404 Not Found for missing ones. FastAPI converts these to proper JSON error responses automatically.
  • {ticker} in the path — /watchlist/{ticker} extracts the ticker from the URL as a path parameter. FastAPI passes it to your function as an argument.

Portfolio Analysis — Request Validation with Pydantic

from pydantic import model_validator

class PortfolioIn(BaseModel):
    tickers: list[str] = Field(..., min_length=1)
    weights: list[float] = Field(..., min_length=1)
    period: str = Field(default="1y")

    @model_validator(mode="after")
    def validate_lengths(self):
        if len(self.tickers) != len(self.weights):
            raise ValueError("tickers and weights must have the same length")
        return self

class AnalysisOut(BaseModel):
    tickers: list[str]
    weights: list[float]
    period: str
    message: str


@app.post("/analyze", response_model=AnalysisOut)
def analyze_portfolio(body: PortfolioIn):
    return AnalysisOut(
        tickers=body.tickers,
        weights=body.weights,
        period=body.period,
        message=f"Analysis stub for {len(body.tickers)} tickers over {body.period}",
    )

If a client sends {"tickers": ["AAPL", "MSFT"], "weights": [0.5]}, FastAPI returns a 422 Unprocessable Entity with a detailed error message — and your endpoint function never runs. The Pydantic model_validator catches the mismatch before your code sees the data. This is the same pattern from the Pydantic exercise, now integrated into an API.

The /analyze endpoint returns a stub for now. Real market data analysis comes in later exercises (yfinance for data, Claude API for narrative). The endpoint shape is ready — we just swap the stub for real logic.

Testing with TestClient

from fastapi.testclient import TestClient
from app import app, _watchlist

@pytest.fixture(autouse=True)
def clear_watchlist():
    _watchlist.clear()

@pytest.fixture
def client():
    return TestClient(app)

class TestWatchlist:
    def test_add_ticker(self, client):
        resp = client.post("/watchlist", json={"ticker": "AAPL"})
        assert resp.status_code == 201
        assert resp.json()["ticker"] == "AAPL"

TestClient runs your FastAPI app in-process — no server, no port, no network. It uses ASGI directly, so tests run in milliseconds. Under the hood it's powered by httpx.

The autouse=True fixture is critical: it clears the in-memory watchlist before every test. Without it, tests share state — test_add_ticker adds "AAPL", and later test_get_empty_watchlist fails because the dict isn't empty. State isolation prevents flaky tests.

Status Codes Cheat Sheet

CodeMeaningWhen to Use
200OKSuccessful GET or POST that returns data
201CreatedResource successfully created (POST)
204No ContentSuccessful DELETE (nothing to return)
404Not FoundResource doesn't exist
409ConflictDuplicate resource (e.g., ticker already in watchlist)
