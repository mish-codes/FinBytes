Why FastAPI

FastAPI is a Python web framework that gives you automatic request validation (via Pydantic), auto-generated API docs (at /docs), and async support out of the box. For the QuantLab scanner, it meant we could define a Pydantic model for the request, and FastAPI would reject malformed input with a 422 error before our code even ran.

FastAPI was introduced in Exercise 03 and became the backbone of the Stock Risk Scanner API.

The Server Stack

Understanding the layers matters when debugging:

  • ASGI (Asynchronous Server Gateway Interface) — the Python standard that connects a web server to your application. Think of it as a protocol, not software.
  • uvicorn — a single-process ASGI server. Great for development. Runs one event loop, handles async natively.
  • gunicorn — a multi-process WSGI/ASGI server. For production, you run gunicorn with uvicorn workers: gunicorn -w 4 -k uvicorn.workers.UvicornWorker.
  • FastAPI — your application, which implements the ASGI interface.
# Development (auto-reload on code changes)
uvicorn src.scanner.main:app --reload --port 8000

# Production (4 worker processes)
gunicorn src.scanner.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000

# Docker CMD (from the scanner Dockerfile)
CMD ["uvicorn", "src.scanner.main:app", "--host", "0.0.0.0", "--port", "8000"]

Route Decorators

Routes are defined with decorators on the app (or a router):

app = FastAPI(title="Stock Risk Scanner")

@app.get("/health")
def health():
    return {"status": "ok"}

@app.post("/api/scan", status_code=202)
async def create_scan(request: ScanRequest):
    # ...
    return {"id": scan_id, "status": "pending"}

@app.get("/api/scans/{scan_id}")
async def read_scan(scan_id: int):
    # scan_id is automatically parsed as int from the URL path
    # ...

@app.get("/api/scans")
async def list_scans(limit: int = 10):
    # limit is a query parameter: /api/scans?limit=5
    # ...

Path parameters go in the URL template ({scan_id}). Query parameters are function arguments with defaults. FastAPI infers which is which from the route definition.

Request and Response Models (Pydantic)

The scanner uses Pydantic models to validate input. If the request body does not match the model, FastAPI returns a 422 with detailed error messages automatically.

from pydantic import BaseModel, field_validator, model_validator


class ScanRequest(BaseModel):
    tickers: list[str]
    weights: list[float]
    period: str = "1y"

    @field_validator("tickers")
    @classmethod
    def uppercase_tickers(cls, v: list[str]) -> list[str]:
        return [t.upper() for t in v]

    @model_validator(mode="after")
    def validate_lengths_and_weights(self) -> "ScanRequest":
        if len(self.tickers) != len(self.weights):
            raise ValueError(
                f"tickers length ({len(self.tickers)}) must match "
                f"weights length ({len(self.weights)})"
            )
        if abs(sum(self.weights) - 1.0) > 0.01:
            raise ValueError(
                f"weights must sum to 1.0 (got {sum(self.weights):.4f})"
            )
        return self


class RiskMetrics(BaseModel):
    var_pct: float
    cvar_pct: float
    max_drawdown_pct: float
    volatility_pct: float
    sharpe_ratio: float


class ScanResult(BaseModel):
    tickers: list[str]
    weights: list[float]
    metrics: RiskMetrics
    narrative: str
    generated_at: datetime

This is the actual code from the scanner. The @field_validator normalizes ticker symbols to uppercase. The @model_validator ensures the tickers and weights arrays have the same length and that weights sum to 1.0.

Status Codes We Used

  • 200 — OK. Default for GET routes. Used for health checks and scan retrieval.
  • 201 — Created. Use when a resource is created synchronously (we did not use this; scans are async).
  • 202 — Accepted. The scan request is accepted and processing will happen in the background. The scanner's POST /api/scan returns 202.
  • 204 — No Content. Use for successful DELETE operations where there is nothing to return.
  • 404 — Not Found. Returned when a scan ID does not exist.
  • 409 — Conflict. Use when a resource already exists (e.g., duplicate ticker in a watchlist).
  • 422 — Unprocessable Entity. FastAPI returns this automatically when Pydantic validation fails.

The Async Background Task Pattern

The scanner uses HTTP 202 (Accepted) to immediately return a scan ID, then processes the scan in the background. The client polls for results:

import asyncio
from fastapi import Request

@app.post("/api/scan", status_code=202)
async def create_scan(
    request: ScanRequest,
    session: AsyncSession = Depends(get_session),
    http_request: Request = None,
):
    record = await create_pending_scan(session, request)
    asyncio.ensure_future(
        _run_scan(record.id, request, http_request.app.state.session_factory)
    )
    return {"id": record.id, "status": "pending"}


async def _run_scan(scan_id: int, request: ScanRequest, session_factory):
    async with session_factory() as session:
        try:
            result = scan(request)
            await complete_scan(session, scan_id, result)
        except Exception as e:
            await fail_scan(session, scan_id, str(e))

The client flow: POST to create, receive ID, GET /api/scans/{id} in a loop until status is "complete" or "failed".

Dependency Injection

FastAPI's Depends() is how you share database sessions, auth, and other cross-cutting concerns across routes:

from fastapi import Depends, Request
from sqlalchemy.ext.asyncio import AsyncSession


async def get_session(request: Request) -> AsyncSession:
    factory = request.app.state.session_factory
    async with factory() as session:
        yield session


@app.get("/api/scans/{scan_id}")
async def read_scan(scan_id: int, session: AsyncSession = Depends(get_session)):
    record = await get_scan(session, scan_id)
    if record is None:
        raise HTTPException(status_code=404, detail="Scan not found")
    return _record_to_dict(record)

The yield in get_session makes it a context manager dependency. The session is created before the route runs and disposed after. FastAPI handles the lifecycle automatically.

CORS Middleware

The scanner API is called from a Streamlit dashboard on a different domain. Without CORS headers, the browser blocks the request:

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://mishcodesfinbytes.github.io",
        "https://finbytes.streamlit.app",
        "http://localhost:8501",
    ],
    allow_methods=["*"],
    allow_headers=["*"],
)

Testing with TestClient / AsyncClient

FastAPI integrates with httpx for async testing. The pattern from the scanner test suite:

import pytest
from httpx import AsyncClient, ASGITransport
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker

@pytest.fixture
async def app():
    from src.scanner.db_models import Base
    from src.scanner.main import create_app

    engine = create_async_engine("sqlite+aiosqlite:///:memory:")
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    factory = async_sessionmaker(engine, expire_on_commit=False)

    app = create_app()
    app.state.session_factory = factory
    yield app
    await engine.dispose()

@pytest.fixture
async def client(app):
    transport = ASGITransport(app=app)
    async with AsyncClient(transport=transport, base_url="http://test") as client:
        yield client

class TestCreateScan:
    async def test_returns_pending(self, client):
        resp = await client.post(
            "/api/scan",
            json={"tickers": ["AAPL", "MSFT"], "weights": [0.6, 0.4]},
        )
        assert resp.status_code == 202
        data = resp.json()
        assert data["status"] == "pending"
        assert "id" in data

Key points: the test uses an in-memory SQLite database (no real PostgreSQL needed), and the ASGITransport means no actual HTTP server is started.

Auto-Generated Docs

FastAPI automatically generates interactive API documentation at two endpoints:

  • /docs — Swagger UI. Interactive, lets you try endpoints from the browser.
  • /redoc — ReDoc. Read-only, cleaner layout for documentation.

Both are generated from your Pydantic models and route decorators. No extra configuration needed. This was genuinely useful during development — you can test endpoints without Postman or curl.

Lifespan Events

The scanner uses FastAPI's lifespan context manager to set up and tear down the database engine:

from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):
    engine = create_async_engine(DATABASE_URL)
    app.state.session_factory = async_sessionmaker(engine, expire_on_commit=False)

    # Auto-create tables on startup
    from src.scanner.db_models import Base
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)

    yield  # app runs here

    await engine.dispose()

app = FastAPI(title="Stock Risk Scanner", lifespan=lifespan)

Things That Tripped Me Up

  • 422 vs 400: FastAPI returns 422 for validation errors, not 400. Some frontend libraries expect 400. If your client is not handling 422, you will think the API is broken when it is actually rejecting bad input correctly.
  • Sync vs async routes: FastAPI runs sync functions (def) in a thread pool and async functions (async def) on the event loop. If you define async def but call blocking code (like time.sleep or synchronous DB queries), you block the entire event loop.
  • Depends() in background tasks: Dependency injection runs during the request lifecycle. Background tasks run after the response is sent, so injected sessions may be closed. The scanner works around this by passing the session factory to the background task, not a session.
  • Order of middleware: Middleware is applied in reverse order of how you add it. If CORS middleware is added after authentication middleware, CORS preflight requests fail because auth rejects them first.
  • TestClient vs AsyncClient: FastAPI's TestClient (from Starlette) is synchronous. For async endpoints with async database access, use httpx.AsyncClient with ASGITransport instead.
  • Pydantic V1 vs V2: FastAPI 0.100+ uses Pydantic V2. The validator decorators changed from @validator to @field_validator and from @root_validator to @model_validator. Old tutorials will show the V1 syntax, which still works but raises deprecation warnings.
