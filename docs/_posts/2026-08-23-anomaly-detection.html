What counts as an anomaly

In financial data, anomalies are observations that deviate significantly from the expected pattern: flash crashes, earnings surprises, data errors, or regime changes. Detecting them automatically is critical for risk management, fraud detection, and data quality pipelines.

Generate synthetic price data

import numpy as np
import pandas as pd

np.random.seed(42)

n_days = 300
dates = pd.date_range("2024-01-01", periods=n_days, freq="B")

# Random walk with drift
returns = np.random.normal(0.0003, 0.012, n_days)

# Inject anomalies: 8 extreme moves
anomaly_idx = np.array([30, 75, 120, 155, 190, 210, 250, 280])
returns[anomaly_idx] = np.array([-0.06, 0.07, -0.08, 0.05, -0.05, 0.06, -0.07, 0.05])

prices = 100 * np.cumprod(1 + returns)
df = pd.DataFrame({"date": dates, "price": prices, "return": returns})
print(f"Data shape: {df.shape}")
print(f"Return range: [{returns.min():.4f}, {returns.max():.4f}]")

Method 1: Z-score

def zscore_anomalies(returns: np.ndarray, threshold: float = 3.0) -> np.ndarray:
    """Flag observations where |z-score| exceeds threshold."""
    mu = np.mean(returns)
    sigma = np.std(returns)
    z_scores = (returns - mu) / sigma
    return np.abs(z_scores) > threshold

df["zscore_anomaly"] = zscore_anomalies(df["return"].values, threshold=2.5)
print(f"Z-score anomalies found: {df['zscore_anomaly'].sum()}")

The Z-score measures how many standard deviations a point is from the mean. A threshold of 3.0 catches roughly 0.3% of observations under a normal distribution. Financial returns have fat tails, so a threshold of 2.5 is more practical. The weakness: it assumes a static mean and variance. During high-volatility regimes, many normal observations get flagged.

Method 2: Isolation Forest

from sklearn.ensemble import IsolationForest

def isolation_forest_anomalies(
    df: pd.DataFrame,
    features: list[str],
    contamination: float = 0.03,
) -> np.ndarray:
    """Use Isolation Forest to detect anomalies. Returns boolean array."""
    model = IsolationForest(
        contamination=contamination,
        random_state=42,
        n_estimators=200,
    )
    X = df[features].values
    predictions = model.fit_predict(X)
    return predictions == -1  # -1 = anomaly, 1 = normal

# Use return and a rolling volatility feature
df["rolling_vol"] = df["return"].rolling(10, min_periods=1).std()

df["iforest_anomaly"] = isolation_forest_anomalies(
    df, features=["return", "rolling_vol"], contamination=0.03
)
print(f"Isolation Forest anomalies found: {df['iforest_anomaly'].sum()}")

Isolation Forest works by randomly partitioning the data. Anomalies are isolated in fewer splits because they are far from the bulk of the data. The contamination parameter sets the expected fraction of anomalies. Unlike Z-scores, Isolation Forest handles multivariate data and non-Gaussian distributions naturally.

Compare methods

both = df["zscore_anomaly"] & df["iforest_anomaly"]
either = df["zscore_anomaly"] | df["iforest_anomaly"]

print(f"Z-score only:        {(df['zscore_anomaly'] & ~df['iforest_anomaly']).sum()}")
print(f"Isolation Forest only: {(df['iforest_anomaly'] & ~df['zscore_anomaly']).sum()}")
print(f"Both agree:          {both.sum()}")
print(f"Either flags:        {either.sum()}")

# Known anomalies we injected
known = np.zeros(n_days, dtype=bool)
known[anomaly_idx] = True
df["known_anomaly"] = known

zscore_recall = (df["zscore_anomaly"] & df["known_anomaly"]).sum() / df["known_anomaly"].sum()
iforest_recall = (df["iforest_anomaly"] & df["known_anomaly"]).sum() / df["known_anomaly"].sum()
print(f"\nRecall (Z-score):        {zscore_recall:.0%}")
print(f"Recall (Isolation Forest): {iforest_recall:.0%}")

Rolling Z-score for regime-aware detection

def rolling_zscore_anomalies(
    returns: pd.Series, window: int = 60, threshold: float = 2.5
) -> pd.Series:
    """Z-score computed on a rolling window to adapt to changing volatility."""
    rolling_mean = returns.rolling(window, min_periods=20).mean()
    rolling_std = returns.rolling(window, min_periods=20).std()
    z = (returns - rolling_mean) / rolling_std
    return z.abs() > threshold

df["rolling_zscore_anomaly"] = rolling_zscore_anomalies(df["return"])
print(f"Rolling Z-score anomalies: {df['rolling_zscore_anomaly'].sum()}")

A rolling Z-score adapts to changing market conditions. During a high-volatility period, the rolling standard deviation is larger, so only truly extreme moves get flagged. This avoids the false-positive flood that a static Z-score produces during volatile regimes.

When to use which

MethodBest forLimitation
Z-scoreQuick univariate screeningAssumes static normal distribution
Rolling Z-scoreRegime-aware univariateStill univariate, needs window tuning
Isolation ForestMultivariate, non-parametricHarder to interpret, contamination must be set

In practice, use both. Points flagged by multiple methods are high-confidence anomalies. Points flagged by only one method deserve manual review.
