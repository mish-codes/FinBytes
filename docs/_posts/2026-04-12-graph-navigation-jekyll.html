What You Get

An interactive force-directed graph where every blog post is a node, connected by shared tags. Click a category to expand its posts, click a post to see related content, double-click to navigate. The graph works as a homepage and as a floating overlay accessible from any page.

This is how the FinBytes blog navigation works. This post explains how to build it on any Jekyll site, step by step.

Prerequisites

  • Jekyll blog with posts that have tags in their front matter
  • Posts organised into sections (via a section field, permalink prefix, or categories)
  • Basic familiarity with HTML, JavaScript, and Liquid templates
  • No build tools, npm, or bundlers needed — D3.js is loaded from a CDN

Step 1: Generate Post Data with Liquid

Jekyll’s Liquid templates can output JavaScript objects at build time. This is the key insight: you do not need a separate data file or API. The graph data is baked into the HTML.

<script src="https://d3js.org/d3.v7.min.js"></script>
<script>
(function() {
  var posts = [
    {% for post in site.posts %}
    {
      id: "post-{{ forloop.index }}",
      title: {{ post.title | jsonify }},
      url: "{{ post.url | relative_url }}",
      tags: {{ post.tags | jsonify }},
      section: {{ post.section | default: "" | jsonify }},
      permalink: {{ post.permalink | default: post.url | jsonify }}
    }{% unless forloop.last %},{% endunless %}
    {% endfor %}
  ];
})();
</script>

jsonify handles escaping quotes in titles. relative_url prepends your baseurl so links work on GitHub Pages subpaths.

Step 2: Assign Sections from Metadata

Each node needs a section for colour-coding. You can derive this from any front matter field. Here we check the section field first, then fall back to the permalink prefix:

function getSection(p) {
  if (p.section === "math-finance") return "math-finance";
  if (p.permalink.indexOf("/python") === 0) return "python";
  if (p.permalink.indexOf("/tech-stack") === 0) return "tech-stack";
  return "general";
}

var colorMap = {
  "python": "#4a90d9",
  "tech-stack": "#2ea043",
  "math-finance": "#e8893c",
  "general": "#999"
};

Step 3: Build the Top-Level Graph

The first view shows one large node per section. Group your posts by section and create a node for each:

var postsBySection = {};
allPosts.forEach(function(p) {
  var sec = getSection(p);
  if (!postsBySection[sec]) postsBySection[sec] = [];
  postsBySection[sec].push(p);
});

var sectionNodes = Object.keys(postsBySection).map(function(sec) {
  return {
    id: sec,
    label: sec,
    color: colorMap[sec],
    radius: 40,
    isSection: true
  };
});

Step 4: D3 Force Simulation

The force simulation positions nodes automatically. Four forces work together:

var simulation = d3.forceSimulation(nodes)
  .force("link", d3.forceLink(links)
    .id(function(d) { return d.id; })
    .distance(linkDist))
  .force("charge", d3.forceManyBody()
    .strength(charge))
  .force("center", d3.forceCenter(width / 2, height / 2))
  .force("collision", d3.forceCollide()
    .radius(function(d) { return d.radius + 8; }));
  • link — pulls connected nodes together
  • charge — pushes all nodes apart (negative value = repulsion)
  • center — pulls everything toward the middle of the SVG
  • collision — prevents nodes from overlapping

Step 5: Expanding on Click

When a section node is clicked, replace the graph with that section’s posts radiating from a center node:

function expandSection(sectionId) {
  var center = { id: sectionId, label: sectionId,
                 radius: 25, isSection: true };

  var children = postsBySection[sectionId].map(function(p) {
    return { id: p.id, title: p.title, url: p.url,
             color: p.color, radius: 8 };
  });

  var links = [];
  children.forEach(function(c) {
    links.push({ source: sectionId, target: c.id });
  });

  renderGraph([center].concat(children), links);
}

This gives a clean star layout — parent-to-child spokes only. If you want to show connections between siblings (posts that share tags), add sibling links with a threshold to avoid a dense web:

var TAG_THRESHOLD = 3;

for (var i = 0; i < children.length; i++) {
  for (var j = i + 1; j < children.length; j++) {
    var shared = children[i].tags.filter(function(t) {
      return children[j].tags.indexOf(t) >= 0;
    });
    if (shared.length >= TAG_THRESHOLD) {
      links.push({
        source: children[i].id,
        target: children[j].id,
        weight: shared.length
      });
    }
  }
}

A threshold of 3+ shared tags keeps the graph readable. At 2+ you get a web of lines because generic tags like “python” connect almost everything. At 3+ only genuinely related posts link up. Set the threshold based on how specific your tags are — if you use broad tags, go higher; if your tags are precise, 2 may be fine.

Step 6: Navigate on Click

The click handler covers four cases: clicking the expanded center node goes back, clicking a section expands it, clicking a post at the deepest level navigates, and clicking a post otherwise expands it to show related content.

node.on("click", function(ev, d) {
  if (d.isSection && d.expanded) {
    showTopLevel();
  } else if (d.isSection) {
    expandSection(d.id);
  } else if (alreadyExpanded) {
    window.location.href = d.url;
  } else {
    expandPost(d);
  }
});

Step 7: Slow Rotation After Settling

A subtle touch that makes the graph feel alive: after the force simulation finishes and nodes reach their final positions, start a slow rotation around the center. This draws the eye and signals that the graph is interactive.

var rotateTimer = null;

simulation.on("end", function() {
  var angle = 0;
  rotateTimer = d3.timer(function() {
    angle += 0.05;
    g.attr("transform",
      "rotate(" + angle + "," + (width / 2) + "," + (height / 2) + ")");
  });
});

Stop the rotation when the user interacts — drag, click, or zoom should take priority. Clear the timer at the start of renderGraph (so it stops whenever the graph re-renders) and the zoom handler will naturally override the rotation transform:

function renderGraph(nodes, links, opts) {
  if (rotateTimer) { rotateTimer.stop(); rotateTimer = null; }
  /* ... rest of render ... */
}

The rotation speed of 0.05 degrees per frame gives roughly one full revolution every two minutes — slow enough to feel ambient, fast enough to notice.

Step 8: Add It to Every Page (Floating Button)

To make the graph accessible from any page, override your theme’s base layout and inject a floating button + full-screen overlay:

<!-- In _layouts/default.html, before </body> -->
{% include graph-overlay.html %}

The overlay include contains:

  • A fixed-position button (bottom-right corner)
  • A hidden div with position: fixed; width: 100vw; height: 100vh
  • The same D3 script and Liquid data generation
  • A close button and Escape key handler

Skip the overlay on the homepage (where the graph is already inline) using a Liquid conditional:

{% unless page.permalink == "/" %}
  <!-- floating button + overlay -->
{% endunless %}

Gotchas

HTML compressors eat JavaScript comments

If your Jekyll theme uses an HTML compressor (Chirpy does via layout: compress), it strips newlines. A // comment in inline JavaScript will swallow everything after it on the same line — including your actual code. Never use // comments in inline scripts. Use /* */ if you must comment, or just don’t comment inline JS.

Container dimensions at script execution time

If the graph container has zero width/height when the script runs (common with theme layouts that defer rendering), the SVG viewBox will be [0,0,0,0] and nothing will render. Use a polling init:

function tryInit() {
  if (container.clientWidth > 0 && container.clientHeight > 0) {
    initGraph();
  } else {
    requestAnimationFrame(tryInit);
  }
}
tryInit();

Force tuning depends on node count

A section with 5 posts and a section with 25 posts need different force parameters. Scale link distance and charge with the number of visible nodes:

var diag = Math.sqrt(width * width + height * height);
var scale = Math.max(1, children.length / 10);
var linkDist = diag * 0.06 * scale;
var charge = -diag * 0.2 * scale;

Sidebar overlap

If your theme has a fixed sidebar, the graph container’s clientWidth reflects the available space (not the viewport). On the graph homepage, you can auto-hide the sidebar with CSS:

#sidebar {
  transform: translateX(-100%);
  transition: transform 0.3s ease;
}
#sidebar:hover {
  transform: translateX(0);
}
