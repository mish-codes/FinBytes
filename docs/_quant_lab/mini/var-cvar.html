What it does

Computes Value at Risk (VaR) and Conditional Value at Risk (CVaR / Expected Shortfall) using both historical and parametric methods. Visualizes the return distribution with VaR and CVaR threshold lines.

The math

VaR (95%): the 5th percentile of the return distribution -- on 95 out of 100 days, the loss will not exceed this value. CVaR: the mean of all losses beyond VaR. If VaR is the door to the tail, CVaR tells you what the room behind it looks like. Parametric VaR assumes normality and reads off ppf(0.05, mu, sigma).

When to use it

Use both methods and compare. When parametric VaR diverges significantly from historical VaR, your returns are non-normal and the parametric estimate is unreliable. CVaR is preferred by regulators because it captures tail severity, not just the threshold.

Core function

import numpy as np
from scipy import stats


def historical_var(returns: np.ndarray, confidence: float = 0.95) -> float:
    """VaR as the percentile of the return distribution."""
    return float(np.percentile(returns, (1 - confidence) * 100))


def parametric_var(returns: np.ndarray, confidence: float = 0.95) -> float:
    """VaR assuming returns are normally distributed."""
    mu = np.mean(returns)
    sigma = np.std(returns)
    return float(stats.norm.ppf(1 - confidence, loc=mu, scale=sigma))


def historical_cvar(returns: np.ndarray, confidence: float = 0.95) -> float:
    """Average loss beyond the VaR threshold."""
    var = historical_var(returns, confidence)
    tail_losses = returns[returns <= var]
    return float(np.mean(tail_losses))

Quick usage

np.random.seed(42)
returns = np.random.normal(0.0004, 0.015, 504)

print(f"Historical VaR (95%): {historical_var(returns):.4f}")
print(f"Parametric VaR (95%): {parametric_var(returns):.4f}")
print(f"CVaR (95%):           {historical_cvar(returns):.4f}")
