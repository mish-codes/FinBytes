Swapping behaviour at runtime

The Strategy pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. The caller uses an interface and does not know which concrete implementation it talks to.

In finance: a pricing engine supports Black-Scholes for vanilla options, Monte Carlo for exotics, and a binomial tree for American options. The caller calls pricer.price(trade) - which algorithm runs is determined by config or trade type.

C++ offers two approaches. Virtual dispatch (runtime) has a small overhead per call but lets you change strategy at runtime. Templates (compile-time) have zero overhead but the strategy must be fixed at compile time. For a hot pricing loop, the template approach is worth benchmarking.

Python strategy pattern

from abc import ABC, abstractmethod
from dataclasses import dataclass
import math
from scipy.stats import norm

@dataclass
class OptionTrade:
    S: float; K: float; T: float; r: float; sigma: float; is_call: bool = True

class PricingStrategy(ABC):
    @abstractmethod
    def price(self, trade: OptionTrade) -> float: ...

class BlackScholesStrategy(PricingStrategy):
    def price(self, t: OptionTrade) -> float:
        d1 = (math.log(t.S/t.K) + (t.r + 0.5*t.sigma**2)*t.T) / (t.sigma*math.sqrt(t.T))
        d2 = d1 - t.sigma * math.sqrt(t.T)
        if t.is_call:
            return t.S*norm.cdf(d1) - t.K*math.exp(-t.r*t.T)*norm.cdf(d2)
        return t.K*math.exp(-t.r*t.T)*norm.cdf(-d2) - t.S*norm.cdf(-d1)

class IntrinsicValueStrategy(PricingStrategy):
    def price(self, t: OptionTrade) -> float:
        return max(t.S - t.K, 0) if t.is_call else max(t.K - t.S, 0)

class OptionPricer:
    def __init__(self, strategy: PricingStrategy): self._strategy = strategy
    def set_strategy(self, s: PricingStrategy) -> None: self._strategy = s
    def price(self, trade: OptionTrade) -> float: return self._strategy.price(trade)

trade  = OptionTrade(S=100, K=100, T=1, r=0.05, sigma=0.2)
pricer = OptionPricer(BlackScholesStrategy())
print(f"BS:       {pricer.price(trade):.4f}")
pricer.set_strategy(IntrinsicValueStrategy())
print(f"Intrinsic:{pricer.price(trade):.4f}")

C# strategy pattern

public record OptionTrade(double S, double K, double T, double r, double Sigma, bool IsCall = true);

public interface IPricingStrategy { double Price(OptionTrade trade); }

public class BlackScholesStrategy : IPricingStrategy {
    public double Price(OptionTrade t) {
        double d1 = (Math.Log(t.S/t.K) + (t.r + 0.5*t.Sigma*t.Sigma)*t.T) / (t.Sigma*Math.Sqrt(t.T));
        double d2 = d1 - t.Sigma * Math.Sqrt(t.T);
        return t.IsCall
            ? t.S * N(d1) - t.K * Math.Exp(-t.r*t.T) * N(d2)
            : t.K * Math.Exp(-t.r*t.T) * N(-d2) - t.S * N(-d1);
    }
    static double N(double x) => 0.5 * (1 + Math.Erf(x / Math.Sqrt(2)));
}

public class OptionPricer {
    private IPricingStrategy _strategy;
    public OptionPricer(IPricingStrategy s) => _strategy = s;
    public void SetStrategy(IPricingStrategy s) => _strategy = s;
    public double Price(OptionTrade t) => _strategy.Price(t);
}

// C# also supports Func as a lightweight strategy
Func<OptionTrade, double> flatStrategy = t => 5.0;

C++ strategy - runtime and compile-time

#include <memory>
#include <cmath>
#include <functional>

struct OptionTrade { double S, K, T, r, sigma; bool is_call = true; };

// Runtime strategy (virtual dispatch)
struct IPricingStrategy {
    virtual ~IPricingStrategy() = default;
    virtual double price(const OptionTrade& t) const = 0;
};

struct BlackScholesStrategy : IPricingStrategy {
    static double ncdf(double x) { return 0.5 * std::erfc(-x / std::sqrt(2.0)); }
    double price(const OptionTrade& t) const override {
        double d1 = (std::log(t.S/t.K) + (t.r + 0.5*t.sigma*t.sigma)*t.T)
                    / (t.sigma * std::sqrt(t.T));
        double d2 = d1 - t.sigma * std::sqrt(t.T);
        return t.is_call
            ? t.S * ncdf(d1) - t.K * std::exp(-t.r*t.T) * ncdf(d2)
            : t.K * std::exp(-t.r*t.T) * ncdf(-d2) - t.S * ncdf(-d1);
    }
};

class DynamicPricer {
    std::unique_ptr<IPricingStrategy> strategy_;
public:
    explicit DynamicPricer(std::unique_ptr<IPricingStrategy> s) : strategy_(std::move(s)) {}
    void set_strategy(std::unique_ptr<IPricingStrategy> s) { strategy_ = std::move(s); }
    double price(const OptionTrade& t) const { return strategy_->price(t); }
};

// Compile-time strategy (zero-overhead template)
template<typename Strategy>
class StaticPricer {
    Strategy strategy_;
public:
    double price(const OptionTrade& t) const { return strategy_.price(t); }
};

StaticPricer<BlackScholesStrategy> fast_pricer;
For batch pricing millions of options, the template strategy avoids a virtual dispatch per call — typically a 5-15ns saving that adds up. For one-at-a-time requests, virtual dispatch is fine and the flexibility is worth more than the nanoseconds.

Rust strategy pattern

use std::f64::consts::PI;

struct OptionTrade { s: f64, k: f64, t: f64, r: f64, sigma: f64, is_call: bool }

trait PricingStrategy {
    fn price(&self, trade: &OptionTrade) -> f64;
}

struct BlackScholesStrategy;

impl BlackScholesStrategy {
    fn ncdf(x: f64) -> f64 { 0.5 * (1.0 + libm::erf(x / (2.0_f64).sqrt())) }
}

impl PricingStrategy for BlackScholesStrategy {
    fn price(&self, t: &OptionTrade) -> f64 {
        let d1 = ((t.s / t.k).ln() + (t.r + 0.5 * t.sigma.powi(2)) * t.t)
                  / (t.sigma * t.t.sqrt());
        let d2 = d1 - t.sigma * t.t.sqrt();
        if t.is_call {
            t.s * Self::ncdf(d1) - t.k * (-t.r * t.t).exp() * Self::ncdf(d2)
        } else {
            t.k * (-t.r * t.t).exp() * Self::ncdf(-d2) - t.s * Self::ncdf(-d1)
        }
    }
}

struct IntrinsicStrategy;
impl PricingStrategy for IntrinsicStrategy {
    fn price(&self, t: &OptionTrade) -> f64 {
        if t.is_call { (t.s - t.k).max(0.0) } else { (t.k - t.s).max(0.0) }
    }
}

// Runtime dispatch (dyn Trait)
struct DynamicPricer {
    strategy: Box<dyn PricingStrategy>,
}
impl DynamicPricer {
    fn new(s: Box<dyn PricingStrategy>) -> Self { Self { strategy: s } }
    fn price(&self, t: &OptionTrade) -> f64 { self.strategy.price(t) }
}

// Compile-time dispatch (generic — zero cost)
struct StaticPricer<S: PricingStrategy> { strategy: S }
impl<S: PricingStrategy> StaticPricer<S> {
    fn price(&self, t: &OptionTrade) -> f64 { self.strategy.price(t) }
}
Rust mirrors C++'s dual approach: Box<dyn Trait> for runtime dispatch (like unique_ptr<Interface>) and generics for compile-time dispatch (like templates). Unlike C++, the compiler enforces that you handle ownership correctly — Box is the only owner, no accidental dangling pointers.

Go strategy pattern

package main

import "math"

type OptionTrade struct {
    S, K, T, R, Sigma float64
    IsCall            bool
}

type PricingStrategy interface {
    Price(trade OptionTrade) float64
}

type BlackScholesStrategy struct{}

func ncdf(x float64) float64 { return 0.5 * math.Erfc(-x/math.Sqrt2) }

func (bs BlackScholesStrategy) Price(t OptionTrade) float64 {
    d1 := (math.Log(t.S/t.K) + (t.R+0.5*t.Sigma*t.Sigma)*t.T) /
          (t.Sigma * math.Sqrt(t.T))
    d2 := d1 - t.Sigma*math.Sqrt(t.T)
    if t.IsCall {
        return t.S*ncdf(d1) - t.K*math.Exp(-t.R*t.T)*ncdf(d2)
    }
    return t.K*math.Exp(-t.R*t.T)*ncdf(-d2) - t.S*ncdf(-d1)
}

type IntrinsicStrategy struct{}

func (is IntrinsicStrategy) Price(t OptionTrade) float64 {
    if t.IsCall { return math.Max(t.S-t.K, 0) }
    return math.Max(t.K-t.S, 0)
}

type OptionPricer struct {
    strategy PricingStrategy
}

func NewOptionPricer(s PricingStrategy) *OptionPricer {
    return &OptionPricer{strategy: s}
}

func (p *OptionPricer) SetStrategy(s PricingStrategy) { p.strategy = s }

func (p *OptionPricer) Price(t OptionTrade) float64 {
    return p.strategy.Price(t)
}

// Go also supports function-as-strategy
type PriceFunc func(OptionTrade) float64

func (f PriceFunc) Price(t OptionTrade) float64 { return f(t) }

// Use: NewOptionPricer(PriceFunc(func(t OptionTrade) float64 { return 5.0 }))
Go only has runtime dispatch — there are no generics-based strategies like C++ templates or Rust generics. The PriceFunc adapter at the bottom is idiomatic Go: it lets any plain function satisfy the PricingStrategy interface, similar to C#'s Func<T>.

TypeScript strategy pattern

interface OptionTrade {
  S: number; K: number; T: number; r: number; sigma: number; isCall: boolean;
}

interface PricingStrategy {
  price(trade: OptionTrade): number;
}

class BlackScholesStrategy implements PricingStrategy {
  private ncdf(x: number): number {
    // Abramowitz & Stegun approximation
    const a1 = 0.254829592, a2 = -0.284496736, a3 = 1.421413741;
    const a4 = -1.453152027, a5 = 1.061405429, p = 0.3275911;
    const sign = x < 0 ? -1 : 1;
    const t = 1 / (1 + p * Math.abs(x));
    const y = 1 - ((((a5*t + a4)*t + a3)*t + a2)*t + a1)*t * Math.exp(-x*x/2);
    return 0.5 * (1 + sign * y);
  }

  price(t: OptionTrade): number {
    const d1 = (Math.log(t.S/t.K) + (t.r + 0.5*t.sigma**2)*t.T) /
               (t.sigma * Math.sqrt(t.T));
    const d2 = d1 - t.sigma * Math.sqrt(t.T);
    return t.isCall
      ? t.S * this.ncdf(d1) - t.K * Math.exp(-t.r*t.T) * this.ncdf(d2)
      : t.K * Math.exp(-t.r*t.T) * this.ncdf(-d2) - t.S * this.ncdf(-d1);
  }
}

class IntrinsicStrategy implements PricingStrategy {
  price(t: OptionTrade): number {
    return t.isCall ? Math.max(t.S - t.K, 0) : Math.max(t.K - t.S, 0);
  }
}

class OptionPricer {
  constructor(private strategy: PricingStrategy) {}
  setStrategy(s: PricingStrategy): void { this.strategy = s; }
  price(trade: OptionTrade): number { return this.strategy.price(trade); }
}

// Function-as-strategy — TypeScript's first-class functions make this natural
type PriceFn = (trade: OptionTrade) => number;

const flatPrice: PriceFn = (_t) => 5.0;
TypeScript's structural typing means you don't actually need the implements PricingStrategy keyword — any object with a price(trade: OptionTrade): number method satisfies the interface. The explicit implements is optional documentation that gives you a compile error if you drift.

Key observations

  • Compile-time strategy is a C++ and Rust exclusive. Only these two languages let you eliminate virtual dispatch entirely via templates/generics. For a pricing engine doing millions of valuations per second, this is the difference between 5ns and 20ns per call — which adds up to seconds over a batch.
  • Python and TypeScript are the most concise — first-class functions mean you can often skip the class entirely and pass a plain function. The downside: no compile-time check that your function signature matches the expected strategy interface (Python) or that it matches at runtime (TypeScript — interfaces are erased).
  • Go's adapter pattern is elegant. The PriceFunc type that wraps a plain function into an interface implementation is a uniquely Go idiom. The standard library uses this everywhere (http.HandlerFunc).
  • Rust ownership adds friction. Swapping strategies at runtime with Box<dyn Trait> is slightly more ceremony than other languages because ownership must be transferred explicitly. The payoff: no accidental shared mutable state between the old and new strategy.
  • C# is the pragmatic middle ground. Strong typing, clean interface syntax, lambda support for lightweight strategies, and runtime performance that's good enough for most use cases (JIT-compiled, not interpreted).

Decision guide for the Strategy pattern

  • Strategy chosen at startup, never changes: C++ template or Rust generic (zero-cost)
  • Strategy changes based on user input or config: any language's runtime dispatch works; pick by team expertise
  • Quick prototype or notebook: Python function-as-strategy (skip the class)
  • Millions of calls per second: C++ template or Rust generic; benchmark virtual dispatch first — it may be fast enough
  • Microservice with strategy per endpoint: Go or C# (clean DI, easy testing)
