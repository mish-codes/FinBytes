Encapsulating operations as objects

The Command pattern encapsulates a request as an object. This gives you undo/redo (keep the command, reverse it later), queuing (store commands for later execution), logging (every operation is an object you can serialise), and replay (reconstruct state from scratch by replaying commands).

In an OMS: every change to a trade - quantity adjustment, price correction, cancel - is a Command. The command history is the audit trail. Undo reverses the last change. Replay reconstructs the position from scratch.

Python command pattern

from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Optional

@dataclass
class Position:
    symbol: str; quantity: float = 0.0; avg_price: float = 0.0

class Command(ABC):
    @abstractmethod
    def execute(self) -> None: ...
    @abstractmethod
    def undo(self) -> None: ...
    @abstractmethod
    def description(self) -> str: ...

class AdjustQuantity(Command):
    def __init__(self, position: Position, delta: float):
        self._pos = position; self._delta = delta
    def execute(self) -> None: self._pos.quantity += self._delta
    def undo(self) -> None:    self._pos.quantity -= self._delta
    def description(self) -> str: return f"Adjust {self._pos.symbol} qty by {self._delta:+.0f}"

class CorrectPrice(Command):
    def __init__(self, position: Position, new_price: float):
        self._pos = position; self._new = new_price; self._old = position.avg_price
    def execute(self) -> None: self._pos.avg_price = self._new
    def undo(self) -> None:    self._pos.avg_price = self._old
    def description(self) -> str: return f"Correct {self._pos.symbol} price {self._old:.4f}->{self._new:.4f}"

class CommandHistory:
    def __init__(self): self._done: list[Command] = []; self._undone: list[Command] = []
    def execute(self, cmd: Command) -> None:
        cmd.execute(); self._done.append(cmd); self._undone.clear()
    def undo(self) -> Optional[str]:
        if not self._done: return None
        cmd = self._done.pop(); cmd.undo(); self._undone.append(cmd)
        return f"Undone: {cmd.description()}"
    def audit_trail(self) -> list[str]:
        return [c.description() for c in self._done]

pos     = Position("AAPL", 0, 150.0)
history = CommandHistory()
history.execute(AdjustQuantity(pos, 100))
history.execute(CorrectPrice(pos, 151.25))
print(history.audit_trail())
history.undo()
print(pos.avg_price)  # 150.0

C# command pattern

public class Position { public string Symbol = ""; public double Qty; public double AvgPrice; }

public interface ICommand {
    void Execute(); void Undo(); string Description { get; }
}

public class AdjustQty(Position pos, double delta) : ICommand {
    public void Execute() => pos.Qty += delta;
    public void Undo()    => pos.Qty -= delta;
    public string Description => $"Adjust {pos.Symbol} qty by {delta:+0;-0}";
}

public class CommandHistory {
    private readonly Stack<ICommand> _done   = new();
    private readonly Stack<ICommand> _undone = new();
    public void Execute(ICommand cmd) { cmd.Execute(); _done.Push(cmd); _undone.Clear(); }
    public void Undo() { if (_done.TryPop(out var cmd)) { cmd.Undo(); _undone.Push(cmd); } }
    public void Redo() { if (_undone.TryPop(out var cmd)) { cmd.Execute(); _done.Push(cmd); } }
    public IEnumerable<string> AuditTrail() => _done.Select(c => c.Description);
}

C++ command pattern

#include <memory>
#include <vector>
#include <string>

struct Position { std::string symbol; double quantity = 0.0; double avg_price = 0.0; };

struct ICommand {
    virtual ~ICommand() = default;
    virtual void        execute()           = 0;
    virtual void        undo()              = 0;
    virtual std::string description() const = 0;
};

class AdjustQuantity : public ICommand {
    Position& pos_; double delta_;
public:
    AdjustQuantity(Position& pos, double delta) : pos_(pos), delta_(delta) {}
    void execute() override { pos_.quantity += delta_; }
    void undo()    override { pos_.quantity -= delta_; }
    std::string description() const override {
        return "Adjust " + pos_.symbol + " qty by " + std::to_string((int)delta_);
    }
};

class CorrectPrice : public ICommand {
    Position& pos_; double new_price_, old_price_;
public:
    CorrectPrice(Position& pos, double np)
        : pos_(pos), new_price_(np), old_price_(pos.avg_price) {}
    void execute() override { pos_.avg_price = new_price_; }
    void undo()    override { pos_.avg_price = old_price_; }
    std::string description() const override {
        return "Correct " + pos_.symbol + " price";
    }
};

class CommandHistory {
    std::vector<std::unique_ptr<ICommand>> done_, undone_;
public:
    void execute(std::unique_ptr<ICommand> cmd) {
        cmd->execute(); done_.push_back(std::move(cmd)); undone_.clear();
    }
    void undo() {
        if (done_.empty()) return;
        done_.back()->undo();
        undone_.push_back(std::move(done_.back())); done_.pop_back();
    }
    std::vector<std::string> audit_trail() const {
        std::vector<std::string> log;
        for (const auto& c : done_) log.push_back(c->description());
        return log;
    }
};
The Command pattern's audit trail property makes it ideal for financial systems where every change must be traceable. Serialise commands to a database as they execute and you have a complete, replayable history of every position change - invaluable for reconciliation and regulatory reporting.

Rust command pattern

struct Position {
    symbol:    String,
    quantity:  f64,
    avg_price: f64,
}

trait Command {
    fn execute(&mut self, pos: &mut Position);
    fn undo(&mut self, pos: &mut Position);
    fn description(&self) -> String;
}

struct AdjustQuantity { delta: f64 }

impl Command for AdjustQuantity {
    fn execute(&mut self, pos: &mut Position) { pos.quantity += self.delta; }
    fn undo(&mut self, pos: &mut Position)    { pos.quantity -= self.delta; }
    fn description(&self) -> String {
        format!("Adjust qty by {:+.0}", self.delta)
    }
}

struct CorrectPrice { new_price: f64, old_price: f64 }

impl CorrectPrice {
    fn new(new_price: f64, current_price: f64) -> Self {
        Self { new_price, old_price: current_price }
    }
}

impl Command for CorrectPrice {
    fn execute(&mut self, pos: &mut Position) { pos.avg_price = self.new_price; }
    fn undo(&mut self, pos: &mut Position)    { pos.avg_price = self.old_price; }
    fn description(&self) -> String {
        format!("Correct price {:.4} -> {:.4}", self.old_price, self.new_price)
    }
}

struct CommandHistory {
    done:   Vec<Box<dyn Command>>,
    undone: Vec<Box<dyn Command>>,
}

impl CommandHistory {
    fn new() -> Self { Self { done: Vec::new(), undone: Vec::new() } }

    fn execute(&mut self, mut cmd: Box<dyn Command>, pos: &mut Position) {
        cmd.execute(pos);
        self.done.push(cmd);
        self.undone.clear();
    }

    fn undo(&mut self, pos: &mut Position) -> Option<String> {
        let mut cmd = self.done.pop()?;
        cmd.undo(pos);
        let desc = cmd.description();
        self.undone.push(cmd);
        Some(format!("Undone: {desc}"))
    }

    fn redo(&mut self, pos: &mut Position) -> Option<String> {
        let mut cmd = self.undone.pop()?;
        cmd.execute(pos);
        let desc = cmd.description();
        self.done.push(cmd);
        Some(format!("Redone: {desc}"))
    }

    fn audit_trail(&self) -> Vec<String> {
        self.done.iter().map(|c| c.description()).collect()
    }
}

let mut pos = Position { symbol: "AAPL".into(), quantity: 0.0, avg_price: 150.0 };
let mut history = CommandHistory::new();
history.execute(Box::new(AdjustQuantity { delta: 100.0 }), &mut pos);
history.execute(Box::new(CorrectPrice::new(151.25, pos.avg_price)), &mut pos);
history.undo(&mut pos); // avg_price back to 150.0
Rust commands take &mut Position as a parameter rather than storing a reference to it. This avoids lifetime complexity — the history owns the commands, and the position is passed in at execution time. The Box<dyn Command> enables heterogeneous command storage. For maximum performance, use an enum of command types instead of trait objects.

Go command pattern

package main

import "fmt"

type Position struct {
    Symbol   string
    Quantity float64
    AvgPrice float64
}

type Command interface {
    Execute(pos *Position)
    Undo(pos *Position)
    Description() string
}

type AdjustQuantity struct{ Delta float64 }

func (c AdjustQuantity) Execute(pos *Position) { pos.Quantity += c.Delta }
func (c AdjustQuantity) Undo(pos *Position)    { pos.Quantity -= c.Delta }
func (c AdjustQuantity) Description() string {
    return fmt.Sprintf("Adjust qty by %+.0f", c.Delta)
}

type CorrectPrice struct{ NewPrice, OldPrice float64 }

func (c CorrectPrice) Execute(pos *Position) { pos.AvgPrice = c.NewPrice }
func (c CorrectPrice) Undo(pos *Position)    { pos.AvgPrice = c.OldPrice }
func (c CorrectPrice) Description() string {
    return fmt.Sprintf("Correct price %.4f -> %.4f", c.OldPrice, c.NewPrice)
}

type CommandHistory struct {
    done   []Command
    undone []Command
}

func (h *CommandHistory) Execute(cmd Command, pos *Position) {
    cmd.Execute(pos)
    h.done = append(h.done, cmd)
    h.undone = nil
}

func (h *CommandHistory) Undo(pos *Position) string {
    if len(h.done) == 0 { return "" }
    cmd := h.done[len(h.done)-1]
    h.done = h.done[:len(h.done)-1]
    cmd.Undo(pos)
    h.undone = append(h.undone, cmd)
    return "Undone: " + cmd.Description()
}

func (h *CommandHistory) AuditTrail() []string {
    trail := make([]string, len(h.done))
    for i, c := range h.done { trail[i] = c.Description() }
    return trail
}

pos := &Position{Symbol: "AAPL", Quantity: 0, AvgPrice: 150.0}
history := &CommandHistory{}
history.Execute(AdjustQuantity{Delta: 100}, pos)
history.Execute(CorrectPrice{NewPrice: 151.25, OldPrice: pos.AvgPrice}, pos)
history.Undo(pos)
Go's command pattern is straightforward — value-type commands stored in slices. Since Go uses garbage collection, there is no ownership concern when storing commands in the history. Commands are passed by value (small structs), making copies cheap. For large commands, use pointers.

TypeScript command pattern

interface Position {
  symbol: string;
  quantity: number;
  avgPrice: number;
}

interface Command {
  execute(pos: Position): void;
  undo(pos: Position): void;
  readonly description: string;
}

class AdjustQuantity implements Command {
  readonly description: string;
  constructor(private delta: number) {
    this.description = `Adjust qty by ${delta >= 0 ? "+" : ""}${delta}`;
  }
  execute(pos: Position): void { pos.quantity += this.delta; }
  undo(pos: Position): void    { pos.quantity -= this.delta; }
}

class CorrectPrice implements Command {
  readonly description: string;
  private oldPrice: number;
  constructor(private newPrice: number, currentPrice: number) {
    this.oldPrice = currentPrice;
    this.description = `Correct price ${currentPrice.toFixed(4)} -> ${newPrice.toFixed(4)}`;
  }
  execute(pos: Position): void { pos.avgPrice = this.newPrice; }
  undo(pos: Position): void    { pos.avgPrice = this.oldPrice; }
}

class CommandHistory {
  private done:   Command[] = [];
  private undone: Command[] = [];

  execute(cmd: Command, pos: Position): void {
    cmd.execute(pos);
    this.done.push(cmd);
    this.undone = [];
  }

  undo(pos: Position): string | undefined {
    const cmd = this.done.pop();
    if (!cmd) return undefined;
    cmd.undo(pos);
    this.undone.push(cmd);
    return `Undone: ${cmd.description}`;
  }

  redo(pos: Position): string | undefined {
    const cmd = this.undone.pop();
    if (!cmd) return undefined;
    cmd.execute(pos);
    this.done.push(cmd);
    return `Redone: ${cmd.description}`;
  }

  auditTrail(): string[] {
    return this.done.map(c => c.description);
  }
}

const pos: Position = { symbol: "AAPL", quantity: 0, avgPrice: 150.0 };
const history = new CommandHistory();
history.execute(new AdjustQuantity(100), pos);
history.execute(new CorrectPrice(151.25, pos.avgPrice), pos);
console.log(history.auditTrail());
history.undo(pos);
console.log(pos.avgPrice); // 150.0
TypeScript commands use mutable objects — pos.quantity += delta mutates the position directly. This works well for in-process undo/redo. For a distributed system, consider immutable command events (event sourcing) where each command produces a new state snapshot rather than mutating in place.

Key observations

  • Rust's ownership model prevents dangling command references. Commands cannot hold stale references to destroyed positions — the compiler rejects it. The trade-off: commands take the position as a parameter rather than storing a reference, which adds a parameter to every call.
  • C# and TypeScript excel at serialisation — JSON serialisation of command objects is trivial, making them natural fits for event sourcing and audit trails in financial systems.
  • C++ must manage command ownership carefully. The history owns unique_ptr<ICommand> objects. If a command holds a reference to a position that is destroyed, it becomes a dangling pointer. Rust prevents this at compile time; C++ does not.
  • Go's value-type commands are simple and efficient. Small command structs are copied into the history slice. No heap allocation, no pointer management. For concurrent command processing, Go's channels naturally queue commands for serial execution.
  • Python is the quickest to prototype — the command pattern works well for interactive notebook-style exploration where you want to undo/redo position adjustments.

Decision guide for the Command pattern

  • Trade amendment audit trail: C# or TypeScript (easy serialisation to JSON/database)
  • Event sourcing / replay: any language works; Go channels for concurrent command processing
  • Interactive undo/redo (UI): TypeScript (frontend) or Python (notebook)
  • High-frequency command execution: C++ or Rust (minimal overhead per command)
  • Safety-critical position management: Rust (compile-time prevention of stale references)
