A tree stored in an array

A min-heap is a complete binary tree where every parent is smaller than or equal to its children. The smallest element is always at the root. Because the tree is complete, we can store it in a flat array with no pointers:

  • Parent of index i: (i - 1) / 2
  • Left child of index i: 2*i + 1
  • Right child of index i: 2*i + 2

Push: append to the end, then sift up — swap with parent while smaller. O(log n). Pop: swap root with last element, remove last, then sift down — swap with smallest child while larger. O(log n). Peek: return index 0. O(1).

Heapify builds a heap from an unsorted array in O(n) by sifting down from the last non-leaf to index 0. This is faster than pushing n elements individually (O(n log n)).

The stdlib APIs across languages are surprisingly inconsistent — some default to max-heap, some to min-heap, some use classes, others use bare functions on arrays.

Python — MinHeap class (heapq is functions-on-list)

class MinHeap:
    def __init__(self, items=None):
        self._data = list(items) if items else []
        if self._data:
            self._heapify()

    def __len__(self):
        return len(self._data)

    def push(self, val):
        self._data.append(val)
        self._sift_up(len(self._data) - 1)

    def pop(self):
        if not self._data:
            raise IndexError("pop from empty heap")
        self._swap(0, len(self._data) - 1)
        val = self._data.pop()
        if self._data:
            self._sift_down(0)
        return val

    def peek(self):
        if not self._data:
            raise IndexError("peek at empty heap")
        return self._data[0]

    def _sift_up(self, i):
        while i > 0:
            parent = (i - 1) // 2
            if self._data[i] < self._data[parent]:
                self._swap(i, parent)
                i = parent
            else:
                break

    def _sift_down(self, i):
        n = len(self._data)
        while True:
            smallest = i
            left, right = 2*i + 1, 2*i + 2
            if left < n and self._data[left] < self._data[smallest]:
                smallest = left
            if right < n and self._data[right] < self._data[smallest]:
                smallest = right
            if smallest == i:
                break
            self._swap(i, smallest)
            i = smallest

    def _heapify(self):
        for i in range(len(self._data) // 2 - 1, -1, -1):
            self._sift_down(i)

    def _swap(self, i, j):
        self._data[i], self._data[j] = self._data[j], self._data[i]

h = MinHeap([5, 3, 8, 1, 2])
print(h.pop(), h.pop(), h.pop())  # 1 2 3
Python's heapq module provides heappush and heappop as standalone functions that operate on a plain list. There is no Heap class in the stdlib. This design is minimal but means you can accidentally break the heap invariant by modifying the list directly.

C# — generic MinHeap

public class MinHeap<T> where T : IComparable<T>
{
    private readonly List<T> _data = new();

    public int Count => _data.Count;

    public MinHeap() { }
    public MinHeap(IEnumerable<T> items)
    {
        _data.AddRange(items);
        for (int i = _data.Count / 2 - 1; i >= 0; i--) SiftDown(i);
    }

    public void Push(T val)
    {
        _data.Add(val);
        SiftUp(_data.Count - 1);
    }

    public T Pop()
    {
        if (_data.Count == 0) throw new InvalidOperationException("empty");
        T val = _data[0];
        _data[0] = _data[^1];
        _data.RemoveAt(_data.Count - 1);
        if (_data.Count > 0) SiftDown(0);
        return val;
    }

    public T Peek() => _data.Count > 0
        ? _data[0]
        : throw new InvalidOperationException("empty");

    private void SiftUp(int i)
    {
        while (i > 0)
        {
            int p = (i - 1) / 2;
            if (_data[i].CompareTo(_data[p]) < 0) { Swap(i, p); i = p; }
            else break;
        }
    }

    private void SiftDown(int i)
    {
        int n = _data.Count;
        while (true)
        {
            int sm = i, l = 2*i+1, r = 2*i+2;
            if (l < n && _data[l].CompareTo(_data[sm]) < 0) sm = l;
            if (r < n && _data[r].CompareTo(_data[sm]) < 0) sm = r;
            if (sm == i) break;
            Swap(i, sm); i = sm;
        }
    }

    private void Swap(int i, int j) { (_data[i], _data[j]) = (_data[j], _data[i]); }
}

var h = new MinHeap<int>(new[] { 5, 3, 8, 1, 2 });
Console.WriteLine($"{h.Pop()} {h.Pop()} {h.Pop()}"); // 1 2 3
.NET 6+ added PriorityQueue<TElement, TPriority> which separates element from priority. Our implementation is simpler — the element itself is comparable. The stdlib version is a min-heap by default.

C++ — min-heap (stdlib defaults to max)

#include <vector>
#include <stdexcept>
#include <algorithm>

template<typename T>
class MinHeap {
    std::vector<T> data_;

    void sift_up(size_t i) {
        while (i > 0) {
            size_t p = (i - 1) / 2;
            if (data_[i] < data_[p]) { std::swap(data_[i], data_[p]); i = p; }
            else break;
        }
    }

    void sift_down(size_t i) {
        size_t n = data_.size();
        while (true) {
            size_t sm = i, l = 2*i+1, r = 2*i+2;
            if (l < n && data_[l] < data_[sm]) sm = l;
            if (r < n && data_[r] < data_[sm]) sm = r;
            if (sm == i) break;
            std::swap(data_[i], data_[sm]);
            i = sm;
        }
    }

public:
    MinHeap() = default;

    explicit MinHeap(std::vector<T> items) : data_(std::move(items)) {
        for (int i = static_cast<int>(data_.size()) / 2 - 1; i >= 0; --i)
            sift_down(static_cast<size_t>(i));
    }

    size_t size() const { return data_.size(); }
    bool   empty() const { return data_.empty(); }

    void push(T val) {
        data_.push_back(std::move(val));
        sift_up(data_.size() - 1);
    }

    T pop() {
        if (data_.empty()) throw std::out_of_range("empty");
        T val = std::move(data_.front());
        data_.front() = std::move(data_.back());
        data_.pop_back();
        if (!data_.empty()) sift_down(0);
        return val;
    }

    const T& peek() const {
        if (data_.empty()) throw std::out_of_range("empty");
        return data_.front();
    }
};
std::priority_queue is a max-heap by default. To get a min-heap you write priority_queue<int, vector<int>, greater<int>> — three template parameters to flip the comparison. Our custom class avoids this awkwardness.

Rust — min-heap (BinaryHeap is max-only)

pub struct MinHeap<T: Ord> {
    data: Vec<T>,
}

impl<T: Ord> MinHeap<T> {
    pub fn new() -> Self { Self { data: Vec::new() } }

    pub fn from_vec(mut items: Vec<T>) -> Self {
        let mut heap = Self { data: items };
        let n = heap.data.len();
        for i in (0..n / 2).rev() {
            heap.sift_down(i);
        }
        heap
    }

    pub fn len(&self) -> usize { self.data.len() }

    pub fn push(&mut self, val: T) {
        self.data.push(val);
        let i = self.data.len() - 1;
        self.sift_up(i);
    }

    pub fn pop(&mut self) -> Option<T> {
        if self.data.is_empty() { return None; }
        let last = self.data.len() - 1;
        self.data.swap(0, last);
        let val = self.data.pop();
        if !self.data.is_empty() { self.sift_down(0); }
        val
    }

    pub fn peek(&self) -> Option<&T> { self.data.first() }

    fn sift_up(&mut self, mut i: usize) {
        while i > 0 {
            let p = (i - 1) / 2;
            if self.data[i] < self.data[p] {
                self.data.swap(i, p);
                i = p;
            } else {
                break;
            }
        }
    }

    fn sift_down(&mut self, mut i: usize) {
        let n = self.data.len();
        loop {
            let (mut sm, l, r) = (i, 2*i+1, 2*i+2);
            if l < n && self.data[l] < self.data[sm] { sm = l; }
            if r < n && self.data[r] < self.data[sm] { sm = r; }
            if sm == i { break; }
            self.data.swap(i, sm);
            i = sm;
        }
    }
}

fn main() {
    let mut h = MinHeap::from_vec(vec![5, 3, 8, 1, 2]);
    println!("{} {} {}", h.pop().unwrap(), h.pop().unwrap(), h.pop().unwrap());
    // 1 2 3
}
Rust's BinaryHeap is a max-heap only. The workaround is wrapping values in std::cmp::Reverse or implementing a newtype with flipped Ord. Our custom MinHeap avoids this ceremony entirely.

Go — implement heap.Interface (Len, Less, Swap, Push, Pop)

package main

import "fmt"

type MinHeap[T interface{ ~int | ~float64 | ~string }] struct {
    data []T
}

func NewMinHeap[T interface{ ~int | ~float64 | ~string }](items ...T) *MinHeap[T] {
    h := &MinHeap[T]{data: append([]T{}, items...)}
    n := len(h.data)
    for i := n/2 - 1; i >= 0; i-- {
        h.siftDown(i)
    }
    return h
}

func (h *MinHeap[T]) Len() int { return len(h.data) }

func (h *MinHeap[T]) Push(val T) {
    h.data = append(h.data, val)
    h.siftUp(len(h.data) - 1)
}

func (h *MinHeap[T]) Pop() (T, bool) {
    if len(h.data) == 0 {
        var zero T
        return zero, false
    }
    val := h.data[0]
    last := len(h.data) - 1
    h.data[0] = h.data[last]
    h.data = h.data[:last]
    if len(h.data) > 0 {
        h.siftDown(0)
    }
    return val, true
}

func (h *MinHeap[T]) Peek() (T, bool) {
    if len(h.data) == 0 {
        var zero T
        return zero, false
    }
    return h.data[0], true
}

func (h *MinHeap[T]) siftUp(i int) {
    for i > 0 {
        p := (i - 1) / 2
        if h.data[i] < h.data[p] {
            h.data[i], h.data[p] = h.data[p], h.data[i]
            i = p
        } else {
            break
        }
    }
}

func (h *MinHeap[T]) siftDown(i int) {
    n := len(h.data)
    for {
        sm, l, r := i, 2*i+1, 2*i+2
        if l < n && h.data[l] < h.data[sm] { sm = l }
        if r < n && h.data[r] < h.data[sm] { sm = r }
        if sm == i { break }
        h.data[i], h.data[sm] = h.data[sm], h.data[i]
        i = sm
    }
}

func main() {
    h := NewMinHeap(5, 3, 8, 1, 2)
    v1, _ := h.Pop()
    v2, _ := h.Pop()
    v3, _ := h.Pop()
    fmt.Println(v1, v2, v3) // 1 2 3
}
Go's stdlib container/heap requires you to implement the heap.Interface: Len(), Less(i, j), Swap(i, j), Push(x), and Pop(). It is the most ceremony of any language — you write five methods before calling heap.Push. Our generic version is simpler but limited to ordered types.

TypeScript — comparator-based MinHeap

class MinHeap<T> {
  private data: T[] = [];

  constructor(private cmp: (a: T, b: T) => number, items?: T[]) {
    if (items) {
      this.data = [...items];
      for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--)
        this.siftDown(i);
    }
  }

  get size(): number { return this.data.length; }

  push(val: T): void {
    this.data.push(val);
    this.siftUp(this.data.length - 1);
  }

  pop(): T | undefined {
    if (this.data.length === 0) return undefined;
    const val = this.data[0];
    const last = this.data.pop()!;
    if (this.data.length > 0) {
      this.data[0] = last;
      this.siftDown(0);
    }
    return val;
  }

  peek(): T | undefined { return this.data[0]; }

  private siftUp(i: number): void {
    while (i > 0) {
      const p = (i - 1) >>> 1;
      if (this.cmp(this.data[i], this.data[p]) < 0) {
        [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
        i = p;
      } else break;
    }
  }

  private siftDown(i: number): void {
    const n = this.data.length;
    while (true) {
      let sm = i, l = 2*i+1, r = 2*i+2;
      if (l < n && this.cmp(this.data[l], this.data[sm]) < 0) sm = l;
      if (r < n && this.cmp(this.data[r], this.data[sm]) < 0) sm = r;
      if (sm === i) break;
      [this.data[i], this.data[sm]] = [this.data[sm], this.data[i]];
      i = sm;
    }
  }
}

const h = new MinHeap<number>((a, b) => a - b, [5, 3, 8, 1, 2]);
console.log(h.pop(), h.pop(), h.pop()); // 1 2 3
JavaScript/TypeScript has no built-in heap or priority queue. You always roll your own or use a library. The comparator pattern (a, b) => a - b follows the same convention as Array.sort — negative means a comes first.

Key observations

  • C++ and Rust default to max-heap, everyone else defaults to min (or lets you choose). This trips up beginners constantly. In C++ you need priority_queue<int, vector<int>, greater<int>> for a min-heap. In Rust you wrap values in Reverse.
  • Go's heap.Interface is the most ceremony. You implement five methods on your own type, then call heap.Push(&h, val) as a package-level function. The tradeoff is full control over storage and ordering.
  • Python's heapq is minimal but fragile. It operates on a plain list with no encapsulation. Nothing stops you from calling list.append() instead of heapq.heappush() and silently breaking the invariant.
  • TypeScript has no stdlib heap at all. You either build one (as we did) or pull in a library. This is a recurring gap in JavaScript's data structure support.
  • Heapify in O(n) is a universal trick. All implementations above support constructing a heap from an unsorted collection in linear time. The key insight: half the elements are leaves and need no sifting.

Decision guide

  • Dijkstra / event simulation / top-K: use a min-heap (watch out for C++ and Rust defaulting to max)
  • Go project: accept the ceremony and implement heap.Interface, or write a generic wrapper like ours
  • TypeScript project: copy the comparator-based MinHeap above or use a library like mnemonist
  • Need decrease-key (Fibonacci heap): none of the stdlibs support this — you need a custom implementation or indexed priority queue
