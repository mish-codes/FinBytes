Two fundamental access patterns

A stack is last-in, first-out (LIFO). Push adds to the top, pop removes from the top. Think of a stack of plates — you always take the top one. An array-backed stack is trivial: push appends, pop removes the last element. Both are O(1).

A queue is first-in, first-out (FIFO). Enqueue adds to the back, dequeue removes from the front. A naive array-backed queue has a problem: removing from the front shifts every element left — O(n). The fix is a circular buffer.

A circular buffer uses a fixed-size array with two indices: head (front) and tail (back). Both advance forward and wrap around using modulo. Enqueue and dequeue are both O(1) with no shifting. When the buffer fills, you can either reject new items (bounded queue) or resize (unbounded queue).

Most stdlibs have a stack (or use an array/list as one) and a deque or ring buffer for queues. The question is when the stdlib is enough versus when you need to build your own — typically for bounded queues, lock-free concurrency, or memory-constrained systems.

Python — list as stack, circular buffer for queue

class Stack:
    def __init__(self):
        self._data = []

    def push(self, val):
        self._data.append(val)

    def pop(self):
        if not self._data:
            raise IndexError("pop from empty stack")
        return self._data.pop()

    def peek(self):
        if not self._data:
            raise IndexError("peek at empty stack")
        return self._data[-1]

    def __len__(self):
        return len(self._data)


class CircularQueue:
    def __init__(self, capacity):
        self._buf = [None] * capacity
        self._cap = capacity
        self._head = 0
        self._tail = 0
        self._size = 0

    def enqueue(self, val):
        if self._size == self._cap:
            raise OverflowError("queue is full")
        self._buf[self._tail] = val
        self._tail = (self._tail + 1) % self._cap
        self._size += 1

    def dequeue(self):
        if self._size == 0:
            raise IndexError("dequeue from empty queue")
        val = self._buf[self._head]
        self._buf[self._head] = None
        self._head = (self._head + 1) % self._cap
        self._size -= 1
        return val

    def peek(self):
        if self._size == 0:
            raise IndexError("peek at empty queue")
        return self._buf[self._head]

    def is_full(self):
        return self._size == self._cap

    def __len__(self):
        return self._size

# Usage
s = Stack()
s.push(1); s.push(2); s.push(3)
print(s.pop(), s.peek())  # 3 2

q = CircularQueue(3)
q.enqueue("A"); q.enqueue("B"); q.enqueue("C")
print(q.dequeue(), q.dequeue())  # A B
q.enqueue("D")  # wraps around
print(q.dequeue())  # C
Python list.append() and list.pop() are amortized O(1), making list a perfectly fine stack. For queues, list.pop(0) is O(n) — use collections.deque in production, which is a doubly-linked list of fixed-size blocks with O(1) operations at both ends.

C# — array-backed stack and circular queue

public class Stack<T>
{
    private T[] _buf;
    private int _top = -1;

    public Stack(int cap = 16) => _buf = new T[cap];
    public int Count => _top + 1;

    public void Push(T val)
    {
        if (_top + 1 == _buf.Length) Array.Resize(ref _buf, _buf.Length * 2);
        _buf[++_top] = val;
    }

    public T Pop() => _top >= 0
        ? _buf[_top--]
        : throw new InvalidOperationException("empty");

    public T Peek() => _top >= 0
        ? _buf[_top]
        : throw new InvalidOperationException("empty");
}

public class CircularQueue<T>
{
    private readonly T[] _buf;
    private int _head, _tail, _size;
    private readonly int _cap;

    public CircularQueue(int cap)
    {
        _cap = cap;
        _buf = new T[cap];
    }

    public int Count => _size;
    public bool IsFull => _size == _cap;

    public void Enqueue(T val)
    {
        if (_size == _cap) throw new InvalidOperationException("full");
        _buf[_tail] = val;
        _tail = (_tail + 1) % _cap;
        _size++;
    }

    public T Dequeue()
    {
        if (_size == 0) throw new InvalidOperationException("empty");
        T val = _buf[_head];
        _buf[_head] = default!;
        _head = (_head + 1) % _cap;
        _size--;
        return val;
    }

    public T Peek() => _size > 0
        ? _buf[_head]
        : throw new InvalidOperationException("empty");
}

var s = new Stack<int>();
s.Push(1); s.Push(2); s.Push(3);
Console.WriteLine($"{s.Pop()} {s.Peek()}"); // 3 2

var q = new CircularQueue<string>(3);
q.Enqueue("A"); q.Enqueue("B"); q.Enqueue("C");
Console.WriteLine($"{q.Dequeue()} {q.Dequeue()}"); // A B
C#'s System.Collections.Generic.Stack<T> and Queue<T> are already array-backed with automatic resizing. Queue<T> uses a circular buffer internally — exactly what we built here.

C++ — vector-backed stack and circular buffer queue

#include <vector>
#include <stdexcept>
#include <array>
#include <optional>

template<typename T>
class Stack {
    std::vector<T> data_;
public:
    void push(T val) { data_.push_back(std::move(val)); }

    T pop() {
        if (data_.empty()) throw std::out_of_range("empty");
        T val = std::move(data_.back());
        data_.pop_back();
        return val;
    }

    const T& peek() const {
        if (data_.empty()) throw std::out_of_range("empty");
        return data_.back();
    }

    size_t size() const { return data_.size(); }
};

template<typename T, size_t Cap>
class CircularQueue {
    std::array<T, Cap> buf_{};
    size_t head_ = 0, tail_ = 0, size_ = 0;

public:
    void enqueue(T val) {
        if (size_ == Cap) throw std::overflow_error("full");
        buf_[tail_] = std::move(val);
        tail_ = (tail_ + 1) % Cap;
        ++size_;
    }

    T dequeue() {
        if (size_ == 0) throw std::out_of_range("empty");
        T val = std::move(buf_[head_]);
        head_ = (head_ + 1) % Cap;
        --size_;
        return val;
    }

    const T& peek() const {
        if (size_ == 0) throw std::out_of_range("empty");
        return buf_[head_];
    }

    bool is_full() const { return size_ == Cap; }
    size_t size() const { return size_; }
};

// Usage: Stack<int> s; CircularQueue<std::string, 128> q;
The template parameter Cap makes the circular queue fixed-size at compile time — no heap allocation, perfect for embedded or real-time systems. C++ std::stack is an adaptor over deque by default; std::queue is the same. Both are thin wrappers, not new data structures.

Rust — Vec stack and VecDeque-style circular queue

pub struct Stack<T> {
    data: Vec<T>,
}

impl<T> Stack<T> {
    pub fn new() -> Self { Self { data: Vec::new() } }
    pub fn push(&mut self, val: T) { self.data.push(val); }
    pub fn pop(&mut self) -> Option<T> { self.data.pop() }
    pub fn peek(&self) -> Option<&T> { self.data.last() }
    pub fn len(&self) -> usize { self.data.len() }
}

pub struct CircularQueue<T> {
    buf: Vec<Option<T>>,
    head: usize,
    tail: usize,
    size: usize,
    cap: usize,
}

impl<T> CircularQueue<T> {
    pub fn new(cap: usize) -> Self {
        let mut buf = Vec::with_capacity(cap);
        for _ in 0..cap { buf.push(None); }
        Self { buf, head: 0, tail: 0, size: 0, cap }
    }

    pub fn enqueue(&mut self, val: T) -> Result<(), &'static str> {
        if self.size == self.cap { return Err("queue is full"); }
        self.buf[self.tail] = Some(val);
        self.tail = (self.tail + 1) % self.cap;
        self.size += 1;
        Ok(())
    }

    pub fn dequeue(&mut self) -> Option<T> {
        if self.size == 0 { return None; }
        let val = self.buf[self.head].take();
        self.head = (self.head + 1) % self.cap;
        self.size -= 1;
        val
    }

    pub fn peek(&self) -> Option<&T> {
        if self.size == 0 { return None; }
        self.buf[self.head].as_ref()
    }

    pub fn is_full(&self) -> bool { self.size == self.cap }
    pub fn len(&self) -> usize { self.size }
}

fn main() {
    let mut s = Stack::new();
    s.push(1); s.push(2); s.push(3);
    println!("{} {:?}", s.pop().unwrap(), s.peek()); // 3 Some(2)

    let mut q = CircularQueue::new(3);
    q.enqueue("A").unwrap();
    q.enqueue("B").unwrap();
    q.enqueue("C").unwrap();
    println!("{} {}", q.dequeue().unwrap(), q.dequeue().unwrap()); // A B
}
Rust's Vec is a perfect stack — push and pop are built in. For queues, the stdlib provides VecDeque, which is a growable circular buffer. Our bounded version returns Result on enqueue — idiomatic Rust for fallible operations instead of throwing exceptions.

Go — slice as stack, channel as concurrent queue

package main

import (
    "errors"
    "fmt"
)

type Stack[T any] struct {
    data []T
}

func (s *Stack[T]) Push(val T)       { s.data = append(s.data, val) }
func (s *Stack[T]) Len() int         { return len(s.data) }

func (s *Stack[T]) Pop() (T, error) {
    if len(s.data) == 0 {
        var zero T
        return zero, errors.New("empty")
    }
    val := s.data[len(s.data)-1]
    s.data = s.data[:len(s.data)-1]
    return val, nil
}

func (s *Stack[T]) Peek() (T, error) {
    if len(s.data) == 0 {
        var zero T
        return zero, errors.New("empty")
    }
    return s.data[len(s.data)-1], nil
}

type CircularQueue[T any] struct {
    buf        []T
    head, tail int
    size, cap  int
}

func NewCircularQueue[T any](cap int) *CircularQueue[T] {
    return &CircularQueue[T]{buf: make([]T, cap), cap: cap}
}

func (q *CircularQueue[T]) Enqueue(val T) error {
    if q.size == q.cap { return errors.New("full") }
    q.buf[q.tail] = val
    q.tail = (q.tail + 1) % q.cap
    q.size++
    return nil
}

func (q *CircularQueue[T]) Dequeue() (T, error) {
    if q.size == 0 {
        var zero T
        return zero, errors.New("empty")
    }
    val := q.buf[q.head]
    q.head = (q.head + 1) % q.cap
    q.size--
    return val, nil
}

func (q *CircularQueue[T]) Peek() (T, error) {
    if q.size == 0 {
        var zero T
        return zero, errors.New("empty")
    }
    return q.buf[q.head], nil
}

func (q *CircularQueue[T]) IsFull() bool { return q.size == q.cap }
func (q *CircularQueue[T]) Len() int     { return q.size }

func main() {
    s := &Stack[int]{}
    s.Push(1); s.Push(2); s.Push(3)
    v, _ := s.Pop()
    p, _ := s.Peek()
    fmt.Println(v, p) // 3 2

    // Go channels are concurrent bounded queues out of the box
    ch := make(chan string, 3)
    ch <- "A"; ch <- "B"; ch <- "C"
    fmt.Println(<-ch, <-ch) // A B
}
Go slices work as stacks with append and reslicing. For concurrent queues, Go channels are the idiomatic choice — make(chan T, n) creates a bounded FIFO queue with built-in goroutine-safe blocking. No manual locking required. Our CircularQueue is useful for single-goroutine scenarios where channel overhead is unnecessary.

TypeScript — array stack and circular buffer queue

class Stack<T> {
  private data: T[] = [];

  push(val: T): void { this.data.push(val); }

  pop(): T | undefined { return this.data.pop(); }

  peek(): T | undefined { return this.data[this.data.length - 1]; }

  get size(): number { return this.data.length; }
}

class CircularQueue<T> {
  private buf: (T | undefined)[];
  private head = 0;
  private tail = 0;
  private _size = 0;

  constructor(private cap: number) {
    this.buf = new Array(cap);
  }

  enqueue(val: T): boolean {
    if (this._size === this.cap) return false;
    this.buf[this.tail] = val;
    this.tail = (this.tail + 1) % this.cap;
    this._size++;
    return true;
  }

  dequeue(): T | undefined {
    if (this._size === 0) return undefined;
    const val = this.buf[this.head];
    this.buf[this.head] = undefined;
    this.head = (this.head + 1) % this.cap;
    this._size--;
    return val;
  }

  peek(): T | undefined {
    return this._size > 0 ? this.buf[this.head] : undefined;
  }

  get isFull(): boolean { return this._size === this.cap; }
  get size(): number { return this._size; }
}

// Usage
const s = new Stack<number>();
s.push(1); s.push(2); s.push(3);
console.log(s.pop(), s.peek()); // 3 2

const q = new CircularQueue<string>(3);
q.enqueue("A"); q.enqueue("B"); q.enqueue("C");
console.log(q.dequeue(), q.dequeue()); // A B
q.enqueue("D"); // wraps around
console.log(q.dequeue()); // C
JavaScript Array.push() and Array.pop() make arrays perfect stacks. For queues, Array.shift() is O(n) because it re-indexes every element. Our circular buffer avoids this. In Node.js, if you need a production queue, consider a linked-list-based deque or a library like denque.

Key observations

  • Stack is trivially an array in every language. No language requires a custom stack implementation for normal use. The stdlib is always sufficient — push/pop on a dynamic array is O(1) amortized everywhere.
  • Queue is where things diverge. Python, C#, and Rust have efficient deques/circular buffers in the stdlib. TypeScript has nothing — Array.shift() is O(n). Go sidesteps the problem with channels for concurrent use.
  • Go channels are concurrent queues by design. A buffered channel (make(chan T, n)) is a bounded, goroutine-safe FIFO queue. No other language gives you this as a language primitive — it is Go's most distinctive concurrency feature.
  • C++ std::deque is not a circular buffer. It is a sequence of fixed-size arrays. std::queue adapts it for FIFO access but does not expose circular-buffer semantics. For a true ring buffer, use Boost.CircularBuffer or build your own.
  • Build your own when you need bounded capacity, lock-free access, or zero allocation. The stdlib queue in most languages is unbounded and allocates on the heap. For real-time or embedded systems, a fixed-size circular buffer with no allocation is essential.

Decision guide

  • Stack for any purpose: use the stdlib (list, Vec, vector, array) — done
  • Single-threaded queue: Python deque, Rust VecDeque, C# Queue<T>, or build a circular buffer in TS/C++
  • Concurrent producer-consumer: Go channel, Python queue.Queue, Rust mpsc, C# ConcurrentQueue
  • Lock-free high-throughput queue: build your own or use a library (Rust crossbeam, C++ moodycamel) — this is an advanced topic covered in the Queues & Reliability series
  • Bounded buffer (backpressure): build a fixed-size circular queue as shown here, or use Go's buffered channel
