From key to bucket in O(1)

A hash map stores key-value pairs by hashing the key to an integer, then mapping that integer to a bucket index (hash % num_buckets). Each bucket holds a linked list of entries that hashed to the same index — this is separate chaining.

The load factor is num_entries / num_buckets. When it exceeds a threshold (typically 0.75), the table allocates more buckets and rehashes every entry. This keeps chains short and lookups close to O(1).

Two things make or break a hash map: the hash function (must distribute keys uniformly) and the equality function (must be consistent with hash — equal keys must produce equal hashes).

Alternative collision strategies include open addressing (linear probing, Robin Hood hashing), but chaining is the simplest to implement and reason about.

Python — chaining with lists

class HashMap:
    def __init__(self, initial_cap=8):
        self._buckets = [[] for _ in range(initial_cap)]
        self._len = 0

    def _bucket_index(self, key):
        return hash(key) % len(self._buckets)

    def put(self, key, value):
        idx = self._bucket_index(key)
        for i, (k, v) in enumerate(self._buckets[idx]):
            if k == key:
                self._buckets[idx][i] = (key, value)
                return
        self._buckets[idx].append((key, value))
        self._len += 1
        if self._len / len(self._buckets) > 0.75:
            self._rehash()

    def get(self, key, default=None):
        idx = self._bucket_index(key)
        for k, v in self._buckets[idx]:
            if k == key:
                return v
        return default

    def delete(self, key):
        idx = self._bucket_index(key)
        for i, (k, v) in enumerate(self._buckets[idx]):
            if k == key:
                self._buckets[idx].pop(i)
                self._len -= 1
                return True
        return False

    def _rehash(self):
        old = self._buckets
        self._buckets = [[] for _ in range(len(old) * 2)]
        self._len = 0
        for chain in old:
            for k, v in chain:
                self.put(k, v)

    def __len__(self):
        return self._len

m = HashMap()
m.put("AAPL", 185.50)
m.put("GOOG", 140.25)
print(m.get("AAPL"))   # 185.5
m.delete("AAPL")
print(m.get("AAPL"))   # None
Python's built-in hash() works on any immutable type. The real dict uses open addressing with perturbation, not chaining — and since 3.7 it preserves insertion order by maintaining a separate dense index array.

C# — generic HashMap with chaining

public class HashMap<TKey, TValue> where TKey : notnull
{
    private record Entry(TKey Key, TValue Value);
    private List<Entry>[] _buckets;
    private int _count;

    public HashMap(int cap = 8)
    {
        _buckets = new List<Entry>[cap];
        for (int i = 0; i < cap; i++) _buckets[i] = new();
    }

    public int Count => _count;

    private int Index(TKey key) =>
        (key.GetHashCode() & 0x7FFFFFFF) % _buckets.Length;

    public void Put(TKey key, TValue value)
    {
        int idx = Index(key);
        for (int i = 0; i < _buckets[idx].Count; i++)
        {
            if (_buckets[idx][i].Key.Equals(key))
            {
                _buckets[idx][i] = new Entry(key, value);
                return;
            }
        }
        _buckets[idx].Add(new Entry(key, value));
        _count++;
        if ((double)_count / _buckets.Length > 0.75) Rehash();
    }

    public TValue? Get(TKey key)
    {
        int idx = Index(key);
        foreach (var e in _buckets[idx])
            if (e.Key.Equals(key)) return e.Value;
        return default;
    }

    public bool Delete(TKey key)
    {
        int idx = Index(key);
        for (int i = 0; i < _buckets[idx].Count; i++)
        {
            if (_buckets[idx][i].Key.Equals(key))
            {
                _buckets[idx].RemoveAt(i);
                _count--;
                return true;
            }
        }
        return false;
    }

    private void Rehash()
    {
        var old = _buckets;
        _buckets = new List<Entry>[old.Length * 2];
        for (int i = 0; i < _buckets.Length; i++) _buckets[i] = new();
        _count = 0;
        foreach (var chain in old)
            foreach (var e in chain)
                Put(e.Key, e.Value);
    }
}
We mask with 0x7FFFFFFF to ensure a non-negative index — GetHashCode() can return negative values. The real Dictionary<K,V> uses open addressing with a prime-sized bucket array.

C++ — separate chaining with std::list

#include <vector>
#include <list>
#include <functional>
#include <optional>
#include <utility>

template<typename K, typename V>
class HashMap {
    using Entry = std::pair<K, V>;
    using Chain = std::list<Entry>;
    std::vector<Chain> buckets_;
    size_t count_ = 0;

    size_t index(const K& key) const {
        return std::hash<K>{}(key) % buckets_.size();
    }

    void rehash() {
        std::vector<Chain> old(buckets_.size() * 2);
        std::swap(buckets_, old);
        count_ = 0;
        for (auto& chain : old)
            for (auto& [k, v] : chain)
                put(std::move(k), std::move(v));
    }

public:
    explicit HashMap(size_t cap = 8) : buckets_(cap) {}
    size_t size() const { return count_; }

    void put(K key, V value) {
        size_t idx = index(key);
        for (auto& [k, v] : buckets_[idx]) {
            if (k == key) { v = std::move(value); return; }
        }
        buckets_[idx].emplace_back(std::move(key), std::move(value));
        ++count_;
        if (static_cast<double>(count_) / buckets_.size() > 0.75)
            rehash();
    }

    std::optional<V> get(const K& key) const {
        size_t idx = index(key);
        for (auto& [k, v] : buckets_[idx])
            if (k == key) return v;
        return std::nullopt;
    }

    bool erase(const K& key) {
        size_t idx = index(key);
        auto& chain = buckets_[idx];
        for (auto it = chain.begin(); it != chain.end(); ++it) {
            if (it->first == key) {
                chain.erase(it); --count_; return true;
            }
        }
        return false;
    }
};
The key type must support std::hash<K> and operator==. This is the same requirement as std::unordered_map. The standard library version also uses chaining, but with a single-linked list per bucket for lower memory overhead.

Rust — Hash + Eq trait bounds

use std::collections::hash_map::DefaultHasher;
use std::hash::{Hash, Hasher};

pub struct HashMap<K, V> {
    buckets: Vec<Vec<(K, V)>>,
    count: usize,
}

impl<K: Hash + Eq, V> HashMap<K, V> {
    pub fn new() -> Self {
        Self { buckets: (0..8).map(|_| Vec::new()).collect(), count: 0 }
    }

    fn index(&self, key: &K) -> usize {
        let mut h = DefaultHasher::new();
        key.hash(&mut h);
        h.finish() as usize % self.buckets.len()
    }

    pub fn put(&mut self, key: K, value: V) {
        let idx = self.index(&key);
        for entry in &mut self.buckets[idx] {
            if entry.0 == key {
                entry.1 = value;
                return;
            }
        }
        self.buckets[idx].push((key, value));
        self.count += 1;
        if self.count as f64 / self.buckets.len() as f64 > 0.75 {
            self.rehash();
        }
    }

    pub fn get(&self, key: &K) -> Option<&V> {
        let idx = self.index(key);
        self.buckets[idx].iter()
            .find(|(k, _)| k == key)
            .map(|(_, v)| v)
    }

    pub fn delete(&mut self, key: &K) -> bool {
        let idx = self.index(key);
        if let Some(pos) = self.buckets[idx].iter().position(|(k, _)| k == key) {
            self.buckets[idx].swap_remove(pos);
            self.count -= 1;
            true
        } else {
            false
        }
    }

    fn rehash(&mut self) {
        let new_cap = self.buckets.len() * 2;
        let old = std::mem::replace(
            &mut self.buckets,
            (0..new_cap).map(|_| Vec::new()).collect(),
        );
        self.count = 0;
        for chain in old {
            for (k, v) in chain {
                self.put(k, v);
            }
        }
    }

    pub fn len(&self) -> usize { self.count }
}
Rust requires Hash + Eq as trait bounds — the compiler rejects keys that do not implement both. The real std::collections::HashMap uses Robin Hood open addressing with SipHash (a DoS-resistant hash), not chaining.

Go — comparable constraint, manual hash

package main

import (
    "fmt"
    "hash/fnv"
)

type entry[K comparable, V any] struct {
    key K
    val V
}

type HashMap[K comparable, V any] struct {
    buckets [][]entry[K, V]
    count   int
    hashFn  func(K) uint64
}

func NewHashMap[K comparable, V any](h func(K) uint64) *HashMap[K, V] {
    return &HashMap[K, V]{
        buckets: make([][]entry[K, V], 8),
        hashFn:  h,
    }
}

func (m *HashMap[K, V]) index(key K) int {
    return int(m.hashFn(key) % uint64(len(m.buckets)))
}

func (m *HashMap[K, V]) Put(key K, val V) {
    idx := m.index(key)
    for i := range m.buckets[idx] {
        if m.buckets[idx][i].key == key {
            m.buckets[idx][i].val = val
            return
        }
    }
    m.buckets[idx] = append(m.buckets[idx], entry[K, V]{key, val})
    m.count++
    if float64(m.count)/float64(len(m.buckets)) > 0.75 {
        m.rehash()
    }
}

func (m *HashMap[K, V]) Get(key K) (V, bool) {
    idx := m.index(key)
    for _, e := range m.buckets[idx] {
        if e.key == key { return e.val, true }
    }
    var zero V
    return zero, false
}

func (m *HashMap[K, V]) Delete(key K) bool {
    idx := m.index(key)
    for i, e := range m.buckets[idx] {
        if e.key == key {
            m.buckets[idx] = append(m.buckets[idx][:i], m.buckets[idx][i+1:]...)
            m.count--
            return true
        }
    }
    return false
}

func (m *HashMap[K, V]) rehash() {
    old := m.buckets
    m.buckets = make([][]entry[K, V], len(old)*2)
    m.count = 0
    for _, chain := range old {
        for _, e := range chain { m.Put(e.key, e.val) }
    }
}

func hashStr(s string) uint64 {
    h := fnv.New64a()
    h.Write([]byte(s))
    return h.Sum64()
}

func main() {
    m := NewHashMap[string, float64](hashStr)
    m.Put("AAPL", 185.50)
    v, _ := m.Get("AAPL")
    fmt.Println(v) // 185.5
}
Go has no Hashable interface and no operator overloading. You must pass a hash function explicitly for custom types. The built-in map uses a runtime hash that works on any comparable type — but this hash is not exposed to user code.

TypeScript — string-key chaining

class HashMap<V> {
  private buckets: [string, V][][];
  private count = 0;

  constructor(private cap = 8) {
    this.buckets = Array.from({ length: cap }, () => []);
  }

  private hash(key: string): number {
    let h = 0;
    for (let i = 0; i < key.length; i++) {
      h = (h * 31 + key.charCodeAt(i)) | 0;
    }
    return (h >>> 0) % this.buckets.length;
  }

  put(key: string, value: V): void {
    const idx = this.hash(key);
    for (const entry of this.buckets[idx]) {
      if (entry[0] === key) { entry[1] = value; return; }
    }
    this.buckets[idx].push([key, value]);
    this.count++;
    if (this.count / this.buckets.length > 0.75) this.rehash();
  }

  get(key: string): V | undefined {
    const idx = this.hash(key);
    for (const [k, v] of this.buckets[idx]) {
      if (k === key) return v;
    }
    return undefined;
  }

  delete(key: string): boolean {
    const idx = this.hash(key);
    const chain = this.buckets[idx];
    for (let i = 0; i < chain.length; i++) {
      if (chain[i][0] === key) {
        chain.splice(i, 1);
        this.count--;
        return true;
      }
    }
    return false;
  }

  private rehash(): void {
    const old = this.buckets;
    this.buckets = Array.from({ length: old.length * 2 }, () => []);
    this.count = 0;
    for (const chain of old)
      for (const [k, v] of chain) this.put(k, v);
  }

  get size(): number { return this.count; }
}

const m = new HashMap<number>();
m.put("AAPL", 185.50);
console.log(m.get("AAPL")); // 185.5
We restrict keys to string because JavaScript has no universal .hashCode() for objects. The built-in Map uses identity-based hashing (reference equality for objects, value equality for primitives). For custom object keys, you would need to serialize to string first.

Key observations

  • Python dicts preserve insertion order since 3.7. This is a language guarantee, not an implementation detail. Go deliberately randomises iteration order to prevent code from depending on it. C++, Rust, and C# make no promise either way.
  • DoS-resistant hashing matters for web services. An attacker who knows your hash function can craft keys that all collide, turning O(1) into O(n). Python and Rust default to SipHash; Go seeds its hash at startup. C++ and C# leave this to the developer.
  • Go requires you to supply a hash function for custom types. There is no Hashable interface. The built-in map uses a runtime-internal hash that is not accessible to user code, making custom hash maps more verbose.
  • Robin Hood hashing (Rust) reduces worst-case probe length by stealing from rich buckets (short probe distance) and giving to poor buckets (long probe distance). This gives more predictable performance than pure chaining.

Decision guide

  • Need ordered keys: use a tree map (not a hash map) or Python dict (insertion order only, not sorted)
  • Public-facing API (user-supplied keys): ensure DoS-resistant hashing (Python, Rust, Go are safe by default)
  • Custom struct as key: implement hash + equality in all languages; Go is the most work (no operator overloading)
  • High-performance hot path: consider open addressing (fewer cache misses than chaining); Rust's standard HashMap is already Robin Hood
