Crashes in production are expensive

A crashed pricing server during market hours means missed trades, manual intervention, and potential regulatory exposure.

Common crash causes:

  • Null/invalid pointer dereference - SIGSEGV on Linux, access violation on Windows
  • Stack overflow - infinite recursion, or too much stack-allocated data
  • Out-of-memory - allocation fails; in C++ this throws std::bad_alloc
  • Integer divide by zero - a hardware exception (float just gives inf/nan)
  • Abort - assert() failure, unhandled exception

Post-mortem debugging: When a crash happens in production, a core dump (Linux) or minidump (Windows) captures the process state at the moment of crash. Load with gdb ./pricer core to inspect stack traces and variable values without reproducing the issue.

Python crash handling

import sys, traceback, logging, faulthandler, signal
from contextlib import contextmanager

# Enable fault handler - prints stack trace on SIGSEGV
faulthandler.enable()

# Global exception handler
def global_exception_handler(exc_type, exc_value, exc_traceback):
    logging.critical("Unhandled exception",
        exc_info=(exc_type, exc_value, exc_traceback))
    sys.__excepthook__(exc_type, exc_value, exc_traceback)
sys.excepthook = global_exception_handler

# Guard against None in hot path
from typing import Optional
def safe_price(pricer, trade) -> Optional[float]:
    if pricer is None or trade is None:
        logging.error(f"Null argument: pricer={pricer} trade={trade}")
        return None
    try:
        return pricer.price(trade)
    except Exception as e:
        logging.error(f"Pricing failed for {trade}: {e}", exc_info=True)
        return None

# Timeout context manager
@contextmanager
def timeout(seconds: float):
    def handler(signum, frame): raise TimeoutError(f"Timed out after {seconds}s")
    old = signal.signal(signal.SIGALRM, handler)
    signal.setitimer(signal.ITIMER_REAL, seconds)
    try: yield
    finally:
        signal.setitimer(signal.ITIMER_REAL, 0)
        signal.signal(signal.SIGALRM, old)

with timeout(5.0):
    result = price_exotic_option(trade)

// Unhandled exception handlers
AppDomain.CurrentDomain.UnhandledException += (_, args) => {

// <Nullable>enable</Nullable>

    if (pricer is null || trade is null) return null;
    try { return pricer.Price(trade); }
    catch (Exception ex) {

// Iterative instead of recursive to avoid stack overflow
void IterativeProcess(int n) {
    var stack = new Stack<int>();
    stack.Push(n);
    while (stack.Count > 0) {
        var item = stack.Pop();
        if (item <= 0) continue;
        stack.Push(item - 1);
    }
}

C++ crash handling

#include <csignal>
#include <cstdlib>
#include <execinfo.h>  // Linux only
#include <new>

// Signal handler for SIGSEGV / SIGABRT
void crash_handler(int sig) {
    void* bt[64];
    int   n = backtrace(bt, 64);
    char** syms = backtrace_symbols(bt, n);
    for (int i = 0; i < n; ++i) fprintf(stderr, "  %s
", syms[i]);
    free(syms);
    signal(sig, SIG_DFL); raise(sig);  // re-raise to generate core dump
}

// Custom new handler - called when malloc fails
void out_of_memory_handler() {
    fprintf(stderr, "FATAL: out of memory
");
    std::abort();
}

int main() {
    signal(SIGSEGV, crash_handler);
    signal(SIGABRT, crash_handler);
    std::set_new_handler(out_of_memory_handler);

    // Safe null check
    double* ptr = nullptr;
    if (ptr == nullptr) throw std::runtime_error("null pointer");

    // bad_alloc handling
    try {
        auto huge = std::make_unique<double[]>(1'000'000'000'000ULL);
    } catch (const std::bad_alloc& e) {
        // log, degrade, reject new work
    }
}
Enable core dumps in production: ulimit -c unlimited on Linux. When the process crashes it writes a core file you can load with gdb ./pricer core to inspect the stack and variable values at the moment of crash.

Rust crash prevention

Null dereference is impossible in safe Rust

// Rust has no null. Use Option<T> instead.
fn safe_price(pricer: Option<&Pricer>, trade: Option<&Trade>) -> Option<f64> {
    let pricer = pricer?;  // returns None if pricer is None
    let trade  = trade?;   // returns None if trade is None
    Some(pricer.price(trade))
}

// Pattern matching forces you to handle the None case
match safe_price(Some(&pricer), Some(&trade)) {
    Some(price) => println!("Price: {price}"),
    None        => eprintln!("Missing pricer or trade"),
}

// Exhaustive matching - compiler error if you miss a case
enum PriceResult {
    Ok(f64),
    StaleData,
    ModelError(String),
}

fn handle(result: PriceResult) {
    match result {
        PriceResult::Ok(p)          => publish(p),
        PriceResult::StaleData      => use_fallback(),
        PriceResult::ModelError(e)  => log_error(&e),
    }   // removing any arm is a compile error
}

Stack overflow and OOM handling

// Stack overflow: Rust panics with a clear message.
// Avoid deep recursion; use iteration or stacker crate for explicit stack growth.
fn iterative_process(n: u64) {
    let mut stack = vec![n];
    while let Some(item) = stack.pop() {
        if item == 0 { continue; }
        stack.push(item - 1);
    }
}

// OOM: catch_unwind for panic recovery (not for normal control flow)
use std::panic;
let result = panic::catch_unwind(|| {
    let huge: Vec<u8> = vec![0; 1_000_000_000_000]; // may panic on alloc failure
    huge.len()
});
match result {
    Ok(len)  => println!("Allocated {len} bytes"),
    Err(_)   => eprintln!("Allocation failed - degrading gracefully"),
}

// Custom panic hook for structured logging
panic::set_hook(Box::new(|info| {
    eprintln!("PANIC: {info}");
    // log to file, send alert, etc.
}));
Rust eliminates the entire class of null-pointer crashes at compile time. Option<T> forces explicit handling. Panics (the Rust equivalent of crashes) produce clear backtraces and can be caught at thread boundaries with catch_unwind, though this is not meant for normal error handling.

Go crash handling

Nil pointer and panic recovery

package main

import (
    "fmt"
    "log"
    "runtime/debug"
)

// Go panics on nil pointer dereference at runtime.
// Use recover() to catch panics at goroutine boundaries.
func safePriceHandler(trade Trade) (price float64, err error) {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("PANIC in pricing: %v\nStack: %s", r, debug.Stack())
            err = fmt.Errorf("pricing panic: %v", r)
        }
    }()
    price = computePrice(trade)
    return
}

// Guard against nil
func safePrice(pricer *Pricer, trade *Trade) (float64, error) {
    if pricer == nil {
        return 0, fmt.Errorf("pricer is nil")
    }
    if trade == nil {
        return 0, fmt.Errorf("trade is nil")
    }
    return pricer.Price(trade), nil
}

// Stack overflow: Go uses segmented / growable stacks.
// Goroutine stacks start small (few KB) and grow as needed.
// Infinite recursion eventually hits the stack limit and panics.
func iterativeProcess(n int) {
    stack := []int{n}
    for len(stack) > 0 {
        item := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        if item <= 0 { continue }
        stack = append(stack, item-1)
    }
}

// OOM: Go's runtime will crash the process.
// Use resource limits at the OS/container level.
// Monitor with runtime.ReadMemStats.
Go has nil pointers (unlike Rust), but recover() can catch panics at goroutine boundaries. Always put defer recover() at the top of every goroutine that handles external input. Go's growable stacks make stack overflow less common than in C/C++, but infinite recursion will still crash.

TypeScript crash prevention

Null and undefined safety with strict mode

// tsconfig.json: "strict": true enables strictNullChecks

// Without strict: crashes at runtime
function badPrice(pricer: Pricer, trade: Trade): number {
  return pricer.price(trade);  // pricer could be undefined - runtime crash
}

// With strict: compiler forces null checks
function safePrice(pricer: Pricer | null, trade: Trade | null): number | null {
  if (!pricer || !trade) return null;
  return pricer.price(trade);
}

// Optional chaining and nullish coalescing
const price = pricer?.price(trade) ?? fallbackPrice;

// Exhaustive type checking with never
type PriceResult =
  | { status: "ok"; value: number }
  | { status: "stale" }
  | { status: "error"; message: string };

function handle(result: PriceResult): void {
  switch (result.status) {
    case "ok":    publish(result.value); break;
    case "stale": useFallback(); break;
    case "error": logError(result.message); break;
    default: {
      const _exhaustive: never = result;  // compile error if case is missing
    }
  }
}

Unhandled exception and promise rejection

// Node.js: catch unhandled errors
process.on("uncaughtException", (err) => {
  console.error("Uncaught exception:", err);
  process.exit(1);  // exit after logging - state may be corrupt
});

process.on("unhandledRejection", (reason) => {
  console.error("Unhandled rejection:", reason);
  // Don't swallow - either handle or exit
});

// Async error boundary
async function priceWithRecovery(trade: Trade): Promise<number> {
  try {
    return await computePrice(trade);
  } catch (err) {
    console.error(`Pricing failed for ${trade.id}:`, err);
    return NaN;
  }
}
TypeScript's strictNullChecks catches null/undefined errors at compile time — the single most impactful tsconfig flag. Unlike Rust's Option, TypeScript's null safety is erased at runtime, so data from external APIs still needs runtime validation (e.g., with Zod or io-ts).

Key observations

  • Rust eliminates null dereference entirely — there is no null. Option<T> forces compile-time handling. This alone prevents the most common crash category across all other languages.
  • Go has nil pointers but recoverable panics via defer/recover. Growable goroutine stacks make stack overflow rare. The trade-off: nil checks are manual and easy to forget.
  • TypeScript with strictNullChecks approaches Rust-level null safety at compile time, but the checks are erased at runtime. External data boundaries are the weak point.
  • C++ has the best post-mortem debugging (core dumps + gdb), but the most crash-prone runtime — null dereference is undefined behaviour, not an exception.
  • Python and C# throw catchable exceptions on null access, making crashes recoverable but not preventable at compile time.

Decision guide

  • Crash-proof by construction: Rust (no null, exhaustive matching)
  • Graceful degradation under failure: Go (recover) or C# (structured exceptions)
  • Post-mortem forensics: C++ (core dumps) or Rust (panic backtraces)
  • Rapid development with safety net: Python (exceptions for everything) or TypeScript (strict mode)
  • Production service with crash isolation: Go (per-goroutine recovery)
