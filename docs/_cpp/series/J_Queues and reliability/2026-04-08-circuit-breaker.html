Cascading failures

When a downstream service is slow or failing, naively retrying every request amplifies the problem. Every caller blocks waiting for a timeout; thread pools fill up; the calling system fails too. A circuit breaker detects the failure and short-circuits calls immediately, giving the downstream system time to recover.

Three states:

  • Closed: normal operation. Requests pass through. Failures are counted.
  • Open: failure threshold exceeded. Requests fail immediately without calling downstream. A timer runs.
  • Half-open: timer expired. One test request is allowed through. Success closes the circuit; failure re-opens it.

Finance application: a risk engine calls a market data feed. The feed becomes slow (network issue). Without a circuit breaker, every pricing request blocks for 30 seconds. With one, after 5 failures the circuit opens and the risk engine immediately uses stale (last-known-good) prices.

Always alert when a circuit opens. It is a production incident, not a silent fallback.

Python circuit breaker

import time, threading
from enum import Enum, auto
from typing import Callable, TypeVar

T = TypeVar("T")

class State(Enum):
    CLOSED = auto(); OPEN = auto(); HALF_OPEN = auto()

class CircuitBreaker:
    def __init__(self, failure_threshold=5, recovery_timeout=30.0):
        self._threshold = failure_threshold; self._timeout = recovery_timeout
        self._state = State.CLOSED; self._failures = 0; self._opened_at = 0.0
        self._lock = threading.Lock()

    def call(self, fn: Callable[[], T]) -> T:
        with self._lock:
            if self._state == State.OPEN:
                if time.monotonic() - self._opened_at > self._timeout:
                    self._state = State.HALF_OPEN
                else:
                    raise RuntimeError("Circuit open - using fallback")
        try:
            result = fn()
            with self._lock: self._failures = 0; self._state = State.CLOSED
            return result
        except Exception as e:
            with self._lock:
                self._failures += 1
                if self._failures >= self._threshold:
                    self._state = State.OPEN; self._opened_at = time.monotonic()
            raise

breaker = CircuitBreaker(failure_threshold=5, recovery_timeout=30.0)

def get_price_safe(symbol: str) -> float:
    try:
        return breaker.call(lambda: market_feed.get_price(symbol))
    except RuntimeError:
        return last_known_price.get(symbol, float("nan"))

C++ circuit breaker

#include <chrono>
#include <mutex>
#include <functional>
#include <stdexcept>

enum class CBState { Closed, Open, HalfOpen };

class CircuitBreaker {
    int       threshold_;
    double    timeout_s_;
    CBState   state_     = CBState::Closed;
    int       failures_  = 0;
    double    opened_at_ = 0;
    std::mutex mtx_;

    double now() const {
        return std::chrono::duration<double>(
            std::chrono::steady_clock::now().time_since_epoch()).count();
    }
public:
    CircuitBreaker(int threshold = 5, double timeout_s = 30.0)
        : threshold_(threshold), timeout_s_(timeout_s) {}

    template<typename F>
    auto call(F fn) -> decltype(fn()) {
        {
            std::lock_guard lock(mtx_);
            if (state_ == CBState::Open) {
                if (now() - opened_at_ > timeout_s_) state_ = CBState::HalfOpen;
                else throw std::runtime_error("Circuit open");
            }
        }
        try {
            auto result = fn();
            std::lock_guard lock(mtx_);
            failures_ = 0; state_ = CBState::Closed;
            return result;
        } catch (...) {
            std::lock_guard lock(mtx_);
            if (++failures_ >= threshold_) { state_ = CBState::Open; opened_at_ = now(); }
            throw;
        }
    }
    CBState state() const { std::lock_guard l(mtx_); return state_; }
};

CircuitBreaker breaker(5, 30.0);
double get_price_safe(const std::string& sym) {
    try { return breaker.call([&]{ return feed.get_price(sym); }); }
    catch (...) { return last_known_price.at(sym); }
}
Circuit breakers should be monitored and alarmed. An open circuit is a production incident. Alert immediately when a circuit opens so an engineer can investigate - do not silently fall back to stale data without notifying anyone.

Rust circuit breaker

use std::sync::Mutex;
use std::time::Instant;

#[derive(Clone, Copy, PartialEq)]
enum CBState { Closed, Open, HalfOpen }

struct CircuitBreaker {
    threshold:  u32,
    timeout:    std::time::Duration,
    state:      Mutex<(CBState, u32, Option<Instant>)>, // (state, failures, opened_at)
}

impl CircuitBreaker {
    fn new(threshold: u32, timeout_s: f64) -> Self {
        Self {
            threshold,
            timeout: std::time::Duration::from_secs_f64(timeout_s),
            state: Mutex::new((CBState::Closed, 0, None)),
        }
    }

    fn call<F, T>(&self, f: F) -> Result<T, Box<dyn std::error::Error>>
    where
        F: FnOnce() -> Result<T, Box<dyn std::error::Error>>,
    {
        // Check state
        {
            let mut guard = self.state.lock().unwrap();
            let (ref mut state, _, ref opened_at) = *guard;
            if *state == CBState::Open {
                if let Some(at) = opened_at {
                    if at.elapsed() > self.timeout {
                        *state = CBState::HalfOpen;
                    } else {
                        return Err("Circuit open".into());
                    }
                }
            }
        }

        match f() {
            Ok(val) => {
                let mut guard = self.state.lock().unwrap();
                guard.0 = CBState::Closed;
                guard.1 = 0;
                Ok(val)
            }
            Err(e) => {
                let mut guard = self.state.lock().unwrap();
                guard.1 += 1;
                if guard.1 >= self.threshold {
                    guard.0 = CBState::Open;
                    guard.2 = Some(Instant::now());
                    eprintln!("Circuit OPENED after {} failures", guard.1);
                }
                Err(e)
            }
        }
    }
}

// Usage with fallback
fn get_price_safe(breaker: &CircuitBreaker, symbol: &str) -> f64 {
    match breaker.call(|| market_feed_get_price(symbol)) {
        Ok(price) => price,
        Err(_) => last_known_prices.get(symbol).copied().unwrap_or(f64::NAN),
    }
}
Rust's Result type makes circuit breaker integration natural — every call returns Result<T, E> and the caller is forced to handle both paths. No unchecked exceptions can bypass the breaker. The Mutex ensures thread safety without the manual lock/unlock ceremony of C++.

Go circuit breaker

package main

import (
    "errors"
    "log"
    "sync"
    "time"
)

type CBState int

const (
    Closed CBState = iota
    Open
    HalfOpen
)

type CircuitBreaker struct {
    mu        sync.Mutex
    state     CBState
    failures  int
    threshold int
    timeout   time.Duration
    openedAt  time.Time
}

func NewCircuitBreaker(threshold int, timeout time.Duration) *CircuitBreaker {
    return &CircuitBreaker{threshold: threshold, timeout: timeout}
}

var ErrCircuitOpen = errors.New("circuit open")

func (cb *CircuitBreaker) Call(fn func() error) error {
    cb.mu.Lock()
    if cb.state == Open {
        if time.Since(cb.openedAt) > cb.timeout {
            cb.state = HalfOpen
        } else {
            cb.mu.Unlock()
            return ErrCircuitOpen
        }
    }
    cb.mu.Unlock()

    err := fn()

    cb.mu.Lock()
    defer cb.mu.Unlock()
    if err != nil {
        cb.failures++
        if cb.failures >= cb.threshold {
            cb.state = Open
            cb.openedAt = time.Now()
            log.Printf("Circuit OPENED after %d failures", cb.failures)
        }
        return err
    }
    cb.failures = 0
    cb.state = Closed
    return nil
}

func (cb *CircuitBreaker) State() CBState {
    cb.mu.Lock()
    defer cb.mu.Unlock()
    return cb.state
}

// Usage with fallback
func getPriceSafe(cb *CircuitBreaker, symbol string) float64 {
    var price float64
    err := cb.Call(func() error {
        var e error
        price, e = marketFeed.GetPrice(symbol)
        return e
    })
    if err != nil {
        if cached, ok := lastKnownPrices[symbol]; ok {
            return cached
        }
        return math.NaN()
    }
    return price
}

// Monitor circuit state — goroutine logs state transitions
func monitorCircuit(cb *CircuitBreaker, name string, ctx context.Context) {
    prev := Closed
    ticker := time.NewTicker(time.Second)
    defer ticker.Stop()
    for {
        select {
        case <-ctx.Done():
            return
        case <-ticker.C:
            curr := cb.State()
            if curr != prev {
                log.Printf("Circuit %s: %v -> %v", name, prev, curr)
                prev = curr
            }
        }
    }
}
Go's error return pattern fits circuit breakers naturally — Call() returns error, and the caller checks immediately. The monitoring goroutine with context.Context for clean shutdown is idiomatic Go. For production, the sony/gobreaker library adds sliding windows and more sophisticated trip conditions.

TypeScript circuit breaker

enum CBState { Closed, Open, HalfOpen }

class CircuitBreaker {
  private state = CBState.Closed;
  private failures = 0;
  private openedAt = 0;

  constructor(
    private threshold: number = 5,
    private timeoutMs: number = 30_000,
    private onStateChange?: (from: CBState, to: CBState) => void
  ) {}

  async call<T>(fn: () => Promise<T>): Promise<T> {
    if (this.state === CBState.Open) {
      if (Date.now() - this.openedAt > this.timeoutMs) {
        this.transition(CBState.HalfOpen);
      } else {
        throw new Error("Circuit open");
      }
    }

    try {
      const result = await fn();
      this.failures = 0;
      if (this.state !== CBState.Closed) this.transition(CBState.Closed);
      return result;
    } catch (err) {
      this.failures++;
      if (this.failures >= this.threshold) {
        this.transition(CBState.Open);
        this.openedAt = Date.now();
      }
      throw err;
    }
  }

  private transition(to: CBState): void {
    const from = this.state;
    this.state = to;
    this.onStateChange?.(from, to);
  }

  get currentState(): CBState { return this.state; }
}

// Usage with fallback
const breaker = new CircuitBreaker(5, 30_000, (from, to) => {
  console.error(`Circuit ${CBState[from]} -> ${CBState[to]}`);
});

async function getPriceSafe(symbol: string): Promise<number> {
  try {
    return await breaker.call(() => marketFeed.getPrice(symbol));
  } catch {
    return lastKnownPrices.get(symbol) ?? NaN;
  }
}

// Wrap fetch with circuit breaker
async function resilientFetch(url: string): Promise<Response> {
  return breaker.call(async () => {
    const res = await fetch(url, { signal: AbortSignal.timeout(5000) });
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return res;
  });
}
TypeScript's async/await makes circuit breaker integration clean — the call() method wraps any async function. The AbortSignal.timeout() pattern ensures requests do not hang indefinitely even if the downstream never responds, which is critical for the half-open probe.

Key observations

  • C# Polly is the most mature circuit breaker ecosystem. Retry, circuit breaker, fallback, and bulkhead are composable policies. No other language has an equivalent single library that covers all resilience patterns.
  • Go's error return model is a natural fit. Every function returns error, so circuit breaker wrapping does not change the calling convention. The context.Context pattern gives you clean cancellation propagation when a circuit opens.
  • Rust forces you to handle the open-circuit case. Because call() returns Result, the compiler will not let you ignore a circuit-open error. In exception-based languages, an unhandled open-circuit exception can crash the process.
  • C++ has no standard circuit breaker library. You roll your own. This gives maximum control but means every team reinvents the same state machine. Consider extracting a shared library early.
  • TypeScript's single-threaded model simplifies state management — no mutex needed for the circuit breaker state. The risk is forgetting to add timeouts on the underlying fetch/HTTP calls, causing the half-open probe to hang.

Decision guide for circuit breakers

  • Microservice with many downstream calls: C# Polly or Go gobreaker — battle-tested, configurable, observable
  • Low-latency trading system: C++ hand-rolled breaker with lock-free state reads for the hot path
  • Safety-critical path (must handle open circuit): Rust — the compiler enforces fallback handling
  • Node.js API gateway: TypeScript with opossum — add AbortSignal timeouts on every downstream call
  • Always alert on circuit open. In every language, wire up a callback or log line when the circuit transitions to open. Silent fallback is dangerous in finance.
