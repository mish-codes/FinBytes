Two kinds of health

Liveness: is the process alive? Fails if the process is stuck in an infinite loop, deadlocked, or OOM. Correct response: restart the process.

Readiness: is the process ready to serve traffic? Fails if alive but warming up, draining, or dependencies unavailable. Correct response: stop routing traffic to it, but do not restart it.

Kubernetes uses both. A failed liveness probe triggers a restart. A failed readiness probe removes the pod from the service's endpoint list. For a pricing service: liveness = heartbeat still ticking; readiness = market data feed connected and vol surface loaded.

Silent failures: the hardest to detect. The process is running, the heartbeat is ticking, but it is processing incorrect data. For example: the feed is connected but returning stale prices. The process appears healthy but computes wrong Greeks. Staleness checks are essential.

Python health check server

from flask import Flask, jsonify
import threading, time

app    = Flask(__name__)
state  = {"ready": False, "feed_age_s": 0.0}
last_heartbeat = time.monotonic()

def processing_thread():
    global last_heartbeat
    while True:
        process_some_work()
        last_heartbeat = time.monotonic()
        time.sleep(0.1)

@app.get("/healthz")     # liveness
def liveness():
    age = time.monotonic() - last_heartbeat
    if age > 30.0:
        return jsonify({"status": "unhealthy", "reason": f"no heartbeat for {age:.0f}s"}), 503
    return jsonify({"status": "ok"})

@app.get("/readyz")      # readiness
def readiness():
    checks = {
        "feed_connected":     market_feed.is_connected(),
        "vol_surface_loaded": vol_surface is not None,
        "feed_fresh":         state["feed_age_s"] < 60.0,
    }
    if all(checks.values()):
        return jsonify({"status": "ready", "checks": checks})
    return jsonify({"status": "not_ready", "checks": checks}), 503

C++ health endpoint + watchdog

#include <atomic>
#include <chrono>
#include <string>

std::atomic<int64_t> heartbeat_ns{0};
std::atomic<int64_t> last_feed_ns{0};

void update_heartbeat() {
    heartbeat_ns.store(
        std::chrono::steady_clock::now().time_since_epoch().count(),
        std::memory_order_relaxed);
}

bool is_alive(std::chrono::seconds max_age = std::chrono::seconds(30)) {
    auto now  = std::chrono::steady_clock::now().time_since_epoch().count();
    auto last = heartbeat_ns.load(std::memory_order_relaxed);
    return (now - last) < std::chrono::nanoseconds(max_age).count();
}

bool is_feed_fresh(std::chrono::seconds max_age = std::chrono::seconds(60)) {
    auto now  = std::chrono::steady_clock::now().time_since_epoch().count();
    auto last = last_feed_ns.load(std::memory_order_relaxed);
    return (now - last) < std::chrono::nanoseconds(max_age).count();
}

struct HealthStatus { bool live; bool ready; std::string message; };

HealthStatus get_health() {
    bool live  = is_alive();
    bool ready = live && is_feed_fresh() && vol_surface_loaded.load();
    std::string msg = ready ? "ok" : !live ? "deadlocked" :
                      !is_feed_fresh() ? "feed stale" : "not ready";
    return {live, ready, msg};
}
Separate liveness and readiness endpoints. Kubernetes calls them on different schedules with different thresholds. A liveness failure triggers a restart - make sure liveness only fails for genuinely unrecoverable conditions, not transient dependency issues.

Rust health checks and watchdog

use std::sync::atomic::{AtomicI64, AtomicBool, Ordering};
use std::sync::Arc;
use std::time::{Duration, Instant};
use tokio::time::interval;
use axum::{Router, routing::get, Json};
use serde::Serialize;

// Shared health state
struct HealthState {
    last_heartbeat_ns: AtomicI64,
    last_feed_ns:      AtomicI64,
    vol_surface_loaded: AtomicBool,
}

impl HealthState {
    fn new() -> Self {
        Self {
            last_heartbeat_ns: AtomicI64::new(0),
            last_feed_ns: AtomicI64::new(0),
            vol_surface_loaded: AtomicBool::new(false),
        }
    }

    fn heartbeat(&self) {
        let now = Instant::now().elapsed().as_nanos() as i64;
        self.last_heartbeat_ns.store(now, Ordering::Relaxed);
    }

    fn is_alive(&self, max_age: Duration) -> bool {
        let now = Instant::now().elapsed().as_nanos() as i64;
        let last = self.last_heartbeat_ns.load(Ordering::Relaxed);
        (now - last) < max_age.as_nanos() as i64
    }

    fn is_feed_fresh(&self, max_age: Duration) -> bool {
        let now = Instant::now().elapsed().as_nanos() as i64;
        let last = self.last_feed_ns.load(Ordering::Relaxed);
        (now - last) < max_age.as_nanos() as i64
    }
}

#[derive(Serialize)]
struct HealthResponse {
    status: String,
    live:   bool,
    ready:  bool,
    #[serde(skip_serializing_if = "Option::is_none")]
    reason: Option<String>,
}

// Axum health endpoints
async fn liveness(state: axum::extract::State<Arc<HealthState>>) -> Json<HealthResponse> {
    let live = state.is_alive(Duration::from_secs(30));
    Json(HealthResponse {
        status: if live { "ok" } else { "unhealthy" }.into(),
        live, ready: false,
        reason: if !live { Some("no heartbeat".into()) } else { None },
    })
}

async fn readiness(state: axum::extract::State<Arc<HealthState>>) -> Json<HealthResponse> {
    let live = state.is_alive(Duration::from_secs(30));
    let feed_fresh = state.is_feed_fresh(Duration::from_secs(60));
    let vol_ok = state.vol_surface_loaded.load(Ordering::Relaxed);
    let ready = live && feed_fresh && vol_ok;
    let reason = if !live { Some("deadlocked") }
        else if !feed_fresh { Some("feed stale") }
        else if !vol_ok { Some("vol surface not loaded") }
        else { None };
    Json(HealthResponse {
        status: if ready { "ready" } else { "not_ready" }.into(),
        live, ready, reason: reason.map(String::from),
    })
}

// Watchdog task — kills the process if heartbeat stops
async fn watchdog(state: Arc<HealthState>, timeout: Duration) {
    let mut tick = interval(Duration::from_secs(5));
    loop {
        tick.tick().await;
        if !state.is_alive(timeout) {
            eprintln!("WATCHDOG: heartbeat timeout — aborting");
            std::process::abort();
        }
    }
}
Rust's axum web framework makes health endpoints concise. The Arc<HealthState> is shared between the processing task and the HTTP handlers. The watchdog runs as a separate tokio task and calls std::process::abort() if the heartbeat stops — Kubernetes then restarts the pod.

Go health checks and watchdog

package main

import (
    "context"
    "encoding/json"
    "log"
    "net/http"
    "sync/atomic"
    "time"
)

// Shared health state
type HealthState struct {
    lastHeartbeat  atomic.Int64 // unix nano
    lastFeedUpdate atomic.Int64
    volSurfaceOK   atomic.Bool
}

func (h *HealthState) Heartbeat() {
    h.lastHeartbeat.Store(time.Now().UnixNano())
}

func (h *HealthState) IsAlive(maxAge time.Duration) bool {
    last := h.lastHeartbeat.Load()
    return time.Now().UnixNano()-last < int64(maxAge)
}

func (h *HealthState) IsFeedFresh(maxAge time.Duration) bool {
    last := h.lastFeedUpdate.Load()
    return time.Now().UnixNano()-last < int64(maxAge)
}

type HealthResponse struct {
    Status string `json:"status"`
    Live   bool   `json:"live"`
    Ready  bool   `json:"ready"`
    Reason string `json:"reason,omitempty"`
}

func setupHealthEndpoints(state *HealthState) {
    // Liveness
    http.HandleFunc("/healthz", func(w http.ResponseWriter, r *http.Request) {
        live := state.IsAlive(30 * time.Second)
        resp := HealthResponse{Live: live}
        if live {
            resp.Status = "ok"
        } else {
            resp.Status = "unhealthy"
            resp.Reason = "no heartbeat"
            w.WriteHeader(http.StatusServiceUnavailable)
        }
        json.NewEncoder(w).Encode(resp)
    })

    // Readiness
    http.HandleFunc("/readyz", func(w http.ResponseWriter, r *http.Request) {
        live := state.IsAlive(30 * time.Second)
        feedOK := state.IsFeedFresh(60 * time.Second)
        volOK := state.volSurfaceOK.Load()
        ready := live && feedOK && volOK
        resp := HealthResponse{Live: live, Ready: ready}
        if ready {
            resp.Status = "ready"
        } else {
            resp.Status = "not_ready"
            w.WriteHeader(http.StatusServiceUnavailable)
            if !live { resp.Reason = "deadlocked" } else if !feedOK { resp.Reason = "feed stale" } else { resp.Reason = "vol surface not loaded" }
        }
        json.NewEncoder(w).Encode(resp)
    })
}

// Watchdog — goroutine that monitors heartbeat
func watchdog(ctx context.Context, state *HealthState, timeout time.Duration) {
    ticker := time.NewTicker(5 * time.Second)
    defer ticker.Stop()
    for {
        select {
        case <-ctx.Done():
            return
        case <-ticker.C:
            if !state.IsAlive(timeout) {
                log.Fatal("WATCHDOG: heartbeat timeout — exiting")
            }
        }
    }
}

// Staleness checker — detects silent failures
func stalenessChecker(ctx context.Context, state *HealthState) {
    ticker := time.NewTicker(10 * time.Second)
    defer ticker.Stop()
    for {
        select {
        case <-ctx.Done():
            return
        case <-ticker.C:
            if !state.IsFeedFresh(60 * time.Second) {
                log.Println("WARN: feed data is stale — prices may be incorrect")
            }
        }
    }
}
Go's net/http gives you health endpoints with zero dependencies. The watchdog and staleness checker run as goroutines with context.Context for clean shutdown. This is the pattern Kubernetes expects: /healthz for liveness, /readyz for readiness, both returning JSON with appropriate status codes.

TypeScript health checks and watchdog

import express from "express";

// Shared health state
class HealthState {
  private lastHeartbeat = Date.now();
  private lastFeedUpdate = 0;
  private _volSurfaceLoaded = false;

  heartbeat(): void { this.lastHeartbeat = Date.now(); }
  feedUpdate(): void { this.lastFeedUpdate = Date.now(); }
  set volSurfaceLoaded(v: boolean) { this._volSurfaceLoaded = v; }

  isAlive(maxAgeMs: number = 30_000): boolean {
    return Date.now() - this.lastHeartbeat < maxAgeMs;
  }

  isFeedFresh(maxAgeMs: number = 60_000): boolean {
    return Date.now() - this.lastFeedUpdate < maxAgeMs;
  }

  get isReady(): boolean {
    return this.isAlive() && this.isFeedFresh() && this._volSurfaceLoaded;
  }
}

const state = new HealthState();
const app = express();

// Liveness
app.get("/healthz", (_req, res) => {
  if (state.isAlive()) {
    res.json({ status: "ok", live: true });
  } else {
    res.status(503).json({ status: "unhealthy", live: false, reason: "no heartbeat" });
  }
});

// Readiness
app.get("/readyz", (_req, res) => {
  const live = state.isAlive();
  const feedOK = state.isFeedFresh();
  const ready = state.isReady;
  const reason = !live ? "deadlocked" : !feedOK ? "feed stale" : "vol surface not loaded";
  if (ready) {
    res.json({ status: "ready", live, ready });
  } else {
    res.status(503).json({ status: "not_ready", live, ready, reason });
  }
});

// Watchdog — exit if heartbeat stops (PM2/Kubernetes restarts)
function startWatchdog(timeoutMs: number = 30_000): NodeJS.Timeout {
  return setInterval(() => {
    if (!state.isAlive(timeoutMs)) {
      console.error("WATCHDOG: heartbeat timeout — exiting");
      process.exit(1);
    }
  }, 5000);
}

// Staleness detector for silent failures
function startStalenessChecker(): NodeJS.Timeout {
  return setInterval(() => {
    if (!state.isFeedFresh()) {
      console.warn("WARN: feed data is stale — prices may be incorrect");
    }
  }, 10_000);
}

app.listen(8080, () => {
  startWatchdog();
  startStalenessChecker();
  console.log("Health endpoints on :8080");
});
Node.js health endpoints with Express are simple and widely deployed. The watchdog calls process.exit(1) to trigger a restart by the process manager (PM2, Kubernetes, systemd). Because Node.js is single-threaded, a deadlock typically freezes the event loop entirely — the watchdog timer will not fire, so an external health checker (Kubernetes liveness probe) is the real safety net.

Key observations

  • Go's net/http requires zero dependencies for health endpoints. No framework, no middleware, no configuration. A /healthz endpoint is 10 lines of code. Combined with goroutines for watchdog and staleness checking, Go makes reliability monitoring feel like a first-class language feature.
  • C# ASP.NET has the most structured health check system. Built-in AddHealthChecks() with tagged checks, customizable responses, and automatic Kubernetes integration. The downside: heavier than Go's approach for simple services.
  • TypeScript has a subtle weakness: if the event loop blocks (infinite loop, heavy synchronous computation), the watchdog timer will never fire. The process appears hung but cannot detect its own failure. An external liveness probe is the real safety net.
  • Deadlock detection is hardest in single-threaded runtimes (Python GIL, Node.js event loop). In Go, a deadlocked goroutine does not block the health check goroutine because they run on different OS threads. This is a significant reliability advantage.
  • Staleness is the silent killer. Every language can detect "process is alive" easily. Detecting "process is alive but returning stale data" requires application-level checks — timestamp every data update and compare against a threshold.

Decision guide for health checks

  • Kubernetes deployment: Go or C# ASP.NET — both have first-class K8s health check support
  • Minimal dependency footprint: Go — stdlib only, no framework needed
  • Complex dependency tree (DB, feed, cache): C# ASP.NET — tagged health checks with per-dependency status
  • Always implement both liveness and readiness. A pricing service that is alive but has no feed connection should not receive traffic, but should not be restarted either
  • Always add staleness checks. "Feed connected" is not enough — check that the last tick arrived within an acceptable window
