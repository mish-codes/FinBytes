When the consumer cannot keep up

A market data feed can publish 1 million ticks per second. A pricer might handle 50,000 per second. Without backpressure, the queue grows without bound until the process is killed by OOM.

Three strategies:

  • Block: the producer waits when the queue is full. Correct but introduces latency into the producer - risky if the producer is time-sensitive.
  • Drop: discard new items when full. Lose data but the producer is never blocked. Appropriate when old data is less valuable than new (market data: drop old prices, keep latest).
  • Shed: reject work at the API level before it enters the queue. Caller receives an error and can retry. Appropriate for RPC-style requests where the caller can back off.

Key insight: backpressure is a signal, not an error. A queue reaching capacity is telling you the consumer is too slow. Monitor queue depth continuously - alert at 70%, page at 85%.

# Rate limiting at API level (shed)
import time
class RateLimitedPricer:
    def __init__(self, max_rps: int):
        self._max_rps = max_rps; self._window = []; self._lock = threading.Lock()
    def price(self, trade):
        now = time.monotonic()
        with self._lock:
            self._window = [t for t in self._window if now - t < 1.0]
            if len(self._window) >= self._max_rps:
                raise RuntimeError("Rate limit exceeded - retry later")
            self._window.append(now)
        return self._price_impl(trade)

C++ latest-value cache and drop counting

#include <atomic>
#include <array>
#include <optional>

// Latest-value cache: producer overwrites, consumer reads latest
// No queue needed for market data where only the latest price matters
template<typename T, std::size_t N>
class LatestValueCache {
    struct Slot { std::atomic<uint64_t> seq{0}; T value{}; };
    std::array<Slot, N> slots_;
public:
    void put(std::size_t idx, T val) {
        auto& s = slots_[idx % N];
        s.seq.fetch_add(1, std::memory_order_release);
        s.value = val;
        s.seq.fetch_add(1, std::memory_order_release);
    }
    std::optional<T> get(std::size_t idx) const {
        auto& s = slots_[idx % N];
        uint64_t seq1 = s.seq.load(std::memory_order_acquire);
        if (seq1 & 1) return std::nullopt;   // write in progress
        T val = s.value;
        uint64_t seq2 = s.seq.load(std::memory_order_acquire);
        if (seq1 != seq2) return std::nullopt;  // concurrent write
        return val;
    }
};

// Non-blocking push with drop counter
std::atomic<uint64_t> dropped_ticks{0};

// In producer:
if (!queue.push(tick, std::chrono::milliseconds(0))) {
    dropped_ticks.fetch_add(1, std::memory_order_relaxed);
}
Monitor your queue depth continuously. At 50% capacity the system is healthy. At 80% investigate. At 95% you are minutes from blocking the producer or running out of memory. Alert at 70%; page at 85%.

Rust backpressure

use tokio::sync::mpsc;
use std::sync::atomic::{AtomicU64, Ordering};
use std::sync::Arc;
use std::collections::HashMap;
use tokio::sync::Mutex;

// Block strategy — send().await blocks when channel is full
async fn block_strategy() {
    let (tx, mut rx) = mpsc::channel::<Tick>(1000);

    tokio::spawn(async move {
        for tick in simulate_feed() {
            tx.send(tick).await.unwrap(); // awaits when buffer full
        }
    });

    while let Some(tick) = rx.recv().await {
        process_tick(tick).await;
    }
}

// Drop strategy — try_send returns Err when full
static DROPPED: AtomicU64 = AtomicU64::new(0);

async fn drop_strategy(tx: mpsc::Sender<Tick>, ticks: Vec<Tick>) {
    for tick in ticks {
        if tx.try_send(tick).is_err() {
            DROPPED.fetch_add(1, Ordering::Relaxed);
        }
    }
}

// Latest-value cache — only keep the newest price per symbol
struct LatestValueCache {
    data: Arc<Mutex<HashMap<String, f64>>>,
}

impl LatestValueCache {
    fn new() -> Self {
        Self { data: Arc::new(Mutex::new(HashMap::new())) }
    }

    async fn put(&self, symbol: String, price: f64) {
        self.data.lock().await.insert(symbol, price); // overwrites old
    }

    async fn get_all(&self) -> HashMap<String, f64> {
        let mut guard = self.data.lock().await;
        let items = guard.clone();
        guard.clear();
        items
    }
}

// Shed strategy — rate limiter using token bucket
use tokio::time::{Instant, Duration};

struct RateLimiter {
    capacity: u32,
    tokens: Arc<Mutex<(u32, Instant)>>,
    refill_rate: u32, // tokens per second
}

impl RateLimiter {
    fn new(max_rps: u32) -> Self {
        Self {
            capacity: max_rps,
            tokens: Arc::new(Mutex::new((max_rps, Instant::now()))),
            refill_rate: max_rps,
        }
    }

    async fn try_acquire(&self) -> bool {
        let mut guard = self.tokens.lock().await;
        let (ref mut tokens, ref mut last) = *guard;
        let now = Instant::now();
        let elapsed = now.duration_since(*last).as_secs_f64();
        *tokens = (*tokens + (elapsed * self.refill_rate as f64) as u32)
            .min(self.capacity);
        *last = now;
        if *tokens > 0 { *tokens -= 1; true } else { false }
    }
}
Rust's mpsc::channel gives you the block-vs-drop choice via send().await (block) vs try_send() (drop). The compiler enforces that only one task owns the receiver, so you cannot accidentally create competing consumers on a single channel.

Go backpressure

package main

import (
    "sync"
    "sync/atomic"
    "time"
)

// Block strategy — default channel send blocks when full
func blockStrategy() {
    feed := make(chan Tick, 1000)

    go func() {
        for _, tick := range ticks {
            feed <- tick // blocks when 1000 items buffered
        }
        close(feed)
    }()

    for tick := range feed {
        processTick(tick)
    }
}

// Drop strategy — select with default
var dropped uint64

func dropStrategy(feed chan<- Tick, ticks []Tick) {
    for _, tick := range ticks {
        select {
        case feed <- tick:
            // sent
        default:
            atomic.AddUint64(&dropped, 1)
        }
    }
}

// Latest-value cache — only keep newest price per symbol
type LatestValueCache struct {
    mu   sync.Mutex
    data map[string]float64
}

func NewLatestValueCache() *LatestValueCache {
    return &LatestValueCache{data: make(map[string]float64)}
}

func (c *LatestValueCache) Put(symbol string, price float64) {
    c.mu.Lock()
    c.data[symbol] = price // overwrites old price
    c.mu.Unlock()
}

func (c *LatestValueCache) GetAll() map[string]float64 {
    c.mu.Lock()
    defer c.mu.Unlock()
    items := c.data
    c.data = make(map[string]float64)
    return items
}

// Shed strategy — rate limiter using token bucket
type RateLimiter struct {
    mu       sync.Mutex
    tokens   float64
    capacity float64
    refill   float64 // per second
    lastTime time.Time
}

func NewRateLimiter(maxRPS int) *RateLimiter {
    return &RateLimiter{
        tokens: float64(maxRPS), capacity: float64(maxRPS),
        refill: float64(maxRPS), lastTime: time.Now(),
    }
}

func (r *RateLimiter) Allow() bool {
    r.mu.Lock()
    defer r.mu.Unlock()
    now := time.Now()
    elapsed := now.Sub(r.lastTime).Seconds()
    r.tokens += elapsed * r.refill
    if r.tokens > r.capacity { r.tokens = r.capacity }
    r.lastTime = now
    if r.tokens >= 1 { r.tokens--; return true }
    return false
}

// Monitor queue depth
func monitorDepth(feed chan Tick, alertThreshold float64) {
    ticker := time.NewTicker(time.Second)
    defer ticker.Stop()
    for range ticker.C {
        usage := float64(len(feed)) / float64(cap(feed))
        if usage > alertThreshold {
            log.Printf("WARN: queue at %.0f%% capacity", usage*100)
        }
    }
}
Go's select with default gives you a one-line non-blocking send — the cleanest drop-on-full syntax in any language. Monitoring queue depth is equally trivial: len(ch) and cap(ch) are built into the language. No wrapper needed.

TypeScript backpressure

// Drop strategy — non-blocking push with counter
class DropOnFullQueue<T> {
  private queue: T[] = [];
  private _dropped = 0;

  constructor(private capacity: number) {}

  tryPush(item: T): boolean {
    if (this.queue.length >= this.capacity) {
      this._dropped++;
      return false;
    }
    this.queue.push(item);
    return true;
  }

  pop(): T | undefined { return this.queue.shift(); }
  get dropped(): number { return this._dropped; }
  get size(): number { return this.queue.length; }
}

// Latest-value cache — overwrites per symbol
class LatestValueCache {
  private data = new Map<string, number>();

  put(symbol: string, price: number): void {
    this.data.set(symbol, price); // overwrites old
  }

  getAll(): Map<string, number> {
    const items = new Map(this.data);
    this.data.clear();
    return items;
  }
}

// Shed strategy — rate limiter
class RateLimiter {
  private tokens: number;
  private lastRefill: number;

  constructor(
    private maxRPS: number,
    private capacity: number = maxRPS
  ) {
    this.tokens = capacity;
    this.lastRefill = Date.now();
  }

  allow(): boolean {
    const now = Date.now();
    const elapsed = (now - this.lastRefill) / 1000;
    this.tokens = Math.min(this.capacity, this.tokens + elapsed * this.maxRPS);
    this.lastRefill = now;
    if (this.tokens >= 1) { this.tokens--; return true; }
    return false;
  }
}

// Backpressure-aware async producer-consumer
class BackpressuredPipeline<T> {
  private queue: T[] = [];
  private resolvers: Array<() => void> = [];

  constructor(
    private capacity: number,
    private dropOldest: boolean = false
  ) {}

  async push(item: T): Promise<boolean> {
    if (this.queue.length >= this.capacity) {
      if (this.dropOldest) {
        this.queue.shift(); // drop oldest
        this.queue.push(item);
        return true;
      }
      // block until space
      await new Promise<void>(resolve => this.resolvers.push(resolve));
    }
    this.queue.push(item);
    return true;
  }

  pop(): T | undefined {
    const item = this.queue.shift();
    if (this.resolvers.length > 0) this.resolvers.shift()!();
    return item;
  }

  get utilisation(): number {
    return this.queue.length / this.capacity;
  }
}

// Monitor and alert
function monitorQueue<T>(queue: BackpressuredPipeline<T>): NodeJS.Timeout {
  return setInterval(() => {
    const usage = queue.utilisation;
    if (usage > 0.85) console.error(`CRITICAL: queue at ${(usage * 100).toFixed(0)}%`);
    else if (usage > 0.70) console.warn(`WARN: queue at ${(usage * 100).toFixed(0)}%`);
  }, 1000);
}
Node.js is single-threaded, so queue operations are atomic by default — no mutex needed. The tradeoff: you cannot truly block a producer. The idiomatic pattern is tryPush with a drop counter, or await-based backpressure that yields to the event loop.

Key observations

  • Go has the most ergonomic backpressure in any language. A buffered channel blocks producers when full (zero code), and select/default turns blocking into non-blocking drop in two lines. Queue depth is len(ch)/cap(ch) — no method call, no wrapper. This is Go's strongest category.
  • C#'s BoundedChannelOptions is the most configurable. Four drop modes (Wait, DropOldest, DropNewest, DropWrite) out of the box. Every other language requires manual implementation of drop policies.
  • Rust's ownership model prevents a subtle bug: you cannot accidentally have two consumers racing on the same channel without explicitly opting in via Arc<Mutex<Receiver>>. This eliminates an entire class of backpressure-related race conditions.
  • TypeScript cannot truly block a producer because Node.js is single-threaded. Blocking would freeze the event loop. The pattern is always non-blocking try-push or await-based cooperative yielding.
  • C++ gives you lock-free options that no other language can match for latency. When microseconds matter (market data hot path), a lock-free SPSC queue with atomic sequence numbers avoids mutex contention entirely.

Decision guide for backpressure

  • Market data where latest price wins: latest-value cache (all languages) — no queue needed
  • Order flow where every message matters: block-when-full (Go channel, Rust mpsc, C# Channel.Wait)
  • High-throughput feed with acceptable loss: drop-when-full with counter (Go select/default is cleanest)
  • API gateway with caller-managed retry: shed via rate limiter — return 429 and let the caller back off
  • Ultra-low-latency path: C++ lock-free SPSC with overwrite semantics
