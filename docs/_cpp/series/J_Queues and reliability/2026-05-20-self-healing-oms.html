The problem

An OMS connects to an exchange. Connections drop. Messages are lost. The process restarts. After any of these events, the OMS's view may be inconsistent with the exchange's. A self-healing OMS detects this automatically and recovers without manual intervention.

Connection lifecycle state machine

enum class ConnState { Disconnected, Connecting, Connected, Active, Recovering, Draining };

class ExchangeConnection {
    std::atomic<ConnState> state_{ConnState::Disconnected};
    std::atomic<uint64_t>  last_seq_rx_{0};
public:
    void on_message(const Message& msg) {
        uint64_t expected = last_seq_rx_.load() + 1;
        if (msg.sequence > expected)
            request_retransmit(expected, msg.sequence - 1);
        last_seq_rx_.store(msg.sequence, std::memory_order_relaxed);
        process_message(msg);
    }
    void on_heartbeat_timeout() {
        state_.store(ConnState::Disconnected, std::memory_order_release);
        reconnect_with_backoff();
    }
    void on_reconnect_success() {
        state_.store(ConnState::Recovering, std::memory_order_release);
        request_fill_history(last_seq_rx_.load());
    }
};

Position reconciliation

from dataclasses import dataclass

@dataclass
class ReconciliationResult:
    symbol: str; oms_qty: float; exch_qty: float; diff: float; action: str

def reconcile(oms: dict, exch: dict) -> list[ReconciliationResult]:
    results = []
    for sym in set(oms) | set(exch):
        o = oms.get(sym, 0.0); e = exch.get(sym, 0.0); diff = abs(o - e)
        if diff < 1e-6:
            action = "ok"
        elif diff < 10:
            oms[sym] = e  # exchange is authoritative for small discrepancies
            action = f"corrected: {o} -> {e}"
        else:
            action = f"ALERT: large discrepancy ({diff:.0f}) - manual review"
        results.append(ReconciliationResult(sym, o, e, diff, action))
    return results

Outbox pattern - guaranteed delivery

import sqlite3, json, time

class OutboxPublisher:
    def __init__(self, db_path: str):
        self._db = sqlite3.connect(db_path)
        self._db.execute(
            "CREATE TABLE IF NOT EXISTS outbox "
            "(id INTEGER PRIMARY KEY, payload TEXT NOT NULL, "
            " created_at REAL NOT NULL, acked_at REAL)"
        )

    def publish(self, payload: dict) -> int:
        row_id = self._db.execute(
            "INSERT INTO outbox (payload, created_at) VALUES (?, ?)",
            (json.dumps(payload), time.time())
        ).lastrowid
        self._db.commit()
        self._send_and_ack(row_id, payload)
        return row_id

    def _send_and_ack(self, row_id: int, payload: dict) -> None:
        try:
            exchange.send(payload)
            self._db.execute("UPDATE outbox SET acked_at=? WHERE id=?", (time.time(), row_id))
            self._db.commit()
        except Exception:
            pass  # background worker retries unacked messages

    def retry_unacked(self) -> int:
        rows = self._db.execute(
            "SELECT id, payload FROM outbox WHERE acked_at IS NULL ORDER BY id"
        ).fetchall()
        for row_id, payload_json in rows:
            self._send_and_ack(row_id, json.loads(payload_json))
        return len(rows)
Write to the outbox atomically with your state change (same DB transaction). A background poller retries unacknowledged messages. Nothing is lost on crash - even if the process dies between send and ack, the message is retried on restart.
Do not implement a self-healing system without comprehensive observability. You need to see current connection state, sequence numbers, reconciliation outcomes, outbox depth. A system that heals silently and logs nothing is harder to trust than one that fails loudly.

Rust self-healing OMS

use std::sync::atomic::{AtomicU64, Ordering};
use std::sync::Arc;
use std::collections::HashMap;
use tokio::sync::Mutex;
use tokio::time::{sleep, Duration};

// Connection lifecycle state machine
#[derive(Clone, Copy, Debug, PartialEq)]
enum ConnState {
    Disconnected, Connecting, Connected, Active, Recovering, Draining,
}

struct ExchangeConnection {
    state:       Arc<Mutex<ConnState>>,
    last_seq_rx: AtomicU64,
}

impl ExchangeConnection {
    fn new() -> Self {
        Self {
            state: Arc::new(Mutex::new(ConnState::Disconnected)),
            last_seq_rx: AtomicU64::new(0),
        }
    }

    async fn on_message(&self, msg: &Message) {
        let expected = self.last_seq_rx.load(Ordering::Relaxed) + 1;
        if msg.sequence > expected {
            self.request_retransmit(expected, msg.sequence - 1).await;
        }
        self.last_seq_rx.store(msg.sequence, Ordering::Relaxed);
        self.process_message(msg).await;
    }

    async fn on_heartbeat_timeout(&self) {
        *self.state.lock().await = ConnState::Disconnected;
        self.reconnect_with_backoff().await;
    }

    async fn on_reconnect_success(&self) {
        *self.state.lock().await = ConnState::Recovering;
        let seq = self.last_seq_rx.load(Ordering::Relaxed);
        self.request_fill_history(seq).await;
    }

    async fn reconnect_with_backoff(&self) {
        let mut delay = Duration::from_secs(1);
        loop {
            *self.state.lock().await = ConnState::Connecting;
            match self.try_connect().await {
                Ok(_) => {
                    self.on_reconnect_success().await;
                    return;
                }
                Err(e) => {
                    let jitter = delay.mul_f64(0.5 + rand::random::<f64>());
                    eprintln!("Reconnect failed: {}. Retry in {:?}", e, jitter);
                    sleep(jitter).await;
                    delay = delay.mul_f64(2.0).min(Duration::from_secs(60));
                }
            }
        }
    }
}

// Position reconciliation
#[derive(Debug)]
struct ReconciliationResult {
    symbol:   String,
    oms_qty:  f64,
    exch_qty: f64,
    diff:     f64,
    action:   String,
}

fn reconcile(
    oms: &mut HashMap<String, f64>,
    exch: &HashMap<String, f64>,
) -> Vec<ReconciliationResult> {
    let all_syms: std::collections::HashSet<&String> =
        oms.keys().chain(exch.keys()).collect();
    let mut results = Vec::new();
    for sym in all_syms {
        let o = *oms.get(sym).unwrap_or(&0.0);
        let e = *exch.get(sym).unwrap_or(&0.0);
        let diff = (o - e).abs();
        let action = if diff < 1e-6 {
            "ok".into()
        } else if diff < 10.0 {
            oms.insert(sym.clone(), e);
            format!("corrected: {} -> {}", o, e)
        } else {
            format!("ALERT: large discrepancy ({:.0}) — manual review", diff)
        };
        results.push(ReconciliationResult {
            symbol: sym.clone(), oms_qty: o, exch_qty: e, diff, action,
        });
    }
    results
}

// Outbox pattern — guaranteed delivery with sqlx
// INSERT INTO outbox (payload, created_at) VALUES ($1, NOW())
// Background task: SELECT * FROM outbox WHERE acked_at IS NULL ORDER BY id
// After successful send: UPDATE outbox SET acked_at = NOW() WHERE id = $1
Rust's enum ConnState combined with Mutex ensures state transitions are atomic and exhaustively handled. The reconciliation function takes &mut HashMap for the OMS side, making it clear that the OMS positions may be corrected in place. The compiler prevents accidentally mutating exchange positions (passed as &HashMap).

Go self-healing OMS

package main

import (
    "context"
    "database/sql"
    "encoding/json"
    "fmt"
    "log"
    "math"
    "math/rand"
    "sync"
    "sync/atomic"
    "time"
)

// Connection lifecycle
type ConnState int

const (
    StateDisconnected ConnState = iota
    StateConnecting
    StateConnected
    StateActive
    StateRecovering
    StateDraining
)

type ExchangeConnection struct {
    mu        sync.Mutex
    state     ConnState
    lastSeqRx atomic.Uint64
}

func (ec *ExchangeConnection) OnMessage(msg Message) {
    expected := ec.lastSeqRx.Load() + 1
    if msg.Sequence > expected {
        ec.requestRetransmit(expected, msg.Sequence-1)
    }
    ec.lastSeqRx.Store(msg.Sequence)
    ec.processMessage(msg)
}

func (ec *ExchangeConnection) OnHeartbeatTimeout(ctx context.Context) {
    ec.mu.Lock()
    ec.state = StateDisconnected
    ec.mu.Unlock()
    ec.reconnectWithBackoff(ctx)
}

func (ec *ExchangeConnection) reconnectWithBackoff(ctx context.Context) {
    delay := time.Second
    for {
        ec.mu.Lock()
        ec.state = StateConnecting
        ec.mu.Unlock()

        if err := ec.tryConnect(); err == nil {
            ec.mu.Lock()
            ec.state = StateRecovering
            ec.mu.Unlock()
            ec.requestFillHistory(ec.lastSeqRx.Load())
            return
        }

        jitter := time.Duration(float64(delay) * (0.5 + rand.Float64()))
        log.Printf("Reconnect failed. Retry in %v", jitter)
        select {
        case <-ctx.Done():
            return
        case <-time.After(jitter):
        }
        if delay < 60*time.Second {
            delay *= 2
        }
    }
}

// Position reconciliation
type ReconciliationResult struct {
    Symbol  string
    OmsQty  float64
    ExchQty float64
    Diff    float64
    Action  string
}

func reconcile(oms, exch map[string]float64) []ReconciliationResult {
    allSyms := make(map[string]bool)
    for s := range oms  { allSyms[s] = true }
    for s := range exch { allSyms[s] = true }

    var results []ReconciliationResult
    for sym := range allSyms {
        o := oms[sym]
        e := exch[sym]
        diff := math.Abs(o - e)
        var action string
        switch {
        case diff < 1e-6:
            action = "ok"
        case diff < 10:
            oms[sym] = e // exchange is authoritative
            action = fmt.Sprintf("corrected: %v -> %v", o, e)
        default:
            action = fmt.Sprintf("ALERT: large discrepancy (%.0f) — manual review", diff)
        }
        results = append(results, ReconciliationResult{sym, o, e, diff, action})
    }
    return results
}

// Outbox pattern — guaranteed delivery
type OutboxPublisher struct {
    db *sql.DB
}

func (op *OutboxPublisher) Publish(ctx context.Context, payload interface{}) (int64, error) {
    data, _ := json.Marshal(payload)
    result, err := op.db.ExecContext(ctx,
        "INSERT INTO outbox (payload, created_at) VALUES ($1, NOW())", string(data))
    if err != nil {
        return 0, err
    }
    id, _ := result.LastInsertId()
    go op.sendAndAck(ctx, id, payload) // fire and forget — poller catches failures
    return id, nil
}

func (op *OutboxPublisher) RetryUnacked(ctx context.Context) int {
    rows, _ := op.db.QueryContext(ctx,
        "SELECT id, payload FROM outbox WHERE acked_at IS NULL ORDER BY id")
    defer rows.Close()
    count := 0
    for rows.Next() {
        var id int64
        var payload string
        rows.Scan(&id, &payload)
        go op.sendAndAck(ctx, id, payload)
        count++
    }
    return count
}
Go's context.Context propagates through reconnection, reconciliation, and outbox operations — a single cancel() call cleanly shuts down the entire OMS. The select on ctx.Done() ensures reconnection backoff respects shutdown signals. This is the pattern that makes Go the natural choice for self-healing infrastructure.

TypeScript self-healing OMS

// Connection lifecycle state machine
type ConnState = "disconnected" | "connecting" | "connected" | "active" | "recovering" | "draining";

class ExchangeConnection {
  private state: ConnState = "disconnected";
  private lastSeqRx = 0;
  private abortController = new AbortController();

  async onMessage(msg: Message): Promise<void> {
    const expected = this.lastSeqRx + 1;
    if (msg.sequence > expected) {
      await this.requestRetransmit(expected, msg.sequence - 1);
    }
    this.lastSeqRx = msg.sequence;
    await this.processMessage(msg);
  }

  async onHeartbeatTimeout(): Promise<void> {
    this.state = "disconnected";
    await this.reconnectWithBackoff();
  }

  private async reconnectWithBackoff(): Promise<void> {
    let delayMs = 1000;
    while (!this.abortController.signal.aborted) {
      this.state = "connecting";
      try {
        await this.tryConnect();
        this.state = "recovering";
        await this.requestFillHistory(this.lastSeqRx);
        this.state = "active";
        return;
      } catch (err) {
        const jitter = delayMs * (0.5 + Math.random());
        console.error(`Reconnect failed: ${err}. Retry in ${(jitter/1000).toFixed(1)}s`);
        await new Promise(r => setTimeout(r, jitter));
        delayMs = Math.min(delayMs * 2, 60_000);
      }
    }
  }

  shutdown(): void { this.abortController.abort(); }
}

// Position reconciliation
interface ReconciliationResult {
  symbol:  string;
  omsQty:  number;
  exchQty: number;
  diff:    number;
  action:  string;
}

function reconcile(
  oms: Map<string, number>,
  exch: Map<string, number>,
): ReconciliationResult[] {
  const allSyms = new Set([...oms.keys(), ...exch.keys()]);
  const results: ReconciliationResult[] = [];

  for (const sym of allSyms) {
    const o = oms.get(sym) ?? 0;
    const e = exch.get(sym) ?? 0;
    const diff = Math.abs(o - e);
    let action: string;

    if (diff < 1e-6) {
      action = "ok";
    } else if (diff < 10) {
      oms.set(sym, e); // exchange is authoritative
      action = `corrected: ${o} -> ${e}`;
    } else {
      action = `ALERT: large discrepancy (${diff.toFixed(0)}) — manual review`;
    }
    results.push({ symbol: sym, omsQty: o, exchQty: e, diff, action });
  }
  return results;
}

// Outbox pattern with SQLite (better-sqlite3 for sync, or knex for async)
import Database from "better-sqlite3";

class OutboxPublisher {
  private db: Database.Database;

  constructor(dbPath: string) {
    this.db = new Database(dbPath);
    this.db.exec(`
      CREATE TABLE IF NOT EXISTS outbox (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        payload TEXT NOT NULL,
        created_at REAL NOT NULL,
        acked_at REAL
      )
    `);
  }

  publish(payload: unknown): number {
    const stmt = this.db.prepare(
      "INSERT INTO outbox (payload, created_at) VALUES (?, ?)"
    );
    const result = stmt.run(JSON.stringify(payload), Date.now() / 1000);
    this.sendAndAck(result.lastInsertRowid as number, payload);
    return result.lastInsertRowid as number;
  }

  retryUnacked(): number {
    const rows = this.db.prepare(
      "SELECT id, payload FROM outbox WHERE acked_at IS NULL ORDER BY id"
    ).all() as Array<{ id: number; payload: string }>;
    for (const row of rows) {
      this.sendAndAck(row.id, JSON.parse(row.payload));
    }
    return rows.length;
  }

  private sendAndAck(id: number, payload: unknown): void {
    try {
      exchange.send(payload);
      this.db.prepare("UPDATE outbox SET acked_at = ? WHERE id = ?")
        .run(Date.now() / 1000, id);
    } catch { /* background poller retries */ }
  }
}
TypeScript's union type for ConnState gives compile-time exhaustiveness checking if you use a switch statement. The AbortController pattern for shutdown is the Node.js equivalent of Go's context.Context — call abort() once and all pending operations respect it.

Key observations

  • Go is purpose-built for self-healing services. The context.Context propagates cancellation through reconnection, reconciliation, and outbox polling. Goroutines for each concern (heartbeat monitor, reconnect loop, outbox poller) run concurrently without thread pool configuration. A single cancel() call drains everything cleanly.
  • Rust's enum-based state machine is the most correct. The compiler enforces exhaustive matching on ConnState, so you cannot forget to handle a state. The &mut vs & distinction in reconciliation makes it clear which side gets corrected.
  • The outbox pattern is language-agnostic — it relies on the database transaction, not the language. Write the state change and the outbox row in the same transaction. A background poller retries unacked rows. This works identically in all languages.
  • TypeScript's AbortController is the closest analogue to Go's context but less pervasive — not every Node.js library respects it. In Go, virtually every stdlib and third-party function accepts a context.Context.
  • Sequence gap detection is critical in all languages. If you receive sequence 1001 after 999, immediately request retransmission of 1000. Without this, the OMS silently misses fills and positions drift.

Decision guide for self-healing OMS

  • Multi-connection OMS with many exchanges: Go — goroutines per connection, context for shutdown, channels for inter-component communication
  • Safety-critical state machine: Rust — exhaustive match on connection states, ownership prevents accidental mutation
  • Ultra-low-latency exchange gateway: C++ — lock-free sequence tracking, direct memory access to messages
  • Always implement the outbox pattern for guaranteed delivery. In-memory queues lose messages on crash. The database is your durability layer.
  • Always reconcile positions. Even with perfect sequence tracking, edge cases (race conditions, partial fills, exchange corrections) will cause drift. Daily reconciliation is not optional.
